2.1 Variables and Declarations

Definition: var, let, and const keywords

Variables are containers for storing data values. JavaScript provides three ways to declare variables:

1. var - The legacy keyword
// Declaring with var
var name = "Alice";
var age = 25;
var isActive = true;

// Redeclaration is allowed
var name = "Bob"; // No error!
2. let - Block-scoped variable
// Declaring with let
let count = 0;
let userName = "Charlie";

// Can be reassigned
count = 10;

// Cannot be redeclared in same scope
let count = 5; // SyntaxError: Identifier 'count' has already been declared
3. const - Constant reference
// Declaring with const
const PI = 3.14159;
const user = {id: 1, name: "Diana"};

// Cannot be reassigned
PI = 3.14; // TypeError: Assignment to constant variable

// BUT object properties can be modified!
user.name = "Eve"; // This works!
user.email = "eve@example.com"; // This also works!

// To make object truly immutable, use Object.freeze()
Object.freeze(user);
user.name = "Frank"; // No effect (in strict mode: TypeError)
üéØ Key Insight: Think of const as "this reference cannot change" not "this value cannot change". Objects and arrays declared with const can still have their contents modified!
Variable naming conventions

Good naming makes your code self-documenting. Follow these conventions:

Rules for Valid Names:
  • Must start with letter, underscore (_), or dollar sign ($)
  • Can contain letters, digits, underscores, dollar signs
  • Cannot contain spaces or special characters
  • Cannot be reserved keywords (if, else, function, etc.)
Recommended Conventions:
Type Convention Examples
Variables & Functions camelCase userName, getTotalPrice, isVerified
Constants UPPER_SNAKE_CASE MAX_USERS, API_ENDPOINT, DEFAULT_TIMEOUT
Classes PascalCase UserProfile, ShoppingCart
Private Properties _leadingUnderscore _privateData, _cache
Bad Examples (Avoid These!):
// ‚ùå Don't do this:
var 1stName = "John";     // Starts with number
var user-name = "Jane";   // Contains hyphen
var function = "invalid"; // Reserved keyword
var $name = "Invalid";    // Unnecessary $
var myVariable = 10;      // Too vague
var x = 5;                // Meaningless single letter
Good Examples:
// ‚úÖ Do this instead:
const MAX_LOGIN_ATTEMPTS = 3;
let userName = "Alice";
let userAge = 25;
function calculateDiscount(price, discountRate) {
    return price * (1 - discountRate);
}
const currentUser = {id: 1, name: "Bob"};
üí° Pro Tip: Use descriptive names that answer "What is this?" rather than "What does it do?". Better: userEmail than email if context isn't clear.
Hoisting behavior

Hoisting is JavaScript's default behavior of moving declarations to the top of their scope before code execution.

How Hoisting Works:

Only the declaration is hoisted, not the initialization.

var Hoisting Example:
console.log(myVar); // Output: undefined (not ReferenceError!)
var myVar = "Hello";

// JavaScript engine interprets this as:
var myVar;           // Declaration hoisted to top
console.log(myVar);  // undefined (default value)
myVar = "Hello";     // Initialization remains here
let/const Hoisting Example:
console.log(myLet); // ReferenceError: Cannot access 'myLet' before initialization
let myLet = "World";

console.log(myConst); // ReferenceError: Cannot access 'myConst' before initialization
const myConst = 42;
Function Hoisting:
// Function declaration - fully hoisted
sayHello(); // Works! Output: "Hello!"
function sayHello() {
    console.log("Hello!");
}

// Function expression - only declaration hoisted
greet(); // TypeError: greet is not a function
const greet = function() {
    console.log("Hi!");
};

// Arrow function - same as function expression
hello(); // TypeError: hello is not a function
const hello = () => console.log("Hey!");
‚ö†Ô∏è Warning: Hoisting can cause bugs if you don't understand it. Always declare variables at the top of their scope!
Temporal Dead Zone (TDZ)

The Temporal Dead Zone (TDZ) is the period between entering a scope and the actual declaration of a variable with let or const.

Visualizing TDZ:
// Execution context created // Memory allocated for variables // TDZ begins here ------------------------+ // | // Code executes... | // Trying to access before declaration ‚Üí ERROR! // | // Declaration encountered ‚Üí TDZ ends ----+ // Variable initialized
TDZ Examples:
// Example 1: Basic TDZ
console.log(a); // ReferenceError
let a = 5;

// Example 2: Nested scope TDZ
function example() {
    console.log(b); // ReferenceError
    let b = 10;
    
    if (true) {
        console.log(c); // ReferenceError
        const c = 20;
    }
}

// Example 3: Function parameters in TDZ
function func(param = foo()) {
    // param is in TDZ until evaluated
    let foo = () => 42;
    // This would throw ReferenceError because foo is called before declaration
}

// Example 4: Circular dependencies in TDZ
let x = y + 1; // ReferenceError: Cannot access 'y' before initialization
let y = 10;
Why TDZ Exists:
  • Prevents accidental use of uninitialized variables
  • Makes code more predictable and less error-prone
  • Enforces better coding practices
  • Eliminates confusion between declaration and initialization
üîí Key Insight: TDZ is JavaScript's safety feature. It forces you to declare variables before using them, preventing many common bugs.
Block scope vs function scope

Scope determines where variables are accessible. JavaScript has two main scopes:

Function Scope (var)
function example() {
    if (true) {
        var functionScoped = "I'm function-scoped!";
    }
    
    console.log(functionScoped); // Works! Output: "I'm function-scoped!"
}

example();
console.log(functionScoped); // ReferenceError: functionScoped is not defined
Block Scope (let/const)
function example() {
    if (true) {
        let blockScoped = "I'm block-scoped!";
        const anotherBlock = 42;
    }
    
    console.log(blockScoped); // ReferenceError: blockScoped is not defined
    console.log(anotherBlock); // ReferenceError: anotherBlock is not defined
}

example();
Loop Scoping Differences:
// With var - all iterations share the same variable
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// Output: 3, 3, 3 (all refer to the same i)

// With let - each iteration has its own variable
for (let j = 0; j < 3; j++) {
    setTimeout(() => console.log(j), 100);
}
// Output: 0, 1, 2 (each j is separate)
Practical Implications:
// Bad: Using var in loops
var buttons = document.querySelectorAll('button');
for (var i = 0; i < buttons.length; i++) {
    buttons[i].addEventListener('click', function() {
        console.log('Button ' + i + ' clicked'); // Always shows last value!
    });
}

// Good: Using let in loops
for (let i = 0; i < buttons.length; i++) {
    buttons[i].addEventListener('click', function() {
        console.log('Button ' + i + ' clicked'); // Correct value!
    });
}

// Even better: Use event.target
buttons.forEach((button, index) => {
    button.addEventListener('click', function() {
        console.log('Button ' + index + ' clicked');
    });
});
‚úÖ Best Practice: Always use let/const for block scoping. Avoid var unless maintaining legacy code.

2.2 Primitive Data Types

Numbers and arithmetic operations

JavaScript uses a single numeric type: Number (IEEE 754 double-precision 64-bit floating point).

Number Literals:
// Integer literals
let wholeNumber = 42;
let negativeNumber = -17;

// Floating-point literals
let decimal = 3.14159;
let scientific = 6.022e23; // 6.022 √ó 10^23

// Hexadecimal (base 16)
let hex = 0xFF; // 255 in decimal

// Binary (base 2)
let binary = 0b1010; // 10 in decimal

// Octal (base 8)
let octal = 0o755; // 493 in decimal
Arithmetic Operations:
// Basic operations
let sum = 10 + 5;      // 15
let difference = 10 - 5; // 5
let product = 10 * 5;  // 50
let quotient = 10 / 5; // 2
let remainder = 10 % 3; // 1 (modulo)

// Exponentiation
let power = 2 ** 8;    // 256 (2 to the power of 8)

// Increment/decrement
let counter = 5;
counter++;             // 6
counter--;             // 5
++counter;             // 6
--counter;             // 5

// Order of operations (PEMDAS)
let result = 2 + 3 * 4; // 14 (multiplication first)
result = (2 + 3) * 4;   // 20 (parentheses override)
Special Number Values:
// Infinity
console.log(1 / 0);     // Infinity
console.log(-1 / 0);    // -Infinity
console.log(Infinity > 1000000); // true

// NaN (Not-a-Number)
console.log(0 / 0);     // NaN
console.log("hello" * 5); // NaN
console.log(Math.sqrt(-1)); // NaN

// Checking for NaN - tricky!
console.log(NaN === NaN); // false!
console.log(isNaN(NaN)); // true
console.log(Number.isNaN(NaN)); // true (better!)

// Safe integer limits
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(Number.MIN_SAFE_INTEGER); // -9007199254740991
‚ö†Ô∏è Warning: Floating-point precision issues! 0.1 + 0.2 !== 0.3 due to binary representation. Use libraries like decimal.js for financial calculations.
Strings and string manipulation

Strings are sequences of characters used for text representation.

String Creation:
// Single quotes
let greeting = 'Hello';

// Double quotes
let message = "Welcome to JavaScript";

// Template literals (backticks)
let name = "Alice";
let welcome = `Hello, ${name}! You're learning JavaScript.`;

// String concatenation
let fullName = "John" + " " + "Doe";
let combined = "Age: " + 25 + " years";
String Methods (Essential):
// Length property
let text = "JavaScript";
console.log(text.length); // 10

// Access characters
console.log(text[0]); // "J"
console.log(text.charAt(0)); // "J"

// Searching
console.log(text.indexOf("S")); // 4
console.log(text.includes("Script")); // true
console.log(text.startsWith("Java")); // true
console.log(text.endsWith("pt")); // false

// Case conversion
console.log(text.toLowerCase()); // "javascript"
console.log(text.toUpperCase()); // "JAVASCRIPT"

// Substrings
console.log(text.substring(0, 4)); // "Java" (start, end)
console.log(text.slice(4, 10)); // "Script" (start, end)
console.log(text.substr(4, 6)); // "Script" (start, length) - deprecated

// Splitting and joining
let sentence = "apple,banana,orange";
let fruits = sentence.split(","); // ["apple", "banana", "orange"]
let joined = fruits.join(" | "); // "apple | banana | orange"

// Trimming whitespace
let dirty = "   Hello World   ";
console.log(dirty.trim()); // "Hello World"
console.log(dirty.trimStart()); // "Hello World   "
console.log(dirty.trimEnd()); // "   Hello World"

// Replacing
let html = "<p>Hello</p>";
console.log(html.replace("<p>", "")); // "Hello</p>"
console.log(html.replaceAll("<", "[")); // "[p]Hello[/p]"
Template Literals Power:
// Multi-line strings
let poem = `Roses are red,
Violets are blue,
JavaScript is fun,
And so are you!`;

// Embedded expressions
let user = {name: "Bob", score: 95};
let report = `User: ${user.name}
Score: ${user.score}
Grade: ${user.score >= 90 ? 'A' : 'B'}`;

// Tagged templates (advanced)
function highlight(strings, ...values) {
    return strings.reduce((result, string, i) => {
        return result + string + (i < values.length ? `${values[i]}` : '');
    }, '');
}

let result = highlight`The answer is ${42} and not ${100}`;
console.log(result); // HTML with highlighted numbers
üí° Pro Tip: Always use template literals (backticks) for complex strings. They're more readable and powerful than concatenation!
Booleans and truthy/falsy values

Boolean represents logical entities: true or false.

Boolean Literals:
let isLoggedIn = true;
let isAdmin = false;

// Boolean coercion in conditions
if (isLoggedIn) {
    console.log("Welcome back!");
}

if (!isAdmin) {
    console.log("You don't have admin rights");
}
Falsy Values (8 total):
Falsy Value Description
false The boolean false
0 Zero (number)
-0 Negative zero
0n BigInt zero
"" Empty string
null Intentional absence of value
undefined Variable not assigned
NaN Not-a-Number
Truthy Values (everything else):
// These are ALL truthy!
if ("0") { console.log("'0' is truthy!"); }           // String with content
if ("false") { console.log("'false' is truthy!"); }   // String with content
if (" ") { console.log("' ' (space) is truthy!"); }   // Non-empty string
if ([]) { console.log("[] (empty array) is truthy!"); } // Empty array
if ({}) { console.log("{} (empty object) is truthy!"); } // Empty object
if (new Date()) { console.log("Date object is truthy!"); }
if (-1) { console.log("-1 is truthy!"); }              // Any non-zero number
if (42) { console.log("42 is truthy!"); }
Explicit Type Conversion:
// Convert to boolean explicitly
console.log(Boolean(0)); // false
console.log(Boolean("hello")); // true
console.log(Boolean([])); // true
console.log(Boolean({})); // true

// Double negation trick (convert to boolean)
console.log(!!"hello"); // true
console.log(!!0); // false

// Practical example: Validate form input
function validateEmail(email) {
    // Return true if email exists and is not empty
    return !!email && email.includes('@');
}

console.log(validateEmail("")); // false
console.log(validateEmail(null)); // false
console.log(validateEmail("test@example.com")); // true
üö® Critical Insight: Never compare with == when you want strict truthiness! Use === for exact equality checks.
null and undefined differences

Both represent "no value," but they mean different things!

undefined: The Default

Means "variable has been declared but not assigned a value."

// When variable is declared but not initialized
let message;
console.log(message); // undefined

// When accessing non-existent object property
let user = {};
console.log(user.name); // undefined

// When function parameter not provided
function greet(name) {
    console.log(name); // undefined if called without argument
}
greet(); // undefined

// When function doesn't return anything
function doNothing() {}
let result = doNothing();
console.log(result); // undefined
null: The Intentional Absence

Means "I intentionally want this to have no value."

// Explicitly setting to null
let userData = null; // We know this will be set later

// Clearing a reference
let cache = {data: "some data"};
cache = null; // We're done with this, release memory

// Database query result
let user = findUserById(999); // User doesn't exist
console.log(user); // null (intentionally, not because of error)
Key Differences:
Aspect undefined null
Type undefined object (historical bug)
Meaning Value not assigned Intentional absence
Assignment Automatic by JS Manual assignment
Common Usage Default state Reset/clear values
Equality undefined == null ‚Üí true undefined === null ‚Üí false
Testing for Both:
// Check if variable is null or undefined
if (value == null) {
    // This catches both null AND undefined
    console.log("Value is either null or undefined");
}

// More explicit check
if (value === null || value === undefined) {
    console.log("Value is explicitly null or undefined");
}

// Modern approach with optional chaining
const firstName = user?.profile?.firstName; // Returns undefined if any part is null/undefined

// Safe default values
const name = userName || "Guest"; // Fallback to "Guest" if userName is falsy
const realName = userName ?? "Guest"; // Only fallback if userName is null/undefined
üéØ Rule of Thumb: Use null when you want to say "I intend for this to be empty." Use undefined when you're waiting to assign a value.
Symbol type introduction

Symbols are unique, immutable identifiers introduced in ES6.

Creating Symbols:
// Basic symbol creation
let sym1 = Symbol();
let sym2 = Symbol();

console.log(sym1 === sym2); // false (even though both are symbols)

// Symbol with description (for debugging only)
let id = Symbol('user_id');
let status = Symbol('user_status');

console.log(id); // Symbol(user_id)
console.log(status); // Symbol(user_status)

// Description is not used for comparison
let sym3 = Symbol('user_id');
console.log(id === sym3); // false
Primary Use Case: Private Object Properties
// Before Symbols: Public properties
let user = {
    name: "Alice",
    age: 25,
    _password: "secret123" // Convention: private, but still accessible!
};

// With Symbols: Truly private properties
const PASSWORD = Symbol('password');
const EMAIL = Symbol('email');

let secureUser = {
    name: "Bob",
    age: 30,
    [PASSWORD]: "securePass456", // Hidden property
    [EMAIL]: "bob@example.com"
};

// Accessing symbol properties
console.log(secureUser[PASSWORD]); // "securePass456"
console.log(secureUser[EMAIL]); // "bob@example.com"

// Regular iteration won't show symbols
for (let key in secureUser) {
    console.log(key); // Only logs: "name", "age"
}

// Object.keys won't show symbols
console.log(Object.keys(secureUser)); // ["name", "age"]

// Get symbol keys specifically
console.log(Object.getOwnPropertySymbols(secureUser)); 
// [Symbol(password), Symbol(email)]

// Symbol as method key
const CALCULATE_TOTAL = Symbol('calculateTotal');
let cart = {
    items: ['apple', 'banana'],
    [CALCULATE_TOTAL]() {
        return this.items.length * 1.5;
    }
};

console.log(cart[CALCULATE_TOTAL]()); // 3
Global Symbols (Registry):
// Global symbol registry - shared across contexts
let globalSym1 = Symbol.for('app_config');
let globalSym2 = Symbol.for('app_config');

console.log(globalSym1 === globalSym2); // true!

// Retrieve symbol description
console.log(Symbol.keyFor(globalSym1)); // "app_config"

// Different registry keys create different symbols
let localSym = Symbol('app_config');
console.log(localSym === globalSym1); // false
üí° Why Symbols Matter: They prevent naming collisions in large applications and provide true privacy for object properties.
BigInt for large numbers

BigInt allows integers larger than Number.MAX_SAFE_INTEGER (2‚Åµ¬≥ - 1).

Creating BigInts:
// Method 1: Add 'n' suffix
let bigNum1 = 123456789012345678901234567890n;

// Method 2: Use BigInt() constructor
let bigNum2 = BigInt("987654321098765432109876543210");

// Method 3: From regular number (limited by safe integer range)
let bigNum3 = BigInt(12345); // 12345n

console.log(bigNum1); // 123456789012345678901234567890n
console.log(bigNum2); // 987654321098765432109876543210n
Big Numbers vs Regular Numbers:
// Regular numbers hit limit
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991
console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // Still 9007199254740992! (precision lost)

// BigInt handles large numbers perfectly
console.log(9007199254740991n + 1n); // 9007199254740992n
console.log(9007199254740991n + 2n); // 9007199254740993n

// Arithmetic operations
let big1 = 123456789012345678901234567890n;
let big2 = 987654321098765432109876543210n;

let sum = big1 + big2;
let product = big1 * big2;
let diff = big2 - big1;

console.log(sum); // 1111111110111111111011111111100n
console.log(product); // 12193263113702179522374637749673689995710252901985023427465800n
Important Restrictions:
// ‚ùå Cannot mix BigInt and Number in operations
let normal = 5;
let big = 10n;
// normal + big; // TypeError: Cannot mix BigInt and other types

// ‚úÖ Must convert explicitly
console.log(normal + Number(big)); // 15
console.log(BigInt(normal) + big); // 15n

// ‚ùå Cannot use Math methods on BigInt
// Math.max(10n, 20n); // TypeError

// ‚úÖ Use simple comparisons
console.log(10n > 5n); // true
console.log(10n === 10); // false (different types)
console.log(10n == 10); // true (loose equality)

// ‚ùå Cannot use division that results in fraction
// 10n / 3n; // TypeError: Division would result in non-integer

// ‚úÖ Use integer division
console.log(10n / 3n); // 3n (truncated)
console.log(10n % 3n); // 1n (remainder)
Practical Use Cases:
  • Cryptocurrency transactions (Bitcoin addresses)
  • High-precision scientific calculations
  • Generating unique IDs for distributed systems
  • Working with large datasets in web applications
  • Financial applications requiring exact precision
üöÄ Pro Tip: Use BigInt when you need mathematical precision beyond 2‚Åµ¬≥. For most web apps, regular numbers are sufficient!
