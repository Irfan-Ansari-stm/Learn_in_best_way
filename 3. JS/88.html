Template Syntax and Directives

Executive summary

Vue templates are declarative HTML enhanced with directives (v-bind, v-model, v-if, v-for, v-on). They provide readable, compact UI code and compile to render functions under the hood. Directives are prefixed with v- and can accept expressions.

Deep explanation

Templates are parsed by the Vue compiler and transformed into Virtual DOM render functions. Directives like v-for and v-if have different compilation behaviors — e.g., v-for creates a list of nodes, while v-if toggles their presence. Use :prop shorthand for v-bind and @event for v-on. Expressions in templates are sandboxed JavaScript limited to the component instance for predictability.

Syntax & pseudocode 
<!-- Bind attribute -->
<img :src="avatarUrl" alt="user avatar" />

<!-- Two-way binding -->
<input v-model="name" />

<!-- Conditional rendering -->
<div v-if="items.length > 0">Has items</div>

<!-- List rendering -->
<li v-for="(item, index) in items" :key="item.id">{{ index }} - {{ item.text }}</li>
Readable example (Template) 
<template>
  <div>
    <h3>Hello, {{ name }}</h3>
    <input v-model="name" placeholder="Enter name" />
    <ul>
      <li v-for="user in users" :key="user.id">{{ user.name }}</li>
    </ul>
  </div>
</template>
Real use cases & caveats

Use v-show when toggling visibility often (it toggles CSS display); use v-if when you need true conditional mounting. Avoid heavy logic in templates—move computations to computed properties or methods to keep templates declarative and fast.

Component Composition

Executive summary

Components are reusable UI pieces. Vue encourages composition via props, emits, slots, and provide/inject. Prefer small, focused components and compose them for complex UIs.

Deep explanation

Components in Vue can be defined as Options API or Composition API (with setup()). Composition API simplifies organizing logic by feature using reactive primitives. Slots allow parent-controlled template fragments; scoped slots (now v-slot) pass data from child to parent templates.

Syntax & pseudocode 
// Parent passes prop and listens for event
<UserCard :user="u" @select="onSelect" />

// Child component emits events
this.$emit('select', user.id);

// Slot example
<Modal>
  <template v-slot:header>Title</template>
</Modal>
Readable example (Composition API) 
<script setup>
import { ref } from 'vue';
const count = ref(0);
function inc(){ count.value++; }
</script>

<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="inc">Increment</button>
  </div>
</template>
Real use cases & best practices

Use props for data down, events (emits) for communication up. Use composables (reusable functions using Composition API) to share logic across components. Use keys on lists and avoid mutating props directly—emit changes to parent.

Reactive Data Binding

Executive summary

Vue's reactivity system tracks dependencies and automatically updates the DOM when reactive values change. In Vue 3, reactivity uses Proxies which handle deep reactivity and better performance.

Deep explanation

Reactive primitives include ref(), reactive(), computed(), and watch(). ref wraps primitive values with a .value accessor. reactive returns a reactive proxy for objects. computed creates memoized derived values based on reactive dependencies.

Syntax & pseudocode 
import { ref, reactive, computed, watch } from 'vue';

const count = ref(0);
const state = reactive({ items: [] });
const doubled = computed(() => count.value * 2);

watch(() => state.items.length, (newLen, oldLen) => {
  console.log('items changed', oldLen, '->', newLen);
});
Readable example (Reactivity) 
<script setup>
import { ref, computed } from 'vue';
const name = ref('Irfan');
const greeting = computed(() => `Hello, ${name.value}`);
</script>

<template>
  <div>{{ greeting }}</div>
</template>
Real use cases & caveats

Use watch for side effects and computed for derived state. Be careful with reactive objects and nested properties—Vue's proxies handle deep reactivity but avoid replacing large objects unnecessarily for performance.

Vue CLI and Ecosystem

Executive summary

Vue has a mature ecosystem: Vue CLI (deprecated in favor of Vite-based tooling), Vite, Vue Router, Pinia (modern store), Vue Test Utils, and Nuxt for SSR/SSG. Choose tools based on needs: Vite for dev speed, Nuxt for fullstack SSR.

Deep explanation

Vite offers lightning-fast cold starts and HMR. Pinia replaces Vuex as the recommended state store—it's simpler, modular, and uses the Composition API. Nuxt (v3) builds on Vue 3 for server-rendered apps and offers file-based routing, server routes, and more.

Commands & typical workflow 
// Create project with Vite + Vue
npm create vite@latest my-vue-app -- --template vue
cd my-vue-app
npm install
npm run dev

// Install Pinia
npm install pinia
Real world uses

Use Vite for single-page apps; use Nuxt for SEO-sensitive or server-rendered applications. Use Pinia for app-wide state with modular stores and better TypeScript support.

Single File Components (SFC)

Executive summary

SFCs (.vue files) contain <template>, <script>, and <style> blocks. They enable co-locating markup, logic, and styles and support features like scoped styles and custom blocks.

Deep explanation

SFCs are processed by the Vue compiler (via Vite or build tools) into render functions and CSS modules. The <script setup> syntax simplifies Composition API usage. Scoped styles limit CSS to the component; CSS variables and modules can be used for advanced styling.

SFC Example 
<!-- HelloWorld.vue -->
<template>
  <div class="hello">Hello {{ msg }}</div>
</template>

<script setup>
import { defineProps } from 'vue';
const props = defineProps({ msg: String });
</script>

<style scoped>
.hello { font-weight:600; }
</style>
Best practices
  • Keep SFCs focused and reasonably sized.
  • Use <script setup> for concise Composition API code.
  • Use scoped styles or CSS modules for component isolation.

Examples & Code Snippets

Template Example 
<template>
  <div>
    <input v-model="query" placeholder="Search" />
    <ul>
      <li v-for="item in filtered" :key="item.id">{{ item.text }}</li>
    </ul>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue';
const query = ref('');
const items = ref([{id:1,text:'apple'},{id:2,text:'banana'}]);
const filtered = computed(() => items.value.filter(i => i.text.includes(query.value)));
</script>
Props & Emits Example 
// Parent
<Counter :initial="3" @update="onUpdate" />

// Child (Counter.vue)
<script setup>
import { defineEmits, defineProps, ref } from 'vue';
const props = defineProps({ initial: Number });
const emit = defineEmits(['update']);
const count = ref(props.initial || 0);
function inc(){ count.value++; emit('update', count.value); }
</script>
Reactivity Example 
// Using reactive object
<script setup>
import { reactive } from 'vue';
const state = reactive({ todos: [] });
function add(text){ state.todos.push({ id: Date.now(), text }); }
</script>
