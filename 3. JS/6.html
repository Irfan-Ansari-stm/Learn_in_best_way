4.1.1 If, if/else Statements

The foundation of decision-making in code

Definition

If statements allow your program to execute different blocks of code based on whether a condition is true or false. They are the most basic form of conditional logic.

Detail

When the JavaScript engine encounters an if statement, it evaluates the condition inside parentheses. If the result is true, the code block within the curly braces executes. If false, it skips the block entirely.

An else clause provides an alternative path when the condition is false.

Syntax 
if (condition) {
    // code to run if condition is true
} else {
    // code to run if condition is false
}
Example 
// Check if user is eligible to vote
let age = 17;

if (age >= 18) {
    console.log("You can vote!");
} else {
    console.log("You must be 18+ to vote.");
}

// Output: "You must be 18+ to vote."
Uses
  • Validating user input
  • Controlling game logic (e.g., win/lose conditions)
  • Handling different user roles (admin vs guest)
  • Branching paths in algorithms

4.1.2 else if Chains

Multiple conditions in sequence

Definition

else if chains allow you to test multiple conditions in order, executing only the first block whose condition is true.

Detail

Think of an else-if chain as a series of doors. You check each door in order. As soon as you find one that opens (condition is true), you enter through it and ignore all remaining doors. This is called early exit.

Only one block in the entire chain will ever execute — even if multiple conditions are true.

Syntax 
if (condition1) {
    // runs if condition1 is true
} else if (condition2) {
    // runs if condition1 is false AND condition2 is true
} else if (condition3) {
    // runs if condition1 & 2 are false AND condition3 is true
} else {
    // runs if ALL above conditions are false
}
Example 
// Grade calculator based on score
let score = 85;

if (score >= 90) {
    console.log("Grade: A");
} else if (score >= 80) {
    console.log("Grade: B");
} else if (score >= 70) {
    console.log("Grade: C");
} else if (score >= 60) {
    console.log("Grade: D");
} else {
    console.log("Grade: F");
}

// Output: "Grade: B"
Uses
  • Classifying data (e.g., age groups, income brackets)
  • Priority-based routing in applications
  • Menu systems with ranked options
  • Game difficulty scaling

4.1.3 switch Statements and Fall-through

Efficient multi-way branching

Definition

A switch statement evaluates an expression and matches its value against multiple cases, executing the matching block.

Detail

Switches are more efficient than long if/else chains when comparing a single variable against many possible values. However, they have a unique behavior called fall-through: if you forget to use break, execution continues into the next case!

This fall-through can be intentional (to group multiple cases together) but often causes bugs if forgotten.

Syntax 
switch (expression) {
    case value1:
        // code
        break;
    case value2:
        // code
        break;
    default:
        // optional fallback
}
Example 
// Day of week to English name
let dayNumber = 3;
let dayName;

switch (dayNumber) {
    case 1:
        dayName = "Sunday";
        break;
    case 2:
        dayName = "Monday";
        break;
    case 3:
        dayName = "Tuesday";
        break;
    case 4:
        dayName = "Wednesday";
        break;
    case 5:
        dayName = "Thursday";
        break;
    case 6:
        dayName = "Friday";
        break;
    case 7:
        dayName = "Saturday";
        break;
    default:
        dayName = "Invalid day";
}

console.log(dayName); // Output: "Tuesday"

// ✅ FALL-THROUGH EXAMPLE: Grouping cases
let month = 12;
let season;

switch (month) {
    case 12:
    case 1:
    case 2:
        season = "Winter";
        break;
    case 3:
    case 4:
    case 5:
        season = "Spring";
        break;
    case 6:
    case 7:
    case 8:
        season = "Summer";
        break;
    case 9:
    case 10:
    case 11:
        season = "Autumn";
        break;
    default:
        season = "Unknown";
}

console.log(season); // Output: "Winter"
Uses
  • Menu selections (1-5 options)
  • HTTP status code handling
  • State machine transitions
  • Color codes or enum-like values

4.1.4 Conditional (Ternary) Operator

One-line conditional assignment

Definition

The ternary operator (condition ? exprIfTrue : exprIfFalse) is a compact way to write simple if/else statements as expressions.

Detail

It's called "ternary" because it takes three operands: a condition, a value if true, and a value if false. Unlike regular if/else, it returns a value — so you can assign it directly to a variable or use it in expressions.

Best used for simple conditions. Avoid nesting ternaries — it becomes unreadable!

Syntax 
condition ? valueIfTrue : valueIfFalse
Example 
// Simple example
let age = 20;
let status = (age >= 18) ? "Adult" : "Minor";
console.log(status); // Output: "Adult"

// Used in string templates
let isLoggedIn = false;
let greeting = isLoggedIn ? "Welcome back!" : "Please log in.";
console.log(greeting); // Output: "Please log in."

// Assigning different values
let temperature = 28;
let clothing = temperature > 25 ? "t-shirt" : "jacket";
console.log(`Wear a ${clothing}`); // Output: "Wear a t-shirt"

// ❌ Avoid this nested mess!
let result = (score >= 90) ? "A" : (score >= 80) ? "B" : (score >= 70) ? "C" : "F";
Uses
  • Assigning default values
  • Dynamic text/content generation
  • Conditional rendering in UI frameworks
  • Short-circuit assignments

4.1.5 Short-Circuit Evaluation Patterns

Smart boolean logic tricks

Definition

Short-circuit evaluation means JavaScript stops evaluating logical expressions as soon as the final result is known.

Detail

For && (AND): If the first operand is falsy, the second isn't evaluated — result is the first operand.

For || (OR): If the first operand is truthy, the second isn't evaluated — result is the first operand.

This isn't just optimization — it enables powerful patterns like default values and safe property access.

Syntax 
value1 && value2   // Returns value1 if falsy, else value2
value1 || value2   // Returns value1 if truthy, else value2
Example 
// ✅ Default parameter pattern
function greet(name) {
    name = name || "Guest";  // if name is null/undefined/empty, use "Guest"
    console.log("Hello, " + name);
}

greet();          // "Hello, Guest"
greet("Alice");   // "Hello, Alice"

// ✅ Safe object property access
let user = { profile: { name: "Bob" } };
let userName = user && user.profile && user.profile.name;
console.log(userName); // "Bob"

// With modern optional chaining (ES2020):
userName = user?.profile?.name; // safer!

// ✅ Conditional execution
let showNotification = true;
showNotification && alert("New message!"); // Only alerts if true

// ✅ Logical OR for choosing first truthy value
let theme = localStorage.getItem("theme") || "light" || "dark";
console.log(theme); // "light" (if stored value is null)
Uses
  • Providing default values for function parameters
  • Safe navigation in nested objects
  • Conditional execution without if statements
  • Choosing between multiple fallback options

4.2.1 for Loops and Variations

Count-controlled iteration

Definition

A for loop repeats a block of code a specific number of times, controlled by initialization, condition, and update expressions.

Detail

The classic for loop has three parts separated by semicolons:

  1. Initialization: Runs once before loop starts
  2. Condition: Checked before each iteration — if false, loop ends
  3. Update: Runs after each iteration

This structure makes it perfect for iterating arrays, counting, or repeating actions.

Syntax 
for (initialization; condition; update) {
    // loop body
}
Example 
// Basic counting
for (let i = 0; i < 5; i++) {
    console.log("Iteration:", i);
}
// Output: 0, 1, 2, 3, 4

// Array iteration
let fruits = ["apple", "banana", "orange"];
for (let i = 0; i < fruits.length; i++) {
    console.log(fruits[i]);
}
// Output: apple, banana, orange

// Reverse iteration
for (let i = fruits.length - 1; i >= 0; i--) {
    console.log(fruits[i]);
}
// Output: orange, banana, apple

// Complex increment
for (let n = 10; n <= 50; n += 5) {
    console.log(n); // 10, 15, 20, 25, 30, 35, 40, 45, 50
}
Uses
  • Iterating over arrays by index
  • Generating sequences (numbers, patterns)
  • Repeating animations or timers
  • Matrix/grid processing

4.2.2 while and do-while Loops

Condition-controlled iteration

Definition

while loops repeat while a condition remains true. do-while loops execute at least once, then check the condition.

Detail

Use while when you don’t know how many iterations you need — e.g., waiting for user input or processing until a state changes.

Use do-while when you want to guarantee at least one execution — like showing a menu before checking selection.

⚠️ Be careful! Infinite loops occur if the condition never becomes false.

Syntax 
// while
while (condition) {
    // body
}

// do-while
do {
    // body
} while (condition);
Example 
// while: count down
let counter = 5;
while (counter > 0) {
    console.log(counter);
    counter--;
}
// Output: 5, 4, 3, 2, 1

// do-while: menu system
let userInput = "";
do {
    userInput = prompt("Enter 'quit' to exit:");
    console.log("You entered:", userInput);
} while (userInput !== "quit");

// Game: roll dice until you get a 6
let roll;
do {
    roll = Math.floor(Math.random() * 6) + 1;
    console.log("Rolled:", roll);
} while (roll !== 6);
console.log("You rolled a 6! Game over!");
Uses
  • Reading files until end
  • Waiting for network responses
  • User input validation loops
  • Game loops and real-time simulations

4.2.3 for...in Loops for Objects

Iterating object keys

Definition

The for...in loop iterates over all enumerable properties (keys) of an object.

Detail

It loops through property names (strings), not values. It includes inherited properties from prototypes unless filtered with hasOwnProperty().

Use for...in only for objects. For arrays, prefer for...of or traditional for loops.

Syntax 
for (key in object) {
    // use object[key] to access value
}
Example 
// Object with properties
let person = {
    name: "Alice",
    age: 28,
    city: "Berlin",
    job: "Designer"
};

for (let key in person) {
    console.log(key + ": " + person[key]);
}
// Output:
// name: Alice
// age: 28
// city: Berlin
// job: Designer

// ✅ Safer version with hasOwnProperty
for (let key in person) {
    if (person.hasOwnProperty(key)) {
        console.log(key + ": " + person[key]);
    }
}

// ⚠️ Don't use for...in on arrays!
let arr = ['a', 'b', 'c'];
for (let index in arr) {
    console.log(typeof index); // "string" — not number!
    console.log(arr[index]);   // works, but confusing
}
Uses
  • Inspecting object configuration
  • Serializing objects to JSON
  • Dynamic property access
  • Debugging object structures

4.2.4 for...of Loops for Iterables

Direct value iteration

Definition

The for...of loop iterates over values of iterable objects (arrays, strings, maps, sets, etc.).

Detail

Unlike for...in (which gives keys), for...of gives you the actual values. It’s clean, readable, and ideal for collections where you care about contents, not positions.

Works on any object implementing the iterator protocol (Array, String, Map, Set, NodeList, etc.)

Syntax 
for (value of iterable) {
    // use value directly
}
Example 
// Array iteration — best practice!
let colors = ["red", "green", "blue"];

for (let color of colors) {
    console.log(color.toUpperCase());
}
// Output: RED, GREEN, BLUE

// String iteration
let word = "hello";
for (let char of word) {
    console.log(char); // h, e, l, l, o
}

// Set iteration
let uniqueNumbers = new Set([1, 2, 2, 3, 3, 4]);
for (let num of uniqueNumbers) {
    console.log(num); // 1, 2, 3, 4 (no duplicates)
}

// Map iteration
let map = new Map([["a", 1], ["b", 2]]);
for (let [key, value] of map) {
    console.log(key, "->", value);
}
// Output: a -> 1, b -> 2
Uses
  • Processing array elements without indexes
  • Looping through characters in strings
  • Iterating Map/Set collections
  • Modern, readable alternatives to for loops

4.2.5 break and continue Statements

Control flow modifiers

Definition

break exits the current loop entirely. continue skips the rest of the current iteration and jumps to the next one.

Detail

These are control statements that alter normal loop progression. Use them to optimize performance or handle edge cases gracefully.

Think of break as "stop everything now" and continue as "skip this one, move to next".

Syntax 
for (...) {
    if (condition) break;      // exit loop immediately
    if (condition) continue;   // skip rest of this iteration
    // other code
}
Example 
// break: find first even number
let numbers = [1, 3, 5, 8, 9, 12];
for (let num of numbers) {
    if (num % 2 === 0) {
        console.log("First even number found:", num);
        break; // Stops loop immediately
    }
}
// Output: "First even number found: 8"

// continue: skip odd numbers
for (let i = 1; i <= 10; i++) {
    if (i % 2 !== 0) continue; // skip odd numbers
    console.log(i); // Only prints: 2, 4, 6, 8, 10
}

// Nested loop break (breaks innermost)
outer: for (let x = 0; x < 3; x++) {
    for (let y = 0; y < 3; y++) {
        if (x === 1 && y === 1) break outer; // breaks both loops!
        console.log(x, y);
    }
}
// Output: (0,0), (0,1), (0,2), (1,0)
Uses
  • Early exit from search functions
  • Skipping invalid data points
  • Optimizing loops by avoiding unnecessary work
  • Breaking out of nested loops cleanly
