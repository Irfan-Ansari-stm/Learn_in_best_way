14.4 – WeakMap and WeakSet in JavaScript

Executive Summary: WeakMap and WeakSet store “weak” object references. If an object has no other references, the garbage collector removes it automatically even if it remains inside a WeakMap/WeakSet. This makes them perfect for private data, caching, memoization, DOM metadata, and leak-free structures.

14.4.1 WeakMap vs Map Differences

Core difference: keys must be objects 

const wm = new WeakMap();
const m  = new Map();

wm.set({}, "ok");    // ✔ allowed
wm.set("x", "nope"); // ❌ TypeError

WeakMap keys must be objects because GC needs to track them.

WeakMap keys are not enumerable 

const wm = new WeakMap();
const obj = {};

wm.set(obj, 123);

console.log(wm.keys);       // undefined
console.log([...wm]);       // ❌ cannot iterate

This prevents accidental leaks or exposure of private keys.

Use cases where WeakMap is superior
  • Private internal data in classes
  • Caching objects with automatic cleanup
  • DOM metadata without memory leaks
  • Event listener book-keeping

14.4.2 WeakSet vs Set Differences

WeakSet stores only objects & no enumeration 

const ws = new WeakSet();

ws.add({});   // ✔
ws.add(5);    // ❌ only objects allowed
No:
  • ws.size
  • for...of ws
  • [...ws]
Use case: object presence tracking 

const visited = new WeakSet();

function visit(node){
  if(visited.has(node)) return;
  visited.add(node);
  console.log("Visiting:", node.id);
}

14.4.3 Garbage Collection Implications

Weak references allow GC to delete unreachable objects 

let obj = {name: "Irfan"};

const wm = new WeakMap();
wm.set(obj, "secret");

obj = null; // remove the only strong reference

// Later GC removes internal entry automatically

Thus the entry disappears without manual cleanup.

No way to observe GC events
JavaScript does not allow:
  • checking if WeakMap key got GC’d
  • checking WeakMap size
  • iterating WeakMap contents

This is intentional for security and engine optimization.

14.4.4 Private Data Patterns Using WeakMap

Pattern: Private fields with WeakMap 

const _data = new WeakMap();

class User {
  constructor(name, password){
    _data.set(this, {password});
    this.name = name;
  }
  checkPassword(p){
    return _data.get(this).password === p;
  }
}

const u = new User("Irfan","123");
console.log(u.checkPassword("123")); // true
console.log(u.password);             // undefined

Cleaner than symbols because data fully disappears after GC.

Pattern: Hidden state machines 

const _state = new WeakMap();

class Machine {
  constructor(){
    _state.set(this, {count:0});
  }

  tick(){
    _state.get(this).count++;
  }

  read(){
    return _state.get(this).count;
  }
}
Pattern: DOM metadata without leaks 

const meta = new WeakMap();

function attachInfo(el, info){
  meta.set(el, info);
}

function getInfo(el){
  return meta.get(el);
}

// When el is removed from DOM, metadata vanishes automatically.

14.4.5 Memory Leak Prevention

Common leak scenario: Map holding DOM keys 

// ❌ leak
const map = new Map();
const el = document.getElementById("box");

map.set(el, "info");

document.body.removeChild(el);
// entry lives forever unless map.delete(el)
Fix with WeakMap 

// ✔ leak-free
const wm = new WeakMap();
const el = document.getElementById("box");

wm.set(el, "info");

document.body.removeChild(el);
// entry disappears automatically
Memoization with automatic cleanup 

const cache = new WeakMap();

function heavy(obj){
  if(cache.has(obj)) return cache.get(obj);

  const result = compute(obj);
  cache.set(obj, result);
  return result;
}
