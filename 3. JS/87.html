JSX and Component Creation

Executive summary

JSX is a syntactic extension that looks like HTML inside JavaScript. Components are the building blocks—either function components or class components (older). Modern React favors function components with hooks. Components accept inputs (props) and return UI description.

Deep explanation

JSX compiles to React.createElement() calls. It is not required but improves readability. Function components are plain functions that return JSX. They can be composed, parameterized with props, and can maintain internal state via hooks (e.g., useState). Understanding the virtual DOM diffing algorithm clarifies why React batches updates and why keeping components pure simplifies reasoning.

Syntax & pseudocode 
<!-- JSX example (displayed as code, with entities) -->
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>
}

// Compiles to
// React.createElement('h1', null, 'Hello, ' + props.name + '!')
Readable example (JS) 
<!-- Example file: App.js -->
import React from 'react';

function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>
}

export default function App() {
  return (
    <div>
      <Greeting name="Irfan" />
    </div>
  );
}
Real use cases & caveats

Use JSX for component templates; avoid complex inline logic in JSX, prefer helper functions. Keep components small (single responsibility). When performance is critical, consider memoization (React.memo) and avoid unnecessary re-renders by proper keying and immutable updates.

Props and State Management

Executive summary

Props are immutable inputs from parent to child. State is local mutable data inside a component, managed via hooks (e.g., useState, useReducer). For app-wide state, use context, Redux, MobX, or React Query for server state.

Deep explanation

Think of props as function arguments and state as internal variables. Changing state triggers a re-render for that component and its subtree. Use reducers for complex state transitions to centralize logic and keep updates predictable. For many async patterns and caching, libraries like React Query (TanStack Query) simplify server state management.

Syntax & pseudocode 
// useState
const [count, setCount] = useState(0);

// useReducer example (pseudocode)
function reducer(state, action) {
  switch(action.type) {
    case 'increment': return {count: state.count + 1};
    default: return state;
  }
}
const [state, dispatch] = useReducer(reducer, {count:0});
Readable example (State & Props) 
<!-- App.js -->
import React, { useState } from 'react';

function Counter({initial=0}){
  const [count, setCount] = useState(initial);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
    </div>
  );
}

export default function App(){
  return <Counter initial={5} />;
}
Real use cases & best practices

Lift state up when multiple components need the same data. Use useMemo and useCallback to avoid recreating expensive values or functions. For large apps, split responsibilities: local, shared (context), and server state.

Event Handling in React

Executive summary

React uses a synthetic event system that wraps browser events, normalizing behavior across browsers. Event handlers are passed as camelCase props (onClick, onChange). Use event pooling awareness (older React) and prefer functional updates when state depends on previous state.

Deep explanation

Handlers receive a SyntheticEvent which mirrors native event API. You can call event.preventDefault() and event.stopPropagation(). When passing handlers, bind arguments carefully — use closures or arrow functions to avoid unintended re-renders.

Syntax & example 
// Button with handler
function MyButton(){
  function handleClick(e){
    e.preventDefault();
    console.log('clicked');
  }
  return <button onClick={handleClick}>Click</button>
}
Real use cases

Form handling: manage controlled components with value + onChange, or use libraries like Formik or React Hook Form for complex forms. Debounce heavy handlers when necessary.

Lifecycle Methods and Hooks

Executive summary

Class components used lifecycle methods (componentDidMount, componentDidUpdate, componentWillUnmount). In function components, useEffect models mounting, updating, and cleanup. Hooks provide powerful primitives: useState, useEffect, useMemo, useCallback, useRef, and custom hooks.

Deep explanation

useEffect runs after render. A return function in useEffect is a cleanup (like componentWillUnmount). Use dependency arrays to control execution; mistake-free patterns include listing dependencies and extracting logic into hooks to avoid stale closures.

Syntax & examples 
// useEffect
useEffect(() => {
  const id = setInterval(() => console.log('tick'), 1000);
  return () => clearInterval(id);
}, []); // [] means run once on mount

// Custom hook pattern
function useCounter(initial=0){
  const [count,setCount] = useState(initial);
  const inc = () => setCount(c => c+1);
  return {count, inc};
}
Real use cases & performance

Use useEffect for data fetching, subscriptions, and DOM interactions. For performance, avoid expensive work in render — memoize derived data and minimize DOM operations. Custom hooks let you share logic succinctly.

React Ecosystem Overview

Executive summary

React sits in a rich ecosystem: routing (React Router), state management (Redux, Zustand, Recoil), server data (React Query), styling (styled-components, Tailwind CSS), bundlers (Vite, Webpack), and frameworks (Next.js). Choose tools based on app size and team familiarity.

Deep explanation

For full-stack needs, frameworks like Next.js provide SSR/SSG and API routes. For rapid frontend apps, Vite offers fast builds. Testing tools: Jest + React Testing Library. Accessibility: follow ARIA roles and semantic HTML.

Best practices
  • Keep components small and focused.
  • Prefer composition over inheritance.
  • Use feature folders for large apps.
  • Use TypeScript for type safety where possible.

Hooks Example 
// useEffect + fetch pattern
import React, {useState, useEffect} from 'react';

function Users(){
  const [users, setUsers] = useState([]);
  useEffect(() => {
    let mounted = true;
    fetch('/api/users')
      .then(r => r.json())
      .then(data => { if(mounted) setUsers(data); });
    return () => { mounted = false; };
  }, []);
  return <div>{users.length} users</div>
}
