Function Parameters vs Arguments

Definition: In JavaScript, parameters are the named variables listed in a function's definition, while arguments are the actual values passed when the function is called.

Example:
function greet(name, age) {           // name, age ‚Üí PARAMETERS
    console.log(`Hello ${name}, you are ${age} years old`);
}

greet("Alice", 25);                     // "Alice", 25 ‚Üí ARGUMENTS

Key Detail: Parameters act as placeholders; arguments provide concrete data. Think of parameters as "slots" and arguments as "fillers".

Use Case: This distinction matters when debugging or reading code. If you see function calculateTax(rate, income), you know what data it expects. When you call calculateTax(0.15, 50000), you're providing the actual values.

Syntax:

function functionName(param1, param2, ..., paramN) {
    // function body
}

functionName(arg1, arg2, ..., argN);

Deep Insight: JavaScript is flexible ‚Äî you can pass more or fewer arguments than parameters. Extra arguments are ignored; missing ones become undefined.

Default Parameters (ES6+)

Definition: Default parameters allow functions to have default values for parameters if no argument is passed or if undefined is passed.

Example:
// Before ES6: Manual check
function multiply(a, b) {
    if (typeof b === 'undefined') {
        b = 1;
    }
    return a * b;
}

// ES6+ with default parameters
function multiply(a, b = 1) {
    return a * b;
}

console.log(multiply(5));       // 5 (b defaults to 1)
console.log(multiply(5, 4));    // 20 (b is 4)

Detail: Default values are evaluated at call time. You can use expressions, other parameters, or even function calls!

Advanced Example:
function createUser(name, role = "user", createdAt = new Date()) {
    return { name, role, createdAt };
}

console.log(createUser("Bob")); 
// Output: { name: "Bob", role: "user", createdAt: Sat Jun 01 2024 ... }

// Using another parameter as default
function createGreeting(name, greeting = `Hi ${name}`) {
    return greeting;
}

console.log(createGreeting("Charlie")); // "Hi Charlie"
Use Case: Default parameters eliminate boilerplate code, improve readability, and make APIs more intuitive. Ideal for optional configuration options like fetch(url, { method: "GET", headers: {} }).

Important Note: Only undefined triggers defaults. Passing null does not.

function logValue(x = "default") {
    console.log(x);
}

logValue(null);     // logs: null (not "default")
logValue();         // logs: "default"
Rest Parameters (...args)

Definition: Rest parameters allow a function to accept an indefinite number of arguments as an array. They must be the last parameter in the function signature.

Example:
// Using rest parameters
function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}

console.log(sum(1, 2, 3));        // 6
console.log(sum(10, 20, 30, 40)); // 100

// Can mix with regular parameters
function describePerson(name, age, ...hobbies) {
    return `${name}, ${age}, loves: ${hobbies.join(", ")}`;
}

console.log(describePerson("Sam", 28, "reading", "swimming", "coding"));
// Output: "Sam, 28, loves: reading, swimming, coding"

Detail: The rest parameter collects remaining arguments into a real Array ‚Äî unlike the legacy arguments object which is array-like.

...restParam ‚Üí converts remaining arguments into an array

Syntax Rules:

  • Only one rest parameter allowed per function
  • Must be the last parameter
  • Cannot be used in arrow functions with curly braces without parentheses
Incorrect Usage:
// ‚ùå Error: Rest parameter must be last
function badExample(a, ...b, c) { }

// ‚ùå Error: Multiple rest params
function badExample2(...a, ...b) { }
Use Case: Perfect for utility functions that handle variable inputs: Math.max(...array), logging multiple values, composing functions, or building API wrappers.

Deep Insight: Rest parameters are true arrays ‚Äî so you get access to .map(), .filter(), etc., unlike arguments.

Arguments Object (Legacy)

Definition: The arguments object is an array-like object available inside all non-arrow functions that contains all arguments passed to the function.

Example:
function logArgs() {
    console.log(arguments.length);   // Number of arguments passed
    console.log(arguments[0]);       // First argument
    console.log(arguments[1]);       // Second argument
    
    // Convert to real array (ES5 way)
    const argsArray = Array.prototype.slice.call(arguments);
    console.log(argsArray);
}

logArgs("apple", 42, true);
// Output:
// 3
// "apple"
// 42
// ["apple", 42, true]

Detail: Although it looks like an array, arguments lacks array methods (map, filter, etc.). It has properties like length and numeric indices.

Use Case: Useful in older codebases or when you need to access all arguments dynamically. Still used in polyfills or libraries supporting legacy browsers.

Limitations:

  • Not available in arrow functions
  • No array methods
  • Performance overhead
  • Can't be destructured easily
Arrow Function Limitation:
const add = () => {
    console.log(arguments); // ‚ùå ReferenceError: arguments is not defined
};

add(1, 2);

Modern Alternative: Use rest parameters instead ‚Äî they‚Äôre cleaner and more powerful.

// ‚úÖ Modern way
const add = (...nums) => {
    console.log(nums); // ‚úÖ Real array!
    return nums.reduce((a,b) => a+b, 0);
};
Parameter Destructuring

Definition: Parameter destructuring allows you to extract values from objects or arrays passed as arguments directly in the function signature.

Object Destructuring:
function displayUser({ name, age, city = "Unknown" }) {
    console.log(`${name}, ${age} from ${city}`);
}

displayUser({ name: "Taylor", age: 30 }); 
// Output: "Taylor, 30 from Unknown"

// With aliasing
function processConfig({ timeout: ms, retries: attempts = 3 }) {
    console.log(`Timeout: ${ms}ms, Attempts: ${attempts}`);
}

processConfig({ timeout: 5000 });
// Output: "Timeout: 5000ms, Attempts: 3"
Array Destructuring:
function swap([a, b]) {
    return [b, a];
}

console.log(swap([1, 2])); // [2, 1]

// With defaults
function pointInfo([x = 0, y = 0] = []) {
    return { x, y };
}

console.log(pointInfo());           // { x: 0, y: 0 }
console.log(pointInfo([5]));        // { x: 5, y: 0 }
console.log(pointInfo([10, 20]));   // { x: 10, y: 20 }

Detail: Destructuring makes function signatures self-documenting and reduces boilerplate. It‚Äôs especially powerful when working with configuration objects.

function fn({ key1, key2 } = {}) { ... } ‚Üí Safe default for object parameter
Use Case: Essential for modern JS frameworks (React props, Node.js config objects). For example: useEffect(() => {}, [dep]) or axios.get(url, { headers, params }).

Pro Tip: Combine with defaults for robust APIs:

function configureServer({
    port = 3000,
    host = "localhost",
    ssl = false,
    middleware = []
} = {}) {
    // Always safe ‚Äî even if no object passed!
}
Summary & Best Practices
  • Parameters = function declaration variables; Arguments = values passed at call time.
  • Default Parameters reduce conditionals and improve clarity. Use them generously.
  • Rest Parameters (...) are preferred over arguments ‚Äî they‚Äôre real arrays and work with arrow functions.
  • Arguments Object is legacy ‚Äî avoid unless maintaining old code.
  • Destructuring makes complex data handling elegant. Use it for configs, APIs, and multi-value returns.
üöÄ Pro Tips:
  1. Always prefer rest parameters over arguments for modern code.
  2. Use destructuring with defaults to avoid null/undefined bugs.
  3. Don‚Äôt overcomplicate ‚Äî if a function needs >3 parameters, consider using an object.
  4. Combine rest + destructuring: function log(...[{name}, ...items]) ‚Äî advanced but powerful!
Real-World Example:
// A modern, clean API design
function createNotification({
    title,
    message,
    type = "info",
    duration = 3000,
    onDismiss = () => {}
} = {}) {
    // Render notification logic
    console.log(`[${type}] ${title}: ${message}`);
    setTimeout(onDismiss, duration);
}

// Call cleanly
createNotification({
    title: "Login Success",
    message: "Welcome back!",
    type: "success",
    onDismiss: () => console.log("Dismissed!")
});
