CSS Custom Properties (Variables)

Unlock dynamic, maintainable, and theme-ready styles with native CSS variables ‚Äî the modern way to manage design systems.

12.1 Custom Property Basics

Defining and using custom properties

Definition: CSS Custom Properties (also called CSS Variables) are entities defined by authors that contain specific values to be reused throughout a document. They are set using the --* syntax.

Syntax:

:root {
  --primary-color: #00f5ff;
  --font-size-base: 16px;
  --spacing-unit: 8px;
}

.button {
  background-color: var(--primary-color);
  font-size: var(--font-size-base);
  padding: calc(var(--spacing-unit) * 2);
}

Key Points:

  • Variables must be declared within a selector context ‚Äî typically :root for global scope.
  • Use var(--name) to reference them anywhere in your CSS.
  • Variable names are case-sensitive: --Color ‚â† --color.

Example: Define a color palette globally and reuse it across components.

:root {
  --brand-accent: #ffb74d;
  --text-dark: #1a1a1a;
  --text-light: #f5f5f5;
}

header { background: var(--brand-accent); color: var(--text-light); }
footer { background: var(--text-dark); color: var(--text-light); }
Inheritance and scope

Definition: CSS variables inherit down the DOM tree like regular CSS properties. Their value is determined by the nearest ancestor declaration.

Detail: Unlike preprocessors (Sass/Less), CSS variables are runtime variables ‚Äî they follow the cascade and inheritance model of CSS itself.

Example:

:root {
  --color: blue;
}

.container {
  --color: red;
}

.box {
  color: var(--color); /* ‚Üí red */
}

.inner-box {
  --color: green;
}

.inner-box .box {
  color: var(--color); /* ‚Üí green */
}

Visualization:

Imagine variables as water flowing through pipes ‚Äî each parent can change the flow, and children receive the latest version.

Best Practice: Always define global variables on :root. Override locally only when needed for component-specific customization.

Fallback values

Definition: The var() function accepts an optional second argument ‚Äî a fallback value ‚Äî used if the variable is undefined or invalid.

Syntax:

element {
  color: var(--undefined-variable, #000);
  padding: var(--padding, 10px);
}

Example:

.card {
  border-color: var(--card-border, #ccc);
  border-width: var(--card-border-width, 1px);
  border-style: solid;
}

If --card-border is not defined anywhere, the browser uses #ccc automatically.

Advanced Fallback Chain:

background-color: var(--theme-bg, var(--default-bg, #fff));

This checks --theme-bg first, then falls back to --default-bg, then finally #fff.

Why it matters: Fallbacks make your CSS resilient during development, debugging, or when using third-party themes.

vs preprocessor variables

Preprocessor Variables (Sass/Less):

  • Compile-time substitution ‚Äî replaced before CSS is sent to browser.
  • Cannot change at runtime.
  • No inheritance or cascade ‚Äî static replacement.
  • Require build step (Node.js, Webpack, etc.)

CSS Custom Properties:

  • Runtime variables ‚Äî modifiable via JavaScript.
  • Follow CSS cascade and inheritance.
  • Dynamic theming without rebuilds.
  • No build tools required ‚Äî pure CSS.

Example Comparison:

// Sass (compile-time)
$primary: #00f5ff;
.btn { background: $primary; } 
// ‚Üí compiles to: .btn { background: #00f5ff; }

/* CSS Variable (runtime) */
:root { --primary: #00f5ff; }
.btn { background: var(--primary); }
// ‚Üí remains dynamic ‚Äî can be changed in JS!

Takeaway: Use CSS variables for dynamic UIs (themes, user preferences). Use preprocessors for complex logic (mixins, functions, loops).

12.2 Advanced Custom Property Techniques

CSS custom properties with calc()

Definition: The calc() function allows mathematical expressions with CSS units and variables.

Syntax:

element {
  width: calc(var(--base-width) + 20px);
  margin: calc(var(--spacing) / 2);
  font-size: calc(1rem + 0.5vw);
}

Important Rules:

  • Always use spaces around operators: calc(10px + 5px) ‚úÖ, calc(10px+5px) ‚ùå
  • You can mix units: pixels, percentages, ems, rems, vw/vh
  • Variables inside calc() resolve at runtime

Real-World Example: Fluid Typography Scale

:root {
  --base-font: 16px;
  --scale-factor: 1.2;
}

h1 { font-size: calc(var(--base-font) * var(--scale-factor) * 2); }
h2 { font-size: calc(var(--base-font) * var(--scale-factor)); }
h3 { font-size: var(--base-font); }

Responsive Enhancement:

@media (min-width: 768px) {
  :root {
    --base-font: 18px;
  }
}

Now all headings scale responsively without rewriting rules!

Dynamic theming and dark mode

Definition: CSS variables enable real-time theme switching ‚Äî perfect for dark/light mode toggles.

Implementation Strategy:

  1. Define two sets of variables: one for light, one for dark.
  2. Toggle a class on <html> or <body>.
  3. Override variables based on that class.

Code Example:

:root {
  --bg-color: #ffffff;
  --text-color: #333333;
  --card-bg: #f8f9fa;
  --border-color: #e0e0e0;
}

[data-theme="dark"] {
  --bg-color: #0a192f;
  --text-color: #e6f1ff;
  --card-bg: #112240;
  --border-color: #2d4a70;
}

body {
  background-color: var(--bg-color);
  color: var(--text-color);
}

.card {
  background: var(--card-bg);
  border: 1px solid var(--border-color);
}

JavaScript Toggle:

const toggleTheme = () => {
  const currentTheme = document.documentElement.getAttribute('data-theme') || 'light';
  const newTheme = currentTheme === 'light' ? 'dark' : 'light';
  document.documentElement.setAttribute('data-theme', newTheme);
  localStorage.setItem('theme', newTheme);
};

// On page load
if (localStorage.getItem('theme') === 'dark') {
  document.documentElement.setAttribute('data-theme', 'dark');
}

Benefits:

  • No page reload needed
  • Smooth transitions possible with transition: all 0.3s ease
  • Accessible via OS preference using prefers-color-scheme
Component-level custom properties

Definition: Declare custom properties directly on a component's root element to allow consumers to override styling without modifying the component‚Äôs CSS file.

Why It Matters: Enables true component reusability ‚Äî think ‚Äúdesign tokens‚Äù per instance.

Example: Reusable Button Component

.btn {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  font-weight: bold;
  cursor: pointer;
  background-color: var(--btn-bg, #00f5ff);
  color: var(--btn-text, #000);
  transition: all 0.2s ease;
}

.btn:hover {
  background-color: var(--btn-bg-hover, #00c7d6);
}

HTML Usage:

<button class="btn" style="--btn-bg: #ffb74d; --btn-text: #000; --btn-bg-hover: #e6a33d">
  Warning Button
</button>

<button class="btn" style="--btn-bg: #00f5ff; --btn-text: #000; --btn-bg-hover: #00c7d6">
  Info Button
</button>

Advantage: No need for 10 different CSS classes (.btn-warning, .btn-info) ‚Äî one component, infinite variations.

Pro Tip: Combine with CSS classes for defaults:

.btn--danger {
  --btn-bg: #ff4757;
  --btn-text: #fff;
  --btn-bg-hover: #ff3344;
}
CSS custom properties and JavaScript

Definition: JavaScript can read and write CSS custom properties using the getPropertyValue() and setProperty() methods on the style property.

Reading Values:

const root = document.documentElement;
const primaryColor = getComputedStyle(root).getPropertyValue('--primary-color');
console.log(primaryColor); // ‚Üí "rgb(0, 245, 255)"

Writing Values Dynamically:

document.documentElement.style.setProperty('--primary-color', '#ffb74d');

Real-Time Slider Example:

<input type="range" min="10" max="30" value="16" id="fontSizeSlider">
<p id="demoText">Adjust font size with slider!</p>
const slider = document.getElementById('fontSizeSlider');
const demoText = document.getElementById('demoText');

slider.addEventListener('input', () => {
  document.documentElement.style.setProperty('--base-font', slider.value + 'px');
  demoText.textContent = `Current font size: ${slider.value}px`;
});

Use Cases:

  • Live theme editors
  • User preference panels
  • Animation triggers
  • Dynamic grid layouts based on viewport

Performance Note: Writing to CSS variables is fast ‚Äî no layout recalculations unless the property affects layout (like width or font-size).

12.3 Design Systems with Custom Properties

Color systems and semantic naming

Definition: Semantic naming means using meaningful names instead of literal colors (e.g., --primary-color instead of --blue-500).

Bad Practice:

:root {
  --blue-500: #00f5ff;
  --red-500: #ff4757;
}

.header { background: var(--blue-500); }
.alert { color: var(--red-500); }

Good Practice ‚Äî Semantic Naming:

:root {
  --color-primary: #00f5ff;
  --color-danger: #ff4757;
  --color-success: #2ed573;
}

.header { background: var(--color-primary); }
.alert { color: var(--color-danger); }
.success-badge { background: var(--color-success); }

Why? If you later decide to change the primary color from cyan to amber, you only update one variable ‚Äî not every occurrence of #00f5ff.

Scale Example:

:root {
  --color-neutral-50: #f8f9fa;
  --color-neutral-100: #e9ecef;
  --color-neutral-200: #dee2e6;
  --color-neutral-300: #ced4da;
  --color-neutral-400: #adb5bd;
  --color-neutral-500: #6c757d;
  --color-neutral-600: #495057;
  --color-neutral-700: #343a40;
  --color-neutral-800: #212529;
  --color-neutral-900: #1a1a1a;
}

Now you can create consistent tones across UI elements while keeping meaning clear.

Spacing scales and typography scales

Definition: A spacing scale defines a system of consistent distances (margins, paddings). A typography scale defines consistent font sizes.

Spacing Scale Example:

:root {
  --space-0: 0px;
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 16px;
  --space-4: 24px;
  --space-5: 32px;
  --space-6: 48px;
}

.container {
  padding: var(--space-4);
}

.card {
  margin-bottom: var(--space-3);
  padding: var(--space-2);
}

.btn {
  padding: var(--space-2) var(--space-4);
}

Typography Scale Example (Using Ratio):

:root {
  --font-size-xs: 0.75rem;
  --font-size-sm: 0.875rem;
  --font-size-md: 1rem;
  --font-size-lg: 1.125rem;
  --font-size-xl: 1.25rem;
  --font-size-2xl: 1.5rem;
  --font-size-3xl: 1.875rem;
  --font-size-4xl: 2.25rem;
}

Combine with CSS Units for Responsiveness:

@media (min-width: 768px) {
  :root {
    --font-size-md: 1.1rem;
    --font-size-lg: 1.25rem;
  }
}

Benefit: Your entire UI becomes predictable, scalable, and easy to audit. Designers and developers speak the same language.

Component tokens and global tokens

Global Tokens: Shared across the entire application. Define foundational values.

:root {
  --color-primary: #00f5ff;
  --font-main: 'Roboto', sans-serif;
  --radius-small: 4px;
  --shadow-default: 0 4px 8px rgba(0,0,0,0.1);
  --transition-fast: 0.2s ease;
}

Component Tokens: Specific to a component, often overriding or extending global tokens.

.modal {
  --modal-bg: rgba(10, 25, 47, 0.95);
  --modal-shadow: 0 20px 40px rgba(0, 245, 255, 0.2);
  --modal-radius: 12px;
  --modal-padding: 2rem;
}

Usage Pattern:

.modal {
  background: var(--modal-bg, var(--bg-color));
  border-radius: var(--modal-radius, var(--radius-small));
  box-shadow: var(--modal-shadow, var(--shadow-default));
  padding: var(--modal-padding, var(--space-4));
}

Benefits:

  • Components are self-contained but still theme-aware.
  • Designers can tweak component appearance without touching core tokens.
  • Easy to swap entire themes by changing just global tokens.

Tooling Tip: Document tokens in a JSON or Markdown file:

{
  "global": {
    "color-primary": "#00f5ff",
    "font-main": "'Roboto', sans-serif"
  },
  "component-modal": {
    "bg": "rgba(10,25,47,0.95)",
    "shadow": "0 20px 40px rgba(0,245,255,0.2)"
  }
}

Strategy 4: Version Control & Naming Conventions

Examples:

--spacing-card-vertical

Final Tip: Treat CSS variables like API endpoints ‚Äî once published, avoid breaking changes. Use deprecation notices and gradual migration.
