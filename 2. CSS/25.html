23.1 Print-Specific CSS

Printing web content requires a completely different approach than screen display. What looks great on screen can be unusable when printed.

Definition: @media print queries

The @media print rule allows you to define styles specifically for printed output — separate from your screen styles.

This is where you:

  • Remove navigation elements
  • Change colors to grayscale
  • Adjust layouts for paper dimensions
  • Control page breaks

Key insight: Print media has no hover states, no scrolling, no JavaScript — it's a static, fixed medium.

Syntax: Basic Print Styles 
// Screen styles (default)
body {
  background-color: #0a192f;
  color: #e6f1ff;
  font-family: 'Segoe UI', sans-serif;
}

.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 2rem;
}

// Print styles
@media print {
  body {
    background-color: white !important;
    color: black !important;
    font-family: 'Times New Roman', serif; /* Better for print */
  }

  .container {
    max-width: none;
    margin: 0;
    padding: 1in;
  }

  /* Hide elements not needed in print */
  nav,
  header,
  footer,
  .no-print {
    display: none !important;
  }

  /* Show URLs for links */
  a[href]:after {
    content: " (" attr(href) ")";
    font-size: 0.9em;
    color: #666;
  }

  /* Ensure images are visible */
  img {
    max-width: 100% !important;
    height: auto !important;
    border: 1px solid #ddd;
  }

  /* Remove shadows and complex effects */
  * {
    box-shadow: none !important;
    text-shadow: none !important;
    filter: none !important;
  }

  /* Force printing of backgrounds */
  * {
    -webkit-print-color-adjust: exact !important;
    color-adjust: exact !important;
  }
}

Important notes:

  • Always use !important in print styles — they override screen styles
  • Use absolute units like inches (in), points (pt), or millimeters (mm)
  • Test with actual printers — browser print previews aren't perfect

Detail: Page Boxes and Margins

CSS defines a "page box" model for printed documents:

  • Page box: The entire printable area
  • Margin box: Outermost boundary
  • Border box: Where borders would go (usually none in print)
  • Padding box: Inside the border
  • Content box: Where your text and elements live
// Set page size and margins
@page {
  size: letter portrait; /* or A4, legal, etc. */
  margin: 1in; /* all sides */
}

/* Different margins for first page */
@page :first {
  margin-top: 1.5in;
}

/* Different margins for left/right pages */
@page :left {
  margin-left: 1.5in;
}

@page :right {
  margin-right: 1.5in;
}

/* Specific page sizes */
@page {
  size: A4;
  margin: 2cm;
}

@page landscape {
  size: letter landscape;
  margin: 1in;
}

.landscape {
  page: landscape;
}

Common page sizes:

  • Letter: 8.5 × 11 inches (North America)
  • A4: 210 × 297 mm (International standard)
  • Ledger: 11 × 17 inches
  • Legal: 8.5 × 14 inches

Pro tip: Always specify page size explicitly — browsers default to Letter, which may not match your users' printers!

Syntax: Avoiding Page Breaks

Page breaks can ruin carefully designed layouts. Control them precisely:

// Prevent break inside element
.element {
  page-break-inside: avoid;
}

// Force break before element
.element {
  page-break-before: always;
}

// Force break after element
.element {
  page-break-after: always;
}

// Allow breaks (default)
.element {
  page-break-before: auto;
  page-break-after: auto;
  page-break-inside: auto;
}

// Common patterns:
h1 {
  page-break-before: always;
  page-break-after: avoid;
}

h2 {
  page-break-before: avoid;
  page-break-after: avoid;
}

img {
  page-break-inside: avoid;
  page-break-before: avoid;
  page-break-after: avoid;
}

table {
  page-break-inside: avoid;
}

blockquote {
  page-break-inside: avoid;
}

/* For modern browsers, use break properties instead */
.element {
  break-inside: avoid;
  break-before: always;
  break-after: always;
}

/* Modern syntax (preferred) */
.element {
  break-inside: avoid;
  break-before: page;
  break-after: page;
}

Pro tip: Use break-inside: avoid on containers that hold related content like:

  • Images with captions
  • Tables
  • Code blocks
  • Quote blocks
  • Form sections
Detail: Print Color Considerations

Printers work differently than screens — here's how to handle color properly:

Color Modes

  • RGB (Screen): Red, Green, Blue — additive light
  • CMYK (Print): Cyan, Magenta, Yellow, Black — subtractive ink
  • Grayscale: Shades of gray

Best Practices

  • Use grayscale for text: Black (#000) on white (#fff) provides maximum contrast
  • Test color contrast: Use tools like WebAIM Contrast Checker
  • Minimize colors: Each additional color increases cost
  • Use spot colors sparingly: They require special ink
  • Consider toner savings: Light gray text uses less ink
// Good print colors
body {
  color: #000; /* Pure black */
  background-color: #fff; /* Pure white */
}

h1, h2, h3 {
  color: #000;
}

a {
  color: #000;
}

// Bad print colors
body {
  color: #333; /* Too light */
  background-color: #eee; /* Too light */
}

h1 {
  color: #00d1ff; /* Blue — prints as dark gray */
}

// Force printing of backgrounds if needed
* {
  -webkit-print-color-adjust: exact !important;
  color-adjust: exact !important;
}

// But use sparingly — it wastes ink!
// Only use for logos or essential graphics

Pro tip: Most people print in grayscale. Design your documents assuming they'll be printed in black and white!

Uses: Why Print CSS Matters
  • Legal documents: Contracts, forms, reports need professional appearance
  • Education: Syllabi, handouts, worksheets
  • Business: Invoices, receipts, proposals
  • Accessibility: Some users prefer physical copies
  • Archival: Permanent records should be printable
  • User trust: Professional print quality builds credibility

23.2 PDF Generation and Print Layouts

Modern browsers can generate PDFs directly from HTML using CSS Paged Media — creating true print-quality documents without server-side processing.

Definition: CSS Paged Media Specifications

CSS Paged Media is the W3C specification that defines how CSS works with paginated output like PDFs and printed pages.

Key features include:

  • Page sizing and orientation
  • Headers and footers
  • Page counters
  • Running headers/footers
  • Page transitions
  • Page regions

Browser support is evolving — Chrome, Edge, and Safari have good support; Firefox is catching up.

Tools like Puppeteer and jsPDF use these standards under the hood.

Syntax: Headers and Footers

CSS Paged Media allows you to create headers and footers that appear on every page:

// Define page regions
@page {
  margin: 1in;
  
  /* Define header region */
  @top-center {
    content: "Professional Report";
    font-family: 'Arial', sans-serif;
    font-size: 12pt;
    color: #333;
    border-bottom: 1px solid #ccc;
    padding-bottom: 0.5em;
  }
  
  /* Define footer region */
  @bottom-center {
    content: "Confidential — Page " counter(page) " of " counter(pages);
    font-family: 'Arial', sans-serif;
    font-size: 10pt;
    color: #666;
    border-top: 1px solid #ccc;
    padding-top: 0.5em;
  }
  
  /* Define left and right headers */
  @top-left {
    content: "Document ID: DOC-2024-001";
    font-size: 9pt;
    color: #666;
  }
  
  @top-right {
    content: "Date: " string(date);
    font-size: 9pt;
    color: #666;
  }
}

/* Define content for headers/footers */
body {
  counter-increment: page;
  string-set: date content();
}

/* Or use custom content */
@page {
  @bottom-right {
    content: "© " string(copyright);
  }
}

.copyright {
  string-set: copyright "2024 My Company";
}

Important limitations:

  • Not all browsers support all Paged Media features
  • Dynamic content (like current date) requires JavaScript
  • Complex layouts are limited compared to desktop publishing software

@media print {
  @page {
    margin: 1in;
    
    @top-center {
      content: "Annual Report";
      font-size: 12pt;
      font-weight: bold;
    }
    
    @bottom-center {
      content: "Page " counter(page) " of " counter(pages);
      font-size: 10pt;
    }
  }
  
  .page-ref::after {
    content: " (page " target-counter(attr(href), page) ")";
    font-style: italic;
    color: #666;
  }
  
  /* Enable cross-referencing */
  body {
    counter-reset: page;
  }
  
  h1, h2 {
    counter-increment: page;
  }
}

Result: Clickable links in digital format become printed page references!

Key functions:

  • counter(page): Current page number
  • counter(pages): Total number of pages
  • target-counter(attr(href), page): Page number of referenced element
Syntax: Print Optimization Techniques

Optimize your print styles for performance and quality:

// 1. Reduce image resolution for print
img {
  max-width: 100%;
  height: auto;
  image-resolution: 300dpi; /* High quality for print */
}

// 2. Optimize fonts
@font-face {
  font-family: 'MyFont';
  src: url('myfont.woff2') format('woff2');
  font-display: swap;
}

// 3. Minimize print styles
@media print {
  /* Keep this minimal — only what's necessary */
  body {
    background-color: white;
    color: black;
    font-family: 'Times New Roman', serif;
    font-size: 12pt;
  }
  
  /* Remove unnecessary animations */
  * {
    animation: none !important;
    transition: none !important;
  }
  
  /* Remove shadows and gradients */
  * {
    box-shadow: none !important;
    text-shadow: none !important;
    filter: none !important;
  }
  
  /* Ensure tables are readable */
  table {
    width: 100%;
    border-collapse: collapse;
    margin: 1rem 0;
  }
  
  table th, table td {
    border: 1px solid #000;
    padding: 0.5rem;
  }
  
  table th {
    background-color: #f0f0f0;
    font-weight: bold;
  }
  
  /* Prevent orphaned headings */
  h1, h2, h3 {
    page-break-after: avoid;
  }
  
  /* Prevent widows/orphans */
  p {
    orphans: 3;
    widows: 3;
  }
  
  /* Ensure code blocks are readable */
  pre {
    white-space: pre-wrap;
    word-wrap: break-word;
    font-family: 'Courier New', monospace;
    font-size: 10pt;
    line-height: 1.4;
  }
  
  /* Hide non-essential elements */
  .no-print {
    display: none !important;
  }
}

Performance tips:

  • Keep print CSS file small — ideally under 5KB
  • Use external print stylesheet for better caching
  • Test with real printers — browser preview ≠ actual print
  • Consider grayscale printing as default
  • Use relative units (pt, cm, in) instead of px
