7.1 Display Property

The most important property for layout — it defines how an element generates boxes in the document flow.

Block vs inline elements

Every element has a default display value that determines its behavior in the document flow.

Property Block Elements Inline Elements
Box Model Takes full width, starts on new line Only takes needed width, flows with text
Width/Height Respects width and height properties Width and height ignored
Margin/Padding All sides respected Horizontal margin/padding respected, vertical ignored
Examples <div>, <p>, <h1>-<h6>, <ul>, <li>, <section> <span>, <a>, <strong>, <em>, <img>, <code> 

/* Block elements */
div {
    display: block; /* Default */
}

/* Inline elements */
span {
    display: inline; /* Default */
}
This is a DIV (block element)
This is a SPAN (inline element) Another SPAN
Another DIV (block element)

Key Insight: Block elements create "sections" while inline elements create "fragments" within text.

Pro Tip: You can change display properties to transform elements:


/* Make a div behave like inline */
.inline-div {
    display: inline;
}

/* Make a span behave like block */
.block-span {
    display: block;
}
Inline-block display

A hybrid display type that combines characteristics of both block and inline elements.


/* Inline-block elements */
.inline-block-element {
    display: inline-block;
    width: 100px;
    height: 100px;
    margin: 10px;
    background-color: #00d1ff;
    vertical-align: top;
}

IB 2
IB 3
This text flows around inline-block elements

Characteristics:

  • Behaves like inline: stays on same line as adjacent content
  • Behaves like block: respects width, height, margin, padding
  • Vertical alignment matters: use vertical-align to align
  • Creates whitespace issues: extra space between elements due to line breaks

Solving Whitespace Issue:


/* Option 1: Remove HTML whitespace */
<div class="ib">Item 1</div><div class="ib">Item 2</div>

/* Option 2: Set font-size: 0 on parent */
.parent {
    font-size: 0;
}

.child {
    display: inline-block;
    font-size: 16px;
}

/* Option 3: Use negative margins */
.child {
    display: inline-block;
    margin-right: -4px;
}

Use Cases:

  • Navigation menus
  • Button groups
  • Image galleries
  • Card layouts (before Flexbox/Grid)
None and contents values

Special display values that remove elements from normal layout flow.


/* Hide element completely */
.hidden {
    display: none;
}

/* Remove container but keep children */
.contents {
    display: contents;
}

/* Practical example */
.container {
    display: contents;
}

/* This structure: */
<div class="container">
    <div class="item">Item 1</div>
    <div class="item">Item 2</div>
</div>

/* Behaves as if container doesn't exist: */
<div class="item">Item 1</div>
<div class="item">Item 2</div>

display: none:

  • Removes element from document flow entirely
  • Doesn't take up any space
  • Not accessible to screen readers
  • Cannot be animated

display: contents:

  • Removes the container from layout but keeps children
  • Children inherit styles from parent container
  • Perfect for semantic HTML without affecting layout
  • Useful with CSS Grid/Flexbox

Accessibility Note: Never use display: none to hide content meant for screen readers. Use visibility: hidden or aria-hidden instead.

Display and accessibility implications

Display properties affect how assistive technologies interpret your content.


/* BAD: Hiding content from screen readers */
.accessible-content {
    display: none; /* Screen readers ignore this */
}

/* GOOD: Visually hiding but accessible */
.visually-hidden {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border: 0;
}

/* GOOD: Semantic hiding */
.sr-only {
    border: 0 !important;
    clip: rect(1px, 1px, 1px, 1px) !important;
    -webkit-clip-path: inset(50%) !important;
    clip-path: inset(50%) !important;
    height: 1px !important;
    margin: -1px !important;
    overflow: hidden !important;
    padding: 0 !important;
    position: absolute !important;
    white-space: nowrap !important;
    width: 1px !important;
}

This text is visible to everyone.

This text is only for screen readers

This text is also visible to everyone.

Accessibility Best Practices:

  1. Never use display: none for content that should be accessible
  2. Use visibility: hidden when you want to hide visually but keep in layout
  3. Use the .sr-only class pattern for screen reader-only content
  4. Ensure interactive elements remain focusable even when visually hidden
  5. Test with screen readers and keyboard navigation

Common Accessibility Mistakes:

  • Hiding form labels with display: none → screen readers can't read them
  • Using display: none for tab panels → users can't navigate to them
  • Setting display: none on navigation links → users can't access them

Pro Tip: Always ask: "Can a keyboard user reach this? Can a screen reader understand this?"

7.2 Position Property

Control an element's placement relative to its normal position or viewport.

Static positioning (default)

The default positioning mode — elements follow the normal document flow.


/* This is the default — no need to specify */
.element {
    position: static; /* Default value */
}
Normal block element
Another normal block element
Yet another normal block element

Characteristics:

  • Elements appear in order they're written in HTML
  • Top/right/bottom/left properties have no effect
  • z-index has no effect
  • Follows normal document flow

Why it matters: Everything starts here. Positioning becomes meaningful only when you change from static.

Relative positioning

Position an element relative to its normal position in the document flow.


.relative-element {
    position: relative;
    top: -10px;
    left: 20px;
    /* Moves element 10px up and 20px right from original position */
}
Normal element
Relative element (moved)
Normal element below

Key Characteristics:

  • Element remains in normal document flow — space preserved
  • Can be moved using top, right, bottom, left properties
  • z-index works — can stack over other elements
  • Perfect for fine-tuning positions

Real-World Use Case: Adjusting a tooltip slightly from its anchor point:


.tooltip-anchor {
    position: relative;
}

.tooltip {
    position: absolute;
    top: -30px;
    left: 50%;
    transform: translateX(-50%);
    background-color: #00d1ff;
    color: #000;
    padding: 5px 10px;
    border-radius: 5px;
    white-space: nowrap;
}

Pro Tip: Relative positioning is often used as a reference point for absolutely positioned children.

Absolute positioning

Remove an element from normal document flow and position it relative to its nearest positioned ancestor.


.parent {
    position: relative; /* Creates positioning context */
}

.child {
    position: absolute;
    top: 0;
    right: 0;
    /* Positioned relative to parent, not viewport */
}
Parent Container
Absolute Child
Normal element below

Key Characteristics:

  • Removed from normal document flow — space collapsed
  • Positioned relative to nearest positioned ancestor (not static)
  • If no positioned ancestor, uses viewport
  • z-index works — can be layered above/below other elements
  • Width/height can be set independently of content

Positioning Reference Points:


/* Top-right corner */
.top-right {
    position: absolute;
    top: 0;
    right: 0;
}

/* Bottom-left corner */
.bottom-left {
    position: absolute;
    bottom: 0;
    left: 0;
}

/* Centered horizontally */
.center-horizontal {
    position: absolute;
    left: 50%;
    transform: translateX(-50%);
}

/* Centered vertically */
.center-vertical {
    position: absolute;
    top: 50%;
    transform: translateY(-50%);
}

/* Perfect center */
.perfect-center {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

Common Use Cases:

  • Modal dialogs
  • Tooltips and popovers
  • Dropdown menus
  • Sticky headers/footers
  • Overlay icons
Fixed positioning

Position an element relative to the viewport — it stays in place during scrolling.


.fixed-element {
    position: fixed;
    top: 10px;
    right: 10px;
    /* Stays in same position even when page scrolls */
}

  • Removed from normal document flow
  • Always positioned relative to viewport (browser window)

Sticky positioning

A hybrid between relative and fixed positioning — behaves normally until scrolled to a threshold.


.sticky-element {
    position: sticky;
    top: 0;
    /* Sticks to top when scrolling past it */
}

Key Characteristics:

  • Behaves like relative until scroll threshold reached
  • Then behaves like fixed
  • Remains in document flow — doesn't collapse space
  • Requires a threshold (top, bottom, left, or right)
  • Excellent for headers, navbars, sidebars

Modern Best Practice: Use sticky positioning for headers instead of fixed!


header {
    position: sticky;
    top: 0;
    z-index: 100;
    background-color: #0a192f;
    padding: 15px;
    border-bottom: 1px solid rgba(0, 209, 255, 0.2);
}

.sidebar {
    position: sticky;
    top: 20px;
    height: calc(100vh - 40px);
    overflow-y: auto;
}

Browser Support: Excellent in modern browsers (Chrome 56+, Firefox 54+, Safari 11+).

Pro Tip: Always set z-index with sticky elements to ensure they appear above content.

7.3 Float and Clear

Historical layout technique — now largely replaced by Flexbox and Grid, but still relevant.

Understanding floats (historical context)

Float was the primary method for creating multi-column layouts before Flexbox and Grid.


.float-left {
    float: left;
}

.float-right {
    float: right;
}

.float-none {
    float: none; /* Default */
}
Float Left
Float Left
Float Left

This text wraps around the floated elements. Float was originally designed for text wrapping around images!

Original Purpose: Float was created for text wrapping around images in newspapers and magazines.

How it worked: Floated elements were removed from normal flow and pushed to one side, allowing content to wrap around them.

Legacy Usage: Before Flexbox, we'd create layouts like this:


/* Two-column layout with floats */
.sidebar {
    float: left;
    width: 250px;
}

.content {
    float: right;
    width: calc(100% - 250px);
}

Why it's outdated:

  • Complex clearing required
  • Difficult to equalize column heights
  • Unpredictable behavior with different content lengths
  • Poor responsive behavior

Modern Alternative: Use Flexbox or Grid instead!

Clearing floats

Prevent content from wrapping around floated elements.


.clearfix {
    clear: both;
}

.clear-left {
    clear: left;
}

.clear-right {
    clear: right;
}

.clear-none {
    clear: none; /* Default */
}
Float Left
Float Left

This paragraph will appear below the floated elements because of clear: both.

Clearing Problem: Without clearing, content can flow into unexpected places:

Tall Float
Short Float

This text wraps around the short float but not the tall one, creating a messy layout!

Why clearing matters: Prevents unintended wrapping and ensures predictable layout flow.

Clearfix techniques

Methods to automatically clear floats without adding extra markup.


/* Method 1: Overflow auto (modern) */
.clearfix {
    overflow: auto;
}

/* Method 2: Flexbox clearfix */
.clearfix {
    display: flex;
    flex-wrap: wrap;
}

/* Method 3: Pseudo-element clearfix (classic) */
.clearfix::after {
    content: "";
    display: table;
    clear: both;
}

/* Method 4: Grid clearfix */
.clearfix {
    display: grid;
}

Float Left
Float Left

This text now appears below the floats thanks to clearfix!

Classic Clearfix (Recommended):


.clearfix::after {
    content: "";
    display: table;
    clear: both;
}

Why it works:

  • content: "": Creates pseudo-element
  • display: table: Makes it a block-level element
  • clear: both: Forces it to clear all floats

Modern Approach: Avoid floats entirely! Use Flexbox:


.flex-container {
    display: flex;
    gap: 15px;
}

.flex-item {
    flex: 1;
    min-width: 100px;
}

Despite being outdated for layout, floats still have valid use cases.

    margin-right: 15px;
    margin-bottom: 15px;
}

img.right {
    float: right;
    margin-left: 15px;
    margin-bottom: 15px;
}

7.4 Overflow and Clipping

Control what happens when content exceeds its container boundaries.

Overflow property values

Determine how content behaves when it overflows its container.


.overflow-visible {
    overflow: visible; /* Default */
}

.overflow-hidden {
    overflow: hidden;
}

.overflow-scroll {
    overflow: scroll;
}

.overflow-auto {
    overflow: auto;
}
Visible: The content extends beyond the container boundary!
Large child element
Hidden: Content is clipped and invisible outside container
Large child element
Scroll: Always shows scrollbars, even when not needed
Large child element
Auto: Shows scrollbars only when content overflows
Large child element

Behavior Summary:

Value Content Visibility Scrollbars Best For
visible Always visible Never shown Overlays, tooltips
hidden Clipped Never shown Cards, modals, containers
scroll Clipped Always shown Fixed-height containers
auto Clipped Shown only when needed ✅ Recommended for most cases

Pro Tip: Use overflow: auto unless you have a specific reason not to.

Text overflow and ellipsis

Truncate long text with an ellipsis (...) for clean UI design.


.text-ellipsis {
    white-space: nowrap;
    overflow: hidden;
    text-overflow: ellipsis;
}

This is a very long text string that will be truncated with an ellipsis when it exceeds the container width.

Three Properties Required:

  1. white-space: nowrap; — Prevents line breaks
  2. overflow: hidden; — Hides overflowing content
  3. text-overflow: ellipsis; — Replaces overflow with "..."

Multi-line Ellipsis (CSS Grid/Flexbox):


.multi-line-ellipsis {
    display: -webkit-box;
    -webkit-line-clamp: 3; /* Number of lines to show */
    -webkit-box-orient: vertical;
    overflow: hidden;
}

This is a very long text string that will be truncated after three lines with an ellipsis. It's perfect for product descriptions, article previews, and comments.

Browser Support: -webkit-line-clamp works in Chrome, Safari, Edge, and newer Firefox versions.

Use Cases:

  • Product titles in catalogs
  • Article headlines
  • Email subject lines
  • File names in file managers
  • Comment previews
Clip and clip-path properties

Advanced methods for masking content shapes.


/* Deprecated: clip property */
.old-clip {
    position: absolute;
    clip: rect(0px, 60px, 60px, 0px);
}

/* Modern: clip-path */
.modern-clip {
    clip-path: circle(50% at 50% 50%);
}

/* Various clip-path shapes */
.clip-circle {
    clip-path: circle();
}

.clip-ellipse {
    clip-path: ellipse(50% 50% at 50% 50%);
}

.clip-polygon {
    clip-path: polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%);
}

.clip-rounded {
    clip-path: inset(10px round 20px);
}
Original Image
Circle Clip
Diamond Shape
Rounded Rectangle

clip-path Values:

  • circle(): Circular shape
  • ellipse(): Elliptical shape
  • polygon(): Any polygon shape
  • inset(): Rectangle with rounded corners
  • path(): SVG path data

Modern Use Cases:

  • Profile pictures with custom shapes
  • Interactive image galleries
  • Unique card designs
  • Animated clipping effects

Performance Note: Complex clip-paths can be expensive — test on low-end devices.

Creating scrollable areas

Build controlled scrolling experiences for better UX.


.scrollable-container {
    overflow-y: auto;
    max-height: 300px;
}

/* Custom scrollbar styling */
.scrollable-container::-webkit-scrollbar {
    width: 8px;
}

.scrollable-container::-webkit-scrollbar-track {
    background: transparent;
}

.scrollable-container::-webkit-scrollbar-thumb {
    background: rgba(0, 209, 255, 0.3);
    border-radius: 10px;
}

.scrollable-container::-webkit-scrollbar-thumb:hover {
    background: rgba(0, 209, 255, 0.5);
}

/* Horizontal scrolling */
.horizontal-scroll {
    overflow-x: auto;
    white-space: nowrap;
}

/* Scroll snap points */
.snap-container {
    overflow-x: auto;
    scroll-snap-type: x mandatory;
}

.snap-item {
    scroll-snap-align: start;
    flex-shrink: 0;
