4.1 Absolute Units

Fixed measurements that don't change based on context ‚Äî useful for precise control.

Pixels (px)

px is the most common unit in web design ‚Äî it represents one device pixel.


.button {
    width: 120px;
    height: 40px;
    font-size: 16px;
    border-radius: 8px;
}

.logo {
    width: 200px;
    height: 100px;
}

Why pixels dominate:

  • Pixel-perfect control ‚Äî consistent across devices
  • Easy to visualize ‚Äî 1px = 1 dot on screen
  • Perfect for borders, icons, and fixed elements

Pro Tip: Use px for:

  • Borders (1px, 2px)
  • Icons and small graphics
  • Fixed spacing between elements
  • Border radius values

Myth Busting: "Pixels aren't really pixels!" ‚Äî On high-DPI screens (Retina), 1 CSS pixel may equal 2 or more physical pixels.

Points (pt), inches (in), centimeters (cm)

Traditional print units that are rarely used on the web.


/* Print styles */
@media print {
    body {
        font-size: 12pt; /* 1 point = 1/72 inch */
    }
    
    .page {
        width: 8.5in;   /* US Letter width */
        height: 11in;   /* US Letter height */
        margin: 1cm;    /* 1 centimeter margin */
    }
}
Unit Equivalent Use Case
1pt 1/72 inch Print styling
1in 96px Print layouts
1cm 37.8px Print layouts
1mm 3.78px Print layouts
1pc 12pt Print typography

Reality Check: These units are unpredictable on screens because they depend on device DPI (dots per inch). A 1cm element might be 1cm on a printer but only 0.8cm on your monitor!

When to use: Only in @media print stylesheets ‚Äî never for web layouts.

When to use absolute units

Absolute units have specific, limited roles in modern web development.

‚úÖ Good Uses:

  • px: Border widths, icon sizes, fixed-spacing elements
  • px: Animation durations (e.g., transition: 0.3s)
  • pt/in/cm: Print stylesheets only
  • px: Canvas dimensions and SVG sizing

‚ùå Avoid Using:

  • Font sizes (use rem instead)
  • Container widths (use %, vw, clamp())
  • Padding/margin for responsive layouts
  • Any unit except px for screen layouts

Deep Insight: The web is fundamentally responsive. Absolute units break this paradigm by creating rigid, non-adaptive layouts.


/* BAD - Rigid layout */
.container {
    width: 960px; /* Forces desktop-only experience */
    padding: 20px;
}

/* GOOD - Flexible layout */
.container {
    width: 100%;
    max-width: 960px;
    padding: 2rem;
}

4.2 Relative Units

Flexible measurements that adapt to context ‚Äî the secret sauce of responsive design.

Em and rem units

Both are relative to font size, but with different reference points.

Unit Reference Behavior Use Case
em Parent element's font-size Compounding ‚Äî multiplies up the hierarchy Components that scale with parent text
rem Root element's font-size (<html>) Consistent ‚Äî no compounding Global sizing (recommended)

Example:


/* Base font size */
html {
    font-size: 16px; /* Default browser setting */
}

/* Using em */
.parent {
    font-size: 2em; /* 32px */
    padding: 1em;   /* 32px */
}

.child {
    font-size: 1.5em; /* 48px! (1.5 √ó 32px) */
    padding: 0.5em; /* 24px */
}

/* Using rem */
.parent {
    font-size: 2rem; /* 32px */
    padding: 1rem;   /* 16px */
}

.child {
    font-size: 1.5rem; /* 24px (always relative to root) */
    padding: 0.5rem; /* 8px */
}

Why rem is preferred: Predictable scaling. No unexpected compounding.

Best Practice: Set root font size for accessibility:


html {
    font-size: 100%; /* 16px */
}

/* For accessibility: allow users to adjust base font size */
@media (prefers-reduced-motion: reduce) {
    html {
        font-size: 110%;
    }
}
Percentages (%)

Relative to parent element's property value.


/* Width relative to parent */
.container {
    width: 80%; /* 80% of parent's width */
}

/* Height relative to parent */
.content {
    height: 50%; /* 50% of parent's height */
}

/* Padding relative to width */
.card {
    padding: 5%; /* 5% of container's WIDTH, not height! */
}

/* Margin relative to width */
.item {
    margin: 2% 1%; /* 2% top/bottom, 1% left/right of parent's width */
}

Important Quirk: Vertical padding/margin percentages are calculated based on the width of the containing block, not its height!


.parent {
    width: 400px;
    height: 200px;
    background-color: #0d1b2a;
}

.child {
    width: 50%; /* 200px */
    padding: 10%; /* 40px (10% of 400px width) */
    background-color: var(--amber);
}

Use Cases:

  • Fluid containers (width: 100%)
  • Responsive grids
  • Percentage-based spacing
  • Aspect ratio boxes (padding-bottom: 56.25%)
Viewport units (vw, vh, vmin, vmax)

Units relative to the viewport (browser window) dimensions.

Unit Description Example
vw 1% of viewport width font-size: 5vw; ‚Üí 5% of screen width
vh 1% of viewport height height: 100vh; ‚Üí full viewport height
vmin 1% of smaller dimension (width or height) Safe for responsive text on any device
vmax 1% of larger dimension (width or height) Fill maximum available space

Real-world examples:

Viewport Text (5vw)
Half-Height Container (50vh)
Circle using vmin

Powerful Pattern: Fluid typography with viewport units:


h1 {
    font-size: clamp(1.5rem, 8vw, 4rem);
}

/* This means: 
   Minimum: 1.5rem
   Ideal: 8vw (scales with screen)
   Maximum: 4rem
*/

Pro Tip: Combine clamp() with vw for truly responsive text that never gets too small or too large.

Character units (ch, ex)

Specialized units based on character dimensions.

  • ch: Width of the "0" (zero) character in the current font
  • ex: Height of the lowercase "x" character in the current font

/* Limit text to approximately 80 characters wide */
.text-limit {
    width: 80ch;
    margin: 0 auto;
}

/* Align text with x-height */
.x-aligned {
    margin-top: 1ex;
}

/* Create typographic rhythm */
blockquote {
    padding-left: 2ch;
    border-left: 3px solid var(--amber);
}

/* Create narrow columns for readability */
.article {
    width: 60ch;
    margin: 0 auto;
    line-height: 1.6;
}

Why ch is powerful: Perfect for creating readable text blocks. 60-80ch is the ideal reading width for paragraphs.


<p style="width: 70ch; line-height: 1.6;">
    Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
</p>

Use Cases:

  • ch: Creating optimal reading width for articles
  • ch: Input fields with character limits
  • ex: Fine-tuning vertical alignment

4.3 CSS Functions

Dynamic calculation functions that bring intelligence to your CSS.

calc() for calculations

Perform mathematical operations directly in CSS.


/* Basic arithmetic */
.element {
    width: calc(100% - 20px);
    margin-left: calc(50% - 100px);
    padding: calc(1rem + 5px);
    font-size: calc(16px + 2vw);
}

/* Complex expressions */
.hero-section {
    height: calc(100vh - 80px); /* Full height minus header */
}

.grid-item {
    width: calc((100% - 3 * 20px) / 3); /* 3 columns with 20px gaps */
}

/* Mixing units */
.responsive-box {
    width: calc(200px + 10vw);
    padding: calc(1em + 5px);
}
                        

/* Practical example: centered element with margins */
.centered-card {
    width: calc(100% - 40px);
    margin: 0 20px;
}

Rules:

  • Always include spaces around operators: calc(100% - 20px) ‚úÖ, calc(100%-20px) ‚ùå
  • Can mix any units: %, px, em, rem, vw, etc.
  • Supports +, -, *, / operators

Pro Tip: Use calc() to create fluid systems that combine fixed and flexible units.

min(), max(), clamp() for responsive design

These functions create intelligent, adaptive sizing.

Function Parameters Behavior Use Case
min(a, b) Two values Returns the smaller value Prevent elements from getting too small
max(a, b) Two values Returns the larger value Ensure minimum size
clamp(min, preferred, max) Three values Bounds a value between min and max, with preferred as target Perfect fluid typography

Examples:


/* Prevent text from becoming too small */
h1 {
    font-size: clamp(1.5rem, 5vw, 3rem);
}

/* Ensure minimum container width */
.container {
    width: max(80%, 800px);
}

/* Limit maximum width */
.max-width {
    max-width: min(100%, 1200px);
}

/* Responsive spacing */
.padding {
    padding: clamp(1rem, 5vw, 3rem);
}

/* Fluid grid gap */
.grid {
    gap: clamp(1rem, 4vw, 4rem);
}

Why clamp() is revolutionary: It replaces multiple media queries with one elegant rule.

Visual Representation:


/* Without clamp() - Requires multiple media queries */
@media (max-width: 480px) { h1 { font-size: 1.5rem; } }
@media (min-width: 481px) and (max-width: 1200px) { h1 { font-size: 5vw; } }
@media (min-width: 1201px) { h1 { font-size: 3rem; } }

/* With clamp() - One line! */
h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
var() for custom properties

Create reusable variables for consistent theming and maintenance.


/* Define custom properties (CSS variables) */
:root {
    --primary-color: #ffa726;
    --secondary-color: #00d1ff;
    --border-radius: 8px;
    --spacing-unit: 1rem;
    --font-size-base: 16px;
    --shadow-small: 0 2px 8px rgba(0, 0, 0, 0.1);
    --animation-speed: 0.3s;
}

/* Use them anywhere */
.button {
    background-color: var(--primary-color);
    color: white;
    padding: calc(var(--spacing-unit) * 1.5);
    border-radius: var(--border-radius);
    transition: all var(--animation-speed) ease;
}

.card {
    border: 1px solid var(--secondary-color);
    padding: var(--spacing-unit);
    border-radius: var(--border-radius);
    box-shadow: var(--shadow-small);
}

/* Update theme dynamically with JavaScript */
document.documentElement.style.setProperty('--primary-color', '#00d1ff');
                        

/* Theme switching example */
body[data-theme="dark"] {
    --bg-color: #0a192f;
    --text-color: #e6f1ff;
    --accent-color: #ffb74d;
}

body[data-theme="light"] {
    --bg-color: #ffffff;
    --text-color: #333333;
    --accent-color: #ff6b6b;
}

/* Now your entire site can switch themes with one attribute change! */

Benefits:

  • Single source of truth for colors, sizes, spacing
  • Easily update theme globally
  • Improved maintainability
  • Works with JavaScript for dynamic theming

Best Practice: Use descriptive names: --primary-color not --color1.

4.4 Color Values

The palette of your design ‚Äî choose wisely for accessibility, aesthetics, and performance.

Named colors

147 predefined color names recognized by browsers.


.element {
    color: red;
    background-color: blue;
    border-color: green;
}

/* Common named colors */
.title {
    color: coral;
    border-bottom: 2px solid goldenrod;
}

.highlight {
    background-color: lightseagreen;
    color: white;
}

Pros: Easy to remember, human-readable

Cons: Limited palette, inconsistent naming

Recommendation: Use sparingly ‚Äî stick to hex/RGB/HSL for consistency.

RoyalBlue

Hexadecimal notation

Color codes in hexadecimal format: #RRGGBB or #RGB.


/* Full hex: #RRGGBB */
.primary {
    color: #ff6b6b;
    background-color: #4ecdc4;
}

/* Short hex: #RGB (each digit repeated) */
.secondary {
    color: #f00;     /* Red (#ff0000) */
    background: #0cf; /* Cyan (#00ccff) */
}

/* Hex with alpha: #RRGGBBAA */
.transparent-bg {
    background-color: #ff6b6b80; /* 50% opacity red */
}

Hex Guide:

  • #000000 = black
  • #ffffff = white
  • #ff0000 = pure red
  • #00ff00 = pure green
  • #0000ff = pure blue
  • #ffff00 = yellow
  • #ff00ff = magenta
  • #00ffff = cyan

Modern Recommendation: Use 8-digit hex (#RRGGBBAA) for transparency instead of RGBA.

RGB and RGBA

Red-Green-Blue model with optional alpha channel.


/* RGB: 0-255 range */
.primary {
    color: rgb(255, 107, 107); /* Same as #ff6b6b */
    background-color: rgb(78, 205, 196); /* Same as #4ecdc4 */
}

/* RGBA: Add alpha (0-1) */
.overlay {
    background-color: rgba(255, 107, 107, 0.8); /* 80% opacity */
}

/* Modern alternative: rgb() with percentage */
.text {
    color: rgb(100% 42% 42%); /* Same as rgb(255, 107, 107) */
}

Alpha Values:

  • 0 = fully transparent
  • 0.5 = 50% transparent
  • 1 = fully opaque

Pro Tip: Use RGBA when you need transparency without affecting child elements' opacity.

HSL and HSLA

Hue-Saturation-Lightness ‚Äî more intuitive than RGB for designers.


/* HSL: hue (0-360¬∞), saturation (0-100%), lightness (0-100%) */
.primary {
    color: hsl(0, 100%, 50%); /* Red */
    background-color: hsl(180, 50%, 60%); /* Teal */
}

/* HSLA: Add alpha */
.overlay {
    background-color: hsla(0, 100%, 50%, 0.7); /* 70% opacity red */
}

/* Adjusting colors easily */
.btn-primary {
    background-color: hsl(30, 90%, 60%);
}

.btn-primary:hover {
    background-color: hsl(30, 90%, 55%); /* Just darken slightly */
}

.btn-primary:active {
    background-color: hsl(30, 90%, 45%); /* Even darker */
}

Why HSL wins:

  • Hue: 0¬∞=red, 120¬∞=green, 240¬∞=blue
  • Saturation: 0%=gray, 100%=pure color
  • Lightness: 0%=black, 50%=normal, 100%=white

CSS Color Level 4 (hwb, lab, lch)

Modern color models that offer superior perceptual uniformity.

HWB (Hue-Whiteness-Blackness): Intuitive for designers


.color-hwb {
    color: hwb(120 0% 0%); /* Pure green */
    background-color: hwb(120 50% 0%); /* Light green */
    border-color: hwb(120 0% 50%); /* Dark green */
}

LAB and LCH (CIELAB/LCH): Perceptually uniform ‚Äî colors appear equally spaced


/* LAB: Lightness, a*, b* */
.color-lab {
    color: lab(53% 80.1 67.1); /* Vibrant red */
    background-color: lab(75% 10 30); /* Soft pink */
}

/* LCH: Lightness, Chroma, Hue */
.color-lch {
    color: lch(53% 80.1 25.3); /* Vibrant red */
    background-color: lch(75% 30 30); /* Soft pink */
}

Why these matter:

  • LCH: Most perceptually accurate ‚Äî changes look natural
  • Lab: Device-independent ‚Äî consistent across displays
  • HWB: Easiest for humans to understand

Browser Support: Modern browsers (Chrome 116+, Firefox 117+, Safari 16.4+)

Future of Colors: These new models solve problems with traditional RGB/HSL where changing one component doesn't produce visually predictable results.


/* Example: Creating a gradient with perceptual accuracy */
.gradient {
    background: linear-gradient(to right, 
        lch(50% 50 25), 
        lch(50% 50 125), 
        lch(50% 50 225));
}

Recommendation: Start with HSL for now. Experiment with LCH for future-proof designs.

üåü Mastery Tip: Choose units based on context ‚Äî not preference.
Use rem for typography, vw for full-width elements, calc() for complex combinations,
and clamp() for truly responsive sizing. Avoid absolute units for scalable design.
For colors, prefer HSL for practicality and LCH for professional-grade color systems.
