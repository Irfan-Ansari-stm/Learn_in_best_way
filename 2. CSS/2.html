CSS Selectors

2.1 Basic Selectors

These are the ABCs of CSS targeting ‚Äî learn them first, master them forever.

Element selectors

Target HTML elements by their tag name. Simple, direct, and universally supported.


/* Targets ALL paragraph elements */
p {
    color: cyan;
    font-size: 1.1rem;
}

/* Targets ALL heading level 1 elements */
h1 {
    color: amber;
    text-transform: uppercase;
}

Real-world example:

This paragraph will be cyan.

This one too!

This heading will be amber and uppercase.

Pros: Lightweight, no extra attributes needed.
Cons: Too broad ‚Äî affects every instance of that element.

Class selectors (.class)

Target elements with a specific class attribute. Reusable across multiple elements.


/* Targets any element with class="btn" */
.btn {
    background-color: #ffa726;
    color: #000;
    padding: 10px 20px;
    border: none;
    border-radius: 4px;
}

/* Targets elements with class="highlight" */
.highlight {
    background-color: rgba(255, 183, 77, 0.2);
    padding: 2px 5px;
    border-radius: 3px;
}

HTML usage:


<button class="btn">Click Me</button>
<p class="btn highlight">This is both a button and highlighted!</p>
<span class="highlight">Just highlighted text.</span>

Why classes rock: One class can style multiple element types. You can apply multiple classes to one element: class="btn primary large"

Pro Tip: Use semantic names like .error-message, not .red-text.

ID selectors (#id)

Target a single, unique element using its ID. Must be unique per page.


/* Targets element with id="main-header" */
#main-header {
    font-size: 2.5rem;
    color: #00d1ff;
    text-align: center;
    margin-bottom: 30px;
}

/* Targets element with id="sidebar" */
#sidebar {
    width: 250px;
    background-color: #0d1b2a;
    padding: 20px;
}

HTML usage:


<header id="main-header">Welcome to My Site</header>
<aside id="sidebar">...</aside>

Key Rules:

  • ID must be unique ‚Äî never use the same ID twice on a page
  • Higher specificity than classes (more on this later)
  • Use for major structural elements: header, footer, main-content

Warning: Avoid overusing IDs. They're hard to reuse and create coupling between HTML and CSS.

Universal selector (*)

Selects EVERY element on the page. Rarely used in production but essential for resets.


/* Resets default margin and padding for ALL elements */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

/* Applies border to EVERY element (for debugging) */
* {
    border: 1px solid rgba(0, 209, 255, 0.3);
}

When to use:

  • CSS resets (normalize browser defaults)
  • Debugging layout issues
  • Applying global properties like box-sizing: border-box

Performance Note: The universal selector is slowest because it checks every element. Use sparingly.

Selector specificity basics

When multiple rules target the same element, CSS uses specificity to decide which wins.

Specificity Hierarchy (Highest to Lowest):

  1. Inline styles (e.g., style="color:red") ‚Üí 1,0,0,0
  2. ID selectors (#id) ‚Üí 0,1,0,0
  3. Class selectors, attributes, pseudo-classes (.class, [type], :hover) ‚Üí 0,0,1,0
  4. Element selectors, pseudo-elements (div, p, ::before) ‚Üí 0,0,0,1
  5. Universal selector, combinators (*, >, +, ~, ) ‚Üí 0,0,0,0

Example:


/* Specificity: 0,0,1,0 */
.title { color: blue; }

/* Specificity: 0,1,0,0 */
#header { color: red; }

/* Specificity: 0,0,0,1 */
h1 { color: green; }

/* Which wins? */
<h1 id="header" class="title">Hello World</h1>

Result: The text will be red ‚Äî ID beats class and element!

Calculating Specificity: Think of it as a 4-digit number: [inline][id][class][element]

Selector Specificity Value
* {} 0,0,0,0 0
p {} 0,0,0,1 1
.btn {} 0,0,1,0 10
#nav {} 0,1,0,0 100
div p span {} 0,0,0,3 3
div #nav .btn {} 0,1,1,1 111
<div style="color:red"> 1,0,0,0 1000

Rule of Thumb: Always prefer classes over IDs. Avoid inline styles. Never use !important unless absolutely necessary.

2.2 Combinators and Relationships

Selectors aren't just about naming elements ‚Äî they're about understanding relationships between them.

Descendant combinator (space)

Selects elements that are descendants (children, grandchildren, etc.) of another element.


/* Selects ALL li elements inside ul */
ul li {
    color: cyan;
}

/* Selects ALL spans inside ANY div */
div span {
    font-weight: bold;
}

HTML Example:


<div>
    <ul>
        <li>Item 1</li>
        <li>Item 2</li>
        <li>
            <ul>
                <li>Nested Item</li>  ‚Üê This gets styled too!
            </ul>
        </li>
    </ul>
    <span>This span is outside the ul</span> ‚Üê Not styled
</div>

Key Insight: Descendant combinator is greedy ‚Äî it selects all levels down.

Child combinator (>)

Selects only DIRECT children ‚Äî not grandchildren or deeper.


/* Selects ONLY li elements that are direct children of ul */
ul > li {
    color: amber;
    font-weight: bold;
}

/* This won't affect nested lists */
ul > li > ul > li {
    color: cyan;
}

HTML Example:


<ul>
    <li>Direct child ‚Üí amber</li>
    <li>
        <ul>
            <li>Grandchild ‚Üí NOT amber (unless explicitly targeted)</li>
        </ul>
    </li>
</ul>

Use Case: When you want precise control ‚Äî e.g., style top-level navigation items differently from dropdown items.

Adjacent sibling combinator (+)

Selects an element that is immediately preceded by another specific element.


/* Selects h2 that comes RIGHT AFTER h1 */
h1 + h2 {
    color: cyan;
    margin-top: 10px;
}

/* Selects p that comes RIGHT AFTER img */
img + p {
    margin-top: 0;
    font-weight: bold;
}

HTML Example:


<h1>Main Title</h1>
<h2>Subtitle</h2>      ‚Üê This gets styled

<h2>Another Subtitle</h2>  ‚Üê This does NOT get styled (not after h1)

<img src="image.jpg" alt="Sample" />
<p>This paragraph follows an image ‚Üí bold</p>
<p>This paragraph follows another paragraph ‚Üí normal</p>

Use Case: Styling headings after titles, paragraphs after images, form labels after inputs.

General sibling combinator (~)

Selects elements that follow another element anywhere in the same parent ‚Äî not necessarily adjacent.


/* Selects ALL p elements that come after h1 (anywhere after) */
h1 ~ p {
    color: amber;
    font-style: italic;
}

/* Selects ALL .note elements after .warning */
.warning ~ .note {
    background-color: rgba(255, 183, 77, 0.1);
    border-left: 3px solid var(--amber);
}

HTML Example:


<h1>Article Title</h1>
<p>First paragraph</p>
<div>Some other content</div>
<p>Third paragraph ‚Üí amber and italic</p>
<p>Fourth paragraph ‚Üí amber and italic</p>

<div class="warning">Warning message</div>
<p>Normal paragraph</p>
<div class="note">Note 1 ‚Üí styled</div>
<div class="note">Note 2 ‚Üí styled</div>

Compare: + = immediate next sibling | ~ = any following sibling

2.3 Attribute Selectors

Target elements based on their attributes ‚Äî powerful for forms, icons, and dynamic content.

Basic attribute selectors ([attr])

Selects elements that have a specific attribute, regardless of its value.


/* Targets ALL elements with a title attribute */
[title] {
    border-bottom: 1px dotted var(--cyan);
    cursor: help;
}

/* Targets ALL links with href attribute */
[href] {
    color: var(--amber);
    text-decoration: none;
}

/* Targets ALL images with alt attribute */
[alt] {
    border: 1px solid rgba(0, 209, 255, 0.3);
}

Use Case: Accessibility styling ‚Äî highlight elements with descriptive attributes.

Exact value matching ([attr="value"])

Selects elements where the attribute equals EXACTLY the specified value.


/* Targets input elements with type="email" */
input[type="email"] {
    background-image: url("email-icon.png");
    background-position: right 10px center;
    background-repeat: no-repeat;
}

/* Targets buttons with variant="primary" */
button[variant="primary"] {
    background-color: #00d1ff;
    color: #000;
}

/* Targets links with target="_blank" */
a[target="_blank"] {
    padding-right: 20px;
    background-image: url("external-link.svg");
    background-repeat: no-repeat;
    background-position: right center;
}

HTML Examples:


<input type="email" placeholder="your@email.com" />
<button variant="primary">Submit</button>
<a href="https://google.com" target="_blank">Visit Google</a>

Important: Values are case-sensitive. [type="Email"] ‚â† [type="email"]

Partial matching ([attr*="value"])

Selects elements whose attribute value contains a substring.


/* Targets all links containing "github" in href */
a[href*="github"] {
    color: #181717;
    background-color: #f0f0f0;
    border-radius: 5px;
    padding: 3px 6px;
}

/* Targets all images with "logo" in src */
img[src*="logo"] {
    width: 100px;
    margin: 10px auto;
    display: block;
}

/* Targets all classes containing "btn" */
[class*="btn"] {
    padding: 8px 16px;
    border: none;
    border-radius: 4px;
}

Use Case: Styling social media icons, file type indicators, component variants.

Beginning/ending matching ([attr^="value"], [attr$="value"])

Match attributes starting with or ending with a specific string.


/* Links that start with "https://" */
a[href^="https://"] {
    background-image: url("lock-icon.svg");
    background-position: left 5px center;
    background-repeat: no-repeat;
    padding-left: 25px;
}

/* Links that end with ".pdf" */
a[href$=".pdf"] {
    background-image: url("pdf-icon.svg");
    background-position: left 5px center;
    background-repeat: no-repeat;
    padding-left: 25px;
}

/* Elements with class starting with "grid-" */
[class^="grid-"] {
    display: grid;
}

/* Elements with class ending with "-btn" */
[class$="-btn"] {
    cursor: pointer;
}

Real-World Example: Styling different file download links:


<a href="resume.pdf">Download Resume</a>
<a href="notes.docx">Download Notes</a>
<a href="https://youtube.com">Watch Video</a>
<a href="https://github.com/user/repo">View Code</a>

All these links can be uniquely styled without adding extra classes!

Case-insensitive matching

Modern browsers support case-insensitive attribute matching using the i flag.


/* Matches "PDF", "Pdf", "pdf", "PdF" ‚Äî all variations */
a[href$=".pdf" i] {
    background-image: url("pdf-icon.svg");
    padding-left: 25px;
}

/* Matches class="btn-primary", "BTN-PRIMARY", "Btn-Primary" */
[class*="btn" i] {
    padding: 8px 16px;
    border-radius: 4px;
}

Browser Support: Supported in all modern browsers (Chrome 104+, Firefox 104+, Safari 15.4+).

Pro Tip: Use this for robustness when dealing with user-generated content or CMS systems that may vary casing.

2.4 Pseudo-classes and Pseudo-elements

These magical selectors let you style elements based on state or generate content ‚Äî no HTML changes needed.

Link states (:hover, :visited, :active)

Pseudo-classes for link interaction states. Order matters due to specificity cascade.


/* Default link state */
a {
    color: var(--cyan);
    text-decoration: none;
}

/* Hover state */
a:hover {
    color: var(--amber);
    text-decoration: underline;
    transition: color 0.3s ease;
}

/* Visited state */
a:visited {
    color: #9c58d1;
}

/* Active state (when clicked) */
a:active {
    color: #ff4081;
    transform: scale(0.98);
}

/* Focus state (keyboard navigation) */
a:focus {
    outline: 2px solid var(--cyan);
    outline-offset: 2px;
}

Remember the LVHA order: Link ‚Üí Visited ‚Üí Hover ‚Üí Active

Why order matters: If hover comes before visited, hovering over visited links won't work!

Accessibility Tip: Always include focus states ‚Äî keyboard users depend on them!

Form states (:focus, :disabled, :checked)

Style interactive form elements based on user interaction.


/* Focus state for inputs */
input:focus {
    outline: none;
    border-color: var(--cyan);
    box-shadow: 0 0 0 2px rgba(0, 209, 255, 0.3);
    transition: border-color 0.3s ease;
}

/* Disabled state */
input:disabled {
    background-color: #0d1b2a;
    color: #555;
    cursor: not-allowed;
}

/* Checked state for checkboxes/radios */
input[type="checkbox"]:checked + label {
    color: var(--amber);
    font-weight: bold;
}

/* Style custom checkbox */
input[type="checkbox"] {
    appearance: none;
    width: 20px;
    height: 20px;
    border: 2px solid var(--cyan);
    border-radius: 4px;
    position: relative;
}

input[type="checkbox"]:checked::after {
    content: "‚úì";
    position: absolute;
    top: -2px;
    left: 5px;
    font-size: 16px;
    color: var(--amber);
}

Structural pseudo-classes (:first-child, :last-child, :nth-child())

Target elements based on their position among siblings.


/* First child of any parent */
:first-child {
    margin-top: 0;
}

/* Last child of any parent */
:last-child {
    margin-bottom: 0;
}

/* First paragraph in container */
.container p:first-child {
    font-size: 1.2rem;
    font-weight: bold;
}

/* Every odd row in a table */
tr:nth-child(odd) {
    background-color: rgba(0, 209, 255, 0.05);
}

/* Every 3rd item */
li:nth-child(3n) {
    color: var(--amber);
    font-weight: bold;
}

/* First 3 items */
li:nth-child(-n+3) {
    font-size: 1.3rem;
}

/* Only child */
:only-child {
    text-align: center;
}

/* Empty elements */
:empty {
    display: none;
}

Understanding nth-child(n):

  • n = every element
  • 2n = even numbers (2,4,6...)
  • 2n+1 = odd numbers (1,3,5...)
  • 3n+2 = 2nd, 5th, 8th, 11th...
  • -n+3 = first 3 elements

Practical Example: Zebra-striped tables without JavaScript:


<table>
    <tr><td>Row 1</td></tr>
    <tr><td>Row 2</td></tr>
    <tr><td>Row 3</td></tr>
</table>

tr:nth-child(odd) { background: #0d1b2a; }
tr:nth-child(even) { background: #0a192f; }
Pseudo-elements (::before, ::after, ::first-line, ::first-letter)

Generate content or style parts of elements without changing HTML.


/* Add quotation marks around blockquotes */
blockquote::before {
    content: "‚Äú";
    font-size: 3rem;
    color: var(--amber);
    opacity: 0.7;
    position: absolute;
    top: -10px;
    left: -10px;
}

blockquote::after {
    content: "‚Äù";
    font-size: 3rem;
    color: var(--amber);
    opacity: 0.7;
    position: absolute;
    bottom: -40px;
    right: -10px;
}

/* Add icon before external links */
a[href^="https"]:not([href*="mydomain.com"])::before {
    content: "‚Üó ";
    font-size: 0.8em;
    color: var(--amber);
}

/* Style first letter of paragraphs */
p::first-letter {
    font-size: 3rem;
    font-weight: bold;
    float: left;
    line-height: 0.8;
    margin-right: 5px;
    color: var(--cyan);
}

/* Style first line of paragraphs */
p::first-line {
    font-weight: bold;
    color: var(--amber);
}

/* Add decorative divider after headings */
h2::after {
    content: "";
    display: block;
    width: 50px;
    height: 2px;
    background-color: var(--amber);
    margin: 10px auto 20px;
}

Key Rules:

  • Use double colon :: for pseudo-elements (single colon : is legacy)
  • Content property is REQUIRED for ::before and ::after
  • Pseudo-elements are not part of DOM ‚Äî cannot be selected by JavaScript

Powerful Use Case: Creating tooltips without JavaScript:


<span class="tooltip" data-tooltip="This is a tooltip">Hover me</span>

    position: relative;
    cursor: pointer;
}

    content: attr(data-tooltip);
    position: absolute;

    transform: translateX(-50%);
    background-color: #000;
    color: white;
    padding: 5px 10px;
    border-radius: 4px;
    font-size: 0.8rem;
    opacity: 0;
    transition: opacity 0.3s;
    white-space: nowrap;
}

.tooltip:hover::after {
    opacity: 1;
}
