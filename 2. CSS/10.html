8.1 Flexbox Fundamentals

Understand the core concepts that make Flexbox work — containers, items, and axes.

Flex container and flex items

Flexbox requires two key elements: a container and its children.


/* Make a container a flex container */
.flex-container {
    display: flex;
}

/* Children become flex items automatically */
.flex-item {
    /* No special properties needed */
}
Item 1
Item 2
Item 3

Key Concepts:

  • Flex Container: Element with display: flex;
  • Flex Items: Direct children of the flex container
  • Implicit Behavior: All direct children become flex items automatically
  • Non-Flex Items: Elements nested deeper than direct children are not affected by flex properties

Important Note: Flexbox only affects direct children — it doesn't cascade down like regular CSS.


<div class="flex-container">
    <div class="flex-item">Item 1</div>
    <div class="flex-item">
        <div class="nested">This is NOT a flex item!</div>
    </div>
    <div class="flex-item">Item 3</div>
</div>

Pro Tip: Always use semantic HTML — don't add unnecessary divs just to create flex containers.

Main axis vs cross axis

Flexbox works in two dimensions: the main axis and the cross axis.

Main Axis (row)
Cross Axis (column)

Main Axis: The primary direction of your layout — determined by flex-direction.

Cross Axis: Perpendicular to the main axis — always runs across your items.

flex-direction Main Axis Cross Axis
row (default) Left to right Top to bottom
row-reverse Right to left Top to bottom
column Top to bottom Left to right
column-reverse Bottom to top Left to right

Memory Trick: Think of the main axis as the "direction you're flowing" and the cross axis as the "direction you're stacking".

Why it matters: Every flex property targets either the main axis or the cross axis:

  • justify-content → main axis
  • align-items → cross axis
  • flex-direction → determines which is which

Wrap Behavior:

8.3 Flex Item Properties

Control individual behavior of flex items within their container.

flex-grow, flex-shrink, flex-basis

The three components that determine how flex items distribute available space.


.item-grow {
    flex-grow: 1; /* Expand to fill available space */
}

.item-shrink {
    flex-shrink: 1; /* Shrink when space is limited */
}

.item-basis {
    flex-basis: 200px; /* Initial size before growing/shrinking */
}

/* Individual values */
.complex-item {
    flex-grow: 2;
    flex-shrink: 1;
    flex-basis: 150px;
}

How they work together:

  1. flex-basis: Sets the initial size (like width/height)
  2. flex-grow: How much to expand beyond basis when space is available
  3. flex-shrink: How much to shrink below basis when space is limited

Visual Formula:


Available Space = Container Width - Total Flex-Basis

Item Size = Flex-Basis + (Available Space × Flex-Grow Ratio)

Example: Three items with flex-grow: 1, 2, 1 in a 800px container with flex-basis: 100px each:

  • Total basis: 300px
  • Available space: 500px
  • Total grow factor: 4 (1+2+1)
  • Item 1: 100 + (500 × 1/4) = 225px
  • Item 2: 100 + (500 × 2/4) = 350px
  • Item 3: 100 + (500 × 1/4) = 225px

Pro Tip: Set flex-basis: 0 when you want items to grow equally regardless of content size.

Key Points:

  • Items are ordered numerically from lowest to highest
  • Default order is 0
  • Negative numbers come before positive numbers
  • Items with same order value maintain source order

Use Cases:

  • Mobile-first design — reorder content for different screen sizes
  • Accessibility — visually reorder without affecting screen reader order
  • Hero sections — move important content to front on desktop

Accessibility Note: Screen readers follow HTML order, not visual order — this is often desirable!


/* Mobile-first approach */
.sidebar {
    order: 2;

.content {
    order: 1;
}

@media (min-width: 768px) {
    .sidebar {
        order: 1;
    }

        order: 2;

8.4 Flexbox Patterns and Use Cases

Apply Flexbox to solve real-world layout challenges with elegant solutions.

Centering content

One of Flexbox's greatest strengths — easily center elements both horizontally and vertically.


/* Perfect centering */
.center-container {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}

/* Horizontal centering only */
.h-center {
    display: flex;
    justify-content: center;
}

/* Vertical centering only */
.v-center {
    display: flex;
    align-items: center;
    height: 100vh;
}
Perfectly Centered Content

Why this works:

  • justify-content: center centers along main axis (horizontal in row)
  • align-items: center centers along cross axis (vertical in row)
  • No need for positioning or transforms!

Real-World Example: Modal dialog centering:


.modal-overlay {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    display: flex;
    justify-content: center;
    align-items: center;
    background-color: rgba(0, 0, 0, 0.5);
}

.modal-content {
    background-color: #0a192f;
    padding: 30px;
    border-radius: 10px;
    max-width: 600px;
    width: 100%;
}

Pro Tip: Combine with max-width and width: 100% for responsive modal dialogs.

Solve the classic problem of columns having different heights.


.equal-height-container {
    display: flex;
}

.equal-height-column {
    flex: 1;
    padding: 20px;
    border: 1px solid rgba(0, 209, 255, 0.2);

How it works:

  • display: flex creates a flex container
  • flex: 1 makes each item grow equally
  • By default, flex items stretch to match the tallest item (align-items: stretch)

Advanced Pattern: Add gaps and borders without breaking layout:


.equal-height-container {
    display: flex;
    gap: 20px;
}

.equal-height-column {
    flex: 1;
    padding: 20px;
    border: 1px solid rgba(0, 209, 255, 0.2);
    border-radius: 8px;
}

Pro Tip: This eliminates the need for JavaScript libraries like MatchHeight!

Modern Navigation Pattern: This approach replaces old float-based navbars and is now industry standard.

Responsive flexbox patterns

Build layouts that adapt seamlessly across all device sizes.


.responsive-grid {
    display: flex;
    flex-wrap: wrap;
    gap: 15px;
}

.responsive-item {
    flex: 1 1 calc(33.333% - 15px);
    min-width: 200px;
}

@media (max-width: 768px) {
    .responsive-item {
        flex: 1 1 calc(50% - 15px);
    }
}

@media (max-width: 480px) {
    .responsive-item {
        flex: 1 1 100%;
    }
}

Understanding the math:

  • calc(33.333% - 15px): Each item takes 1/3 of container minus gap
  • flex: 1 1: Allow growth and shrinkage

Advanced Pattern: Fluid Grid with Minimum Width


/* Modern approach */
.grid {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

.item {
    flex: 1 1 200px; /* Grow, shrink, minimum width */
}

/* Or with percentage */
.item {
    flex: 1 1 33.333%; /* 3 columns */
    min-width: 250px; /* Prevent extreme squeezing */
}

Pro Tip: Use min-width instead of fixed widths for truly responsive designs — it adapts better to content.

Performance Note: Flexbox is highly performant — use it instead of JavaScript-based grid systems!

🌟 Mastery Tip: Flexbox isn't about memorizing properties — it's about understanding axes.
Master the relationship between main axis and cross axis — everything else follows naturally.
Use flex: 1 for equal distribution, gap for spacing, and flex-wrap for responsiveness.
Avoid floats for layout — Flexbox is the modern standard for one-dimensional layouts.
