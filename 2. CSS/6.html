5.1 Font Properties

The foundation of every typographic system — control the very essence of your text.

Font family and font stacks

Font family defines which typeface to use. Always provide fallbacks!


/* Basic font family */
body {
    font-family: Arial, sans-serif;
}

/* Complex font stack */
.heading {
    font-family: "Helvetica Neue", Helvetica, Arial, "Lucida Grande", sans-serif;
}

/* Font stack with quotes for multi-word names */
.paragraph {
    font-family: "Georgia", "Times New Roman", Times, serif;
}

/* System font stack (modern approach) */
.system-text {
    font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif;
}

Font Stack Logic: The browser uses the first available font in the list.

  • First choice: "Helvetica Neue" (macOS/iOS)
  • Fallback 1: "Helvetica" (older macOS)
  • Fallback 2: "Arial" (Windows)
  • Fallback 3: "Lucida Grande" (macOS)
  • Last resort: any sans-serif font on the system

Font Categories:

Category Characteristics Examples Best For
Serif Small lines at ends of strokes Georgia, Times New Roman, Merriweather Long-form reading, print-like feel
Sans-serif No serifs, clean lines Arial, Helvetica, Open Sans, Inter Web interfaces, digital screens
Monospace All characters same width Courier, Consolas, Fira Code Code snippets, technical content
Display Decorative, high contrast Bebas Neue, Playfair Display Headings, logos, emphasis
Handwriting Script-like, cursive Dancing Script, Pacifico Casual, personal contexts

Pro Tip: Always end with a generic family: serif, sans-serif, monospace, cursive, or fantasy.

Font size and scaling

Font size controls how large text appears. Use relative units for scalability.


/* Avoid absolute sizes */
.bad-practice {
    font-size: 16px; /* Too rigid */
}

/* Use relative units */
.good-practice {
    font-size: 1rem; /* Base size = 16px by default */
}

/* Scale based on context */
h1 { font-size: 2.5rem; }  /* 40px */
h2 { font-size: 2rem; }    /* 32px */
h3 { font-size: 1.75rem; } /* 28px */
p { font-size: 1rem; }     /* 16px */
.small { font-size: 0.875rem; } /* 14px */

Rem vs Em:

  • rem: Relative to root element (<html>) — predictable
  • em: Relative to parent — can compound unexpectedly

Recommended Base: Set root font size for accessibility:


html {
    font-size: 100%; /* 16px */
}

/* Allow users to adjust base size */
@media (prefers-reduced-motion: reduce) {
    html {
        font-size: 110%;
    }
}

.tight {
    line-height: 1.2;

    line-height: 2;

5.3 Advanced Typography

Elevate your typography beyond basic properties with modern web technologies.

Web fonts and @font-face

Load custom fonts from external sources.


/* Define custom font */
@font-face {
    font-family: 'MyCustomFont';
    src: url('fonts/mycustomfont.woff2') format('woff2'),
         url('fonts/mycustomfont.woff') format('woff');
    font-weight: 400;
    font-style: normal;
    font-display: swap; /* Critical for performance */
}

/* Use the font */
.body-text {
    font-family: 'MyCustomFont', Arial, sans-serif;
}

/* Multiple weights */
@font-face {
    font-family: 'Inter';
    src: url('fonts/Inter-Regular.woff2') format('woff2');
    font-weight: 400;
    font-style: normal;
    font-display: swap;
}

@font-face {
    font-family: 'Inter';
    src: url('fonts/Inter-Bold.woff2') format('woff2');
    font-weight: 700;
    font-style: normal;
    font-display: swap;
}

@font-face {
    font-family: 'Inter';
    src: url('fonts/Inter-Italic.woff2') format('woff2');
    font-weight: 400;
    font-style: italic;
    font-display: swap;
}

Font Display Strategies:

Value Behavior Use Case
auto Browser default Not recommended
block Hide text briefly until font loads Brand identity critical
swap Show fallback immediately, swap when font loads ✅ Recommended for most sites
fallback Short hide period, then swap Good compromise
optional Only load if fast enough Low bandwidth connections

Pro Tip: Always use font-display: swap; — it's the best balance of performance and user experience.

Font loading strategies

Optimize how fonts are delivered to prevent layout shifts and invisible text.


/* Preload critical fonts */
<link rel="preload" as="font" href="fonts/inter.woff2" type="font/woff2" crossorigin>

/* Preconnect to font hosting domains */
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

/* Load fonts asynchronously */
<script>
// Modern approach using Font Loading API
const font = new FontFace('MyFont', 'url(fonts/myfont.woff2)');
font.load().then(function(loadedFont) {
    document.fonts.add(loadedFont);
    document.documentElement.classList.add('fonts-loaded');
});
</script>

/* CSS classes for state management */
.fonts-loaded .heading {
    font-family: 'MyFont', sans-serif;
}

.fonts-loading .heading {
    font-family: Arial, sans-serif;
}

When fonts are loading:

  • FOIT (Flash of Invisible Text): Text remains invisible until font loads
  • FAIT (Flash of Unstyled Text): Text renders with fallback font, then swaps
  • FOFT (Flash of Faux Text): Text renders with faux bold/italic

Recommendation: Use font-display: swap; to avoid FOIT entirely.

Performance Checklist:

  1. Preload critical fonts
  2. Use WOFF2 format (smallest file size)
  3. Limit number of font weights
  4. Use system fonts as fallback
  5. Implement font-display: swap;
  6. Consider variable fonts to reduce HTTP requests
Variable fonts

Single font files that contain multiple styles — revolutionary for performance.


/* Define variable font */
@font-face {
    font-family: 'Inter Variable';
    src: url('fonts/Inter-Variable.woff2') format('woff2-variations');
    font-weight: 100 900;
    font-stretch: 75% 100%;
    font-style: normal;
    font-display: swap;
}

/* Use variable font properties */
.title {
    font-family: 'Inter Variable', sans-serif;
    font-weight: 600;
    font-stretch: 85%;
}

.subtitle {
    font-family: 'Inter Variable', sans-serif;
    font-weight: 400;
    font-stretch: 100%;
}

/* Control with CSS variables */
:root {
    --font-weight: 500;
    --font-stretch: 90%;
}

.dynamic-text {
    font-family: 'Inter Variable', sans-serif;
    font-weight: var(--font-weight);
    font-stretch: var(--font-stretch);
}

/* Dynamic adjustment with JavaScript */
document.documentElement.style.setProperty('--font-weight', 700);

Advantages:

  • One file instead of multiple files (regular, bold, italic, etc.)
  • Smaller total file size
  • Infinite style variations within range
  • Smooth transitions between weights/styles

Supported Axes:

  • wght: Weight (100-900)
  • wdth: Width (50-200%)
  • slnt: Slant (-20° to 20°)
  • ital: Italic (0-1)
  • opsz: Optical size (9-120pt)

Browser Support: Excellent in modern browsers (Chrome 90+, Firefox 84+, Safari 14+).

5.4 Responsive Typography

Make your text adapt gracefully across all screen sizes — the key to universal readability.

Fluid typography with clamp()

The modern solution for responsive font sizing without media queries.


/* Perfect fluid typography */
h1 {
    font-size: clamp(1.5rem, 5vw, 3rem);
}

h2 {
    font-size: clamp(1.25rem, 4vw, 2.5rem);
}

h3 {
    font-size: clamp(1.125rem, 3.5vw, 2rem);
}

p {
    font-size: clamp(1rem, 3vw, 1.5rem);
}

/* How it works: min, preferred, max */
.clamp-example {
    font-size: clamp(
        16px,      /* Minimum size on smallest screens */
        4vw,       /* Preferred size: scales with viewport */
        48px       /* Maximum size on largest screens */
    );
}

Fluid Heading

This paragraph scales smoothly from 16px on mobile to 24px on desktop.

Formula for Readability:

  • Minimum: 16px (minimum readable size)
  • Preferred: 3–5vw (scales with screen width)
  • Maximum: 2.5–3rem (prevents excessive size on large displays)

Pro Tip: Use clamp() instead of multiple media queries — it's cleaner and more maintainable.

Viewport-based font sizing

Scale fonts proportionally to viewport dimensions.


/* Simple viewport scaling */
.hero-heading {
    font-size: 8vw; /* 8% of viewport width */
}

/* More controlled scaling */
.responsive-title {
    font-size: calc(1rem + 2vw);
}

/* Combined with minimum */
.safe-scaling {
    font-size: max(1rem, 1.5vw);
}

/* Responsive subtitle */
.subheading {
    font-size: 2.5vw;
    font-size: clamp(1.25rem, 2.5vw, 2rem);
}

/* Paragraph scaling */
.article-body {
    font-size: clamp(1rem, 1.5vw + 0.5rem, 1.25rem);
}

Why use vw? It creates natural scaling proportional to screen size — perfect for hero sections and titles.

Caution: Pure vw can make text too small on mobile or too large on desktop — always combine with clamp() or min()/max().

Pure 1.5vw (too small on mobile)

Safe: max(1rem, 1.5vw)

Ideal: clamp(1rem, 1.5vw, 1.5rem)

Font size accessibility considerations

Design for everyone — including users with visual impairments.

WCAG Guidelines:

  • Minimum text size: 16px for body text
  • Text resizing: Must be resizable up to 200% without loss of content
  • Contrast ratio: At least 4.5:1 for normal text, 3:1 for large text
  • Don't disable zoom: Never use user-scalable=no in viewport meta

/* Accessibility-friendly typography */
body {
    font-size: 1rem; /* 16px */
    line-height: 1.6;
}

h1 { font-size: 2.5rem; }   /* 40px */
h2 { font-size: 2rem; }     /* 32px */
h3 { font-size: 1.75rem; }  /* 28px */
h4 { font-size: 1.5rem; }   /* 24px */
h5 { font-size: 1.25rem; }  /* 20px */
h6 { font-size: 1.125rem; } /* 18px */

/* Ensure text can be zoomed */
html {
    font-size: 100%;
}

/* Don't override user preferences */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
    }
}

@media (prefers-contrast: high) {
    body {
        color: #000;
        background-color: #fff;
    }
}

Testing Accessibility:

  1. Zoom browser to 200%
  2. Check if text wraps properly
  3. Verify no content is cut off
  4. Test with grayscale filters
  5. Use browser dev tools to simulate vision deficiencies
Typography scales and modular design

Create harmonious type hierarchies using mathematical ratios.


/* Modular scale: Golden Ratio (1.618) */
:root {
    --font-scale-ratio: 1.618;
    
    --font-size-xs: calc(1rem / var(--font-scale-ratio) / var(--font-scale-ratio));
    --font-size-sm: calc(1rem / var(--font-scale-ratio));
    --font-size-base: 1rem;
    --font-size-md: calc(1rem * var(--font-scale-ratio));
    --font-size-lg: calc(1rem * var(--font-scale-ratio) * var(--font-scale-ratio));
    --font-size-xl: calc(1rem * var(--font-scale-ratio) * var(--font-scale-ratio) * var(--font-scale-ratio));
    --font-size-xxl: calc(1rem * var(--font-scale-ratio) * var(--font-scale-ratio) * var(--font-scale-ratio) * var(--font-scale-ratio));
}

/* Apply scale */
h1 { font-size: var(--font-size-xxl); }
