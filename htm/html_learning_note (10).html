Module 5: Links and Anchors

Links are the backbone of the web. This module deeply explores anchor elements, linking strategies, accessibility, and user experience considerations.

5.1 Basic Linking

Executive Summary

The <a> element creates hyperlinks. Links can point to external sites, internal pages, sections within a page, or actions like email or phone dialing.

Anchor Elements (<a>)

Basic syntax

<a href="https://example.com">Visit Example</a>

Every anchor must include the href attribute to define the destination.

Absolute vs Relative URLs

Absolute URL: Full path including protocol.

<a href="https://www.wikipedia.org">Wikipedia</a>

Relative URL: Referencing paths within the same website.

<a href="/about.html">About Us</a>
<a href="contact.html">Contact</a>
Internal Page Linking

Use #id to scroll to a specific section.

<a href="#section2">Go to Section 2</a>
...
<h2 id="section2">Section 2</h2>
Email & Telephone Links

Email link:

<a href="mailto:info@example.com">Send Email</a>

Telephone link:

<a href="tel:+11234567890">Call Us</a>

5.3 Link States & User Experience

Executive Summary

CSS link states and focus management help create accessible and intuitive navigation.

Visited, Hover, Active States

CSS offers link-state pseudo-classes:

a:link { color: cyan; }
a:visited { color: purple; }
a:hover { color: yellow; }
a:active { color: red; }
Keyboard Navigation & Focus Management

Accessible websites must support navigation via Tab keys.

a:focus {
  outline: 2px solid #4dd0e1;
}
