1.3 Basic HTML Syntax & Structure

In this note you will build a deep mental model of how an HTML document is structured: document skeleton, tags vs elements vs attributes, proper opening and closing, self-closing tags, nesting hierarchy, and how comments work. Designed for college students up to master-level learners.
Level: Beginner syntax → strong professional discipline Topics: document structure, tags, attributes, nesting, comments
0 · Big Picture

Why basic syntax & structure matters

“Small” syntax mistakes often break whole pages or make bugs invisible.
Executive summary

HTML syntax is like grammar in a language: you can speak with broken grammar, but serious writing requires correctness. Browsers are forgiving, but as a developer you must be strict, because:

  • Broken nesting can ruin layouts in unpredictable ways.
  • Missing closing tags can merge sections that should be separate.
  • Wrong attribute usage can break links, images, or accessibility.
  • Messy structure makes CSS & JavaScript harder to reason about.

Mastering basic HTML syntax once gives you a foundation that holds for the rest of your web-development journey.

1 · Document Skeleton

HTML document structure

The outermost structure every valid HTML page should respect.
Executive summary

Every HTML document follows the same high-level structure:

  • A doctype declaration.
  • A root <html> element.
  • A <head> section for metadata.
  • A <body> section for visible content.

When the browser sees this skeleton, it knows how to interpret and render the rest of your code.

Minimal HTML5 skeleton (syntax and explanation)

Standard HTML5 document structure

<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My HTML Page</title>
  </head>

  <body>
    <h1>Hello, world!</h1>
    <p>This is my first properly structured page.</p>
  </body>

</html>

Line by line:

  • <!DOCTYPE html> tells the browser that this document uses HTML5.
  • <html lang="en"> is the root element; the lang attribute specifies the language for accessibility and SEO.
  • <head> contains metadata: encoding, viewport, title, styles, etc.
  • <body> contains all visible content: headings, paragraphs, images, buttons, forms, etc.
Real-world notes & common mistakes
Typical beginner mistakes: forgetting <!DOCTYPE html>, putting visible text inside <head>, or creating multiple <html>/<body> tags.
2 · Building Blocks

Tags, elements, and attributes

The vocabulary and grammar of HTML.
Executive summary: difference between tag, element & attribute
  • Tag – the actual text between angle brackets, e.g. <p> or </p>.
  • Element – the complete structure: opening tag + content + closing tag.
  • Attribute – extra information placed inside the opening tag, in name="value" form.

Visual example

<p class="intro">Hello, Irfan!</p>
^ ^^^^^^^^^^^^^^^        ^^^^^^^^
| |                      |
| +-- opening tag with an attribute
+---- element name ("p")

Whole thing = one <p> element.
Deep explanation with more examples

Consider this small snippet: 

<a href="https://example.com" target="_blank">
  Visit Example
</a>
  • <a> = opening tag.
  • </a> = closing tag.
  • Together with the text Visit Example, they form an <a> element.
  • href and target are attributes. Their values are given in quotes.

Attribute syntax pattern

<tag-name attribute1="value1" attribute2="value2">
  content
</tag-name>
Real use cases: why attributes matter
  • id, class – used heavily by CSS & JS.
  • src – tells the browser where to load an image or script from.
  • alt – provides alternative text for images (accessibility & SEO).
  • title – small tooltip text on hover.
  • data-* – custom attributes for JavaScript logic.

In modern development, writing clean, meaningful attributes is a big part of writing clean HTML.

3 · Tag Pairs

Opening and closing tags

How HTML wraps content and defines boundaries.
Executive summary

Most HTML elements use a pair of tags: an opening tag and a closing tag. The opening tag looks like <tagname>, and the closing tag looks like </tagname>. All content for that element must sit between them.

Syntax & examples

Basic pattern

<tagname>content goes here</tagname>

Examples

<p>This is a paragraph.</p>

<strong>Important text</strong>

<h1>Main heading</h1>

The closing tag must match exactly: same name, same spelling, same case (browsers are case-insensitive for HTML, but consistency is a good habit).

Common mistakes & how browsers “fix” them

Missing closing tag

<p>First paragraph
<p>Second paragraph</p>

Browsers often auto-close the first paragraph when a second <p> appears, but this can cause unexpected layout, especially with nested elements.

Rule of thumb: Always type both opening and closing tags explicitly; use your editor’s auto-completion to avoid mistakes.
4 · Void Elements

Self-closing tags

Elements that do not wrap content.
Executive summary

Some HTML elements are void elements: they do not have any inner content, so they do not need a closing tag. Examples: <img>, <br>, <hr>, <meta>, <link>.

In HTML5, you typically write them as <br> or <img src="..." alt="..."> (without the trailing /).

Syntax patterns (HTML vs XHTML style)
HTML5 style (recommended)
<br>
<hr>
<img src="photo.jpg" alt="Profile photo">
XHTML style (still seen in old code)
<br />
<hr />
<img src="photo.jpg" alt="Profile photo" />

Browsers accept both syntaxes, but in pure HTML5, the space and slash before > are optional and not required.

Real examples & practical notes

Line breaks & horizontal rules

<p>First line<br>Second line</p>

<h2>Section 1</h2>
<p>Content...</p>
<hr>
<h2>Section 2</h2>

Because these elements don’t have children, you never write </br> or </img>. Doing so is invalid HTML.

5 · Tree Structure

Nesting and hierarchy

How elements contain other elements to form a DOM tree.
Executive summary

HTML is inherently hierarchical. Elements can contain other elements, forming a nested tree. This nested structure:

  • Defines which content belongs together.
  • Affects how CSS selectors work (parent/child relationships).
  • Impacts how JavaScript traverses the DOM.
Correct vs incorrect nesting

Correct nesting

<p>
  This is a <strong>very important</strong> sentence.
</p>

Incorrect nesting

<p>
  This is a <strong>very important</p> sentence.</strong>

In the incorrect example, the <p> tag is closed inside the <strong> tag, so the tree becomes inconsistent. Browsers try to auto-correct, but the result may not match what you expect.

Visualizing the DOM tree

HTML snippet

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Conceptual tree

UL
├── LI ("HTML")
├── LI ("CSS")
└── LI ("JavaScript")

Every time you nest tags inside one another, you are building this tree. Thinking in tree-structure terms is crucial for mastering DOM, CSS, and JS.

6 · Comments

Comments in HTML

Explaining your code to your future self & your team.
Executive summary

Comments in HTML are pieces of text ignored by the browser. They help you:

  • Explain sections of your code.
  • Temporarily disable elements during debugging.
  • Mark TODOs or important notes.

Syntax: <!-- comment text here -->

Comment syntax and examples

Basic comment

<!-- Main navigation bar starts here -->
<nav>...</nav>

Commenting out a block temporarily

<!--
<section id="old-hero">
  <h1>Old Landing Section</h1>
</section>
-->

Everything between <!-- and --> is ignored by the browser and not displayed on the page.

Best practices & warnings
  • Do not put sensitive information (passwords, keys) in comments: they are still visible in page source.
  • Use comments to clarify why something is done, not just what it does.
  • Keep comments short and meaningful; avoid cluttering the HTML.
Pro-tip: In collaborative projects, comments help align the team’s understanding. But once code is stable and obvious, you can clean up old comments.
