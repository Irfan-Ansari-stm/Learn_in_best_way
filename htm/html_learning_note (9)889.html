Module 4: Lists and Navigation

Lists are not just bullets and numbers; they represent collections, sequences, and term–definition pairs. Navigation structures transform these lists into paths through your site’s information graph. This module explores unordered, ordered, and description lists, and shows how they power menus, breadcrumbs, and pagination in real-world applications.
Level: Conceptual basics → information architecture Topics: <ul>, <ol>, <dl>, nav bars, breadcrumbs, pagination
0 · Big Picture

Lists as data structures for content & navigation

From bullet points to menus and breadcrumbs: same HTML primitives.
Executive summary

In HTML, lists provide a structured way to represent collections:

  • Unordered lists <ul> – items without inherent order (sets).
  • Ordered lists <ol> – items with a meaningful sequence.
  • Description lists <dl> – term–description pairs (like dictionaries).

Navigation UI (menus, breadcrumbs, pagination) is usually built on top of these list elements, because they are:

  • Accessible (screen readers understand the “list of items” semantics).
  • Easy to style with CSS.
  • Semantically correct from an information-architecture perspective.
4.1 · Unordered Lists

Basic & nested unordered lists (<ul>, <li>)

Representing collections where order does not matter.
Executive summary

Unordered lists use <ul> as a container and <li> (list item) for each item. They are ideal when the sequence of items is not semantically important: features, shopping list, bullet points, navigation menus, etc.

Basic <ul> and <li> structure

Syntax pattern

<ul>
  <li>First item</li>
  <li>Second item</li>
  <li>Third item</li>
</ul>

Example: technologies list

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Browsers render bullets by default, but the underlying semantic is “a list of related items”.

Nested lists

A list item can contain another list to represent hierarchy or grouping.

Example: nested unordered lists

<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Node.js</li>
      <li>Python</li>
      <li>Java</li>
    </ul>
  </li>
</ul>
Best practice: keep nested levels reasonable (usually up to 2–3) for readability and accessibility.
Styling list markers

CSS provides control over marker style using list-style-type, list-style-position, and custom bullets.

Common marker styles

ul {
  list-style-type: disc;      /* default solid bullet */
}

ul.square {
  list-style-type: square;    /* ■ */
}

ul.circle {
  list-style-type: circle;    /* ○ */
}

Custom marker with background icon

ul.custom-bullets {
  list-style: none;           /* remove default bullet */
  padding-left: 1.2rem;
}

ul.custom-bullets li {
  position: relative;
  padding-left: 1.1rem;
}

ul.custom-bullets li::before {
  content: "◆";              /* custom symbol */
  position: absolute;
  left: 0;
  top: 0;
  color: #4dd0e1;
}

In advanced UI, lists in navigation bars often have their bullets removed entirely and are styled as horizontal menus.

4.2 · Ordered Lists

Numbered lists (<ol>), start & type

Representing sequences, steps, or rankings.
Executive summary

Ordered lists <ol> are used when order matters: steps in a process, ranking, instructions, timelines, etc. Items are still <li>, but markers are numbers, letters, or Roman numerals.

Basic <ol> numbered lists

Syntax

<ol>
  <li>Gather requirements</li>
  <li>Design the UI</li>
  <li>Implement the features</li>
  <li>Test and deploy</li>
</ol>

Browser default markers are 1, 2, 3, ....

Custom numbering with the start attribute

You can start numbering from a custom value using the start attribute.

Example: continuing a list from 5

<ol start="5">
  <li>Fifth step</li>
  <li>Sixth step</li>
</ol>

Useful when you split a long list into multiple parts on the page but want consistent numbering.

Different numbering types (roman, alphabetical)

The type attribute controls marker style:

  • type="1" – 1, 2, 3 (default)
  • type="a" – a, b, c
  • type="A" – A, B, C
  • type="i" – i, ii, iii (roman)
  • type="I" – I, II, III

Example: mixed types

<ol type="A">
  <li>Introduction</li>
  <li>Literature Review</li>
  <li>Methodology</li>
</ol>

<ol type="i">
  <li>Sub-point one</li>
  <li>Sub-point two</li>
</ol>
4.3 · Description Lists

Definition lists (<dl>, <dt>, <dd>)

Pairing terms with their descriptions or definitions.
Executive summary

Description lists represent term–description pairs, like a dictionary or glossary:

  • <dl> – container (description list).
  • <dt> – term name.
  • <dd> – definition/description for the term.

They are semantically richer than trying to fake definitions with just <p> and <strong>.

Syntax & example

Basic description list

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language, used to structure content on the web.</dd>

  <dt>CSS</dt>
  <dd>Cascading Style Sheets, used to style and layout web pages.</dd>

  <dt>JavaScript</dt>
  <dd>Programming language that adds interactivity to web pages.</dd>
</dl>
Use cases & semantic meaning
  • Glossaries of technical terms.
  • FAQ-style question–answer pairs (Q as <dt>, A as <dd>).
  • Metadata lists: property name + value (e.g., product specifications).
Semantics: screen readers announce description lists differently from normal lists, making them more accessible for definitions and Q&A content.
4.4 · Navigation

Navigation bars, breadcrumbs, and pagination

Using lists to model paths and menus in user interfaces.
Executive summary

Most navigation patterns are built with lists because they represent:

  • A set of navigation options (menu).
  • A path of pages (breadcrumb).
  • A sequence of pages (pagination).

Underneath the CSS, you typically use <ul> and <li> with links <a>.

Navigation bar using lists

Simple horizontal nav bar

<nav>
  <ul class="main-nav">
    <li><a href="/">Home</a></li>
    <li><a href="/courses">Courses</a></li>
    <li><a href="/blog">Blog</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

CSS idea (conceptual)

.main-nav {
  list-style: none;
  display: flex;
  gap: 1.5rem;
}

.main-nav a {
  text-decoration: none;
  color: #e0e6f0;
}

The list structure remains semantically a list of navigation options; CSS just changes its presentation.

Breadcrumb navigation

Breadcrumbs represent the path of pages from root to current page: Home > Category > Subcategory > Current page.

Example breadcrumb list

<nav aria-label="Breadcrumb">
  <ol class="breadcrumb">
    <li><a href="/">Home</a></li>
    <li><a href="/html">HTML</a></li>
    <li><a href="/html/lists">Lists</a></li>
    <li aria-current="page">Navigation Patterns</li>
  </ol>
</nav>
Accessibility: the aria-label="Breadcrumb" and aria-current="page" attributes help screen readers understand the structure and current location.
Pagination elements

Pagination is another ordered sequence, commonly represented as an ordered or unordered list:

Example pagination component

<nav aria-label="Page navigation">
  <ul class="pagination">
    <li><a href="?page=1">1</a></li>
    <li><a href="?page=2">2</a></li>
    <li><a href="?page=3">3</a></li>
    <li><a href="?page=4">Next »</a></li>
  </ul>
</nav>

CSS (or Bootstrap classes like .pagination) control layout and active states, but the underlying list remains accessible and meaningful.
