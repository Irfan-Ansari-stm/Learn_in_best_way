12.1 CSS Modules

üìå Summary: Scoped CSS Without JavaScript

CSS Modules are a build-time system that automatically scopes your CSS classes to avoid naming collisions. You write regular CSS, but it gets transformed into unique class names at compile time.

üîπ How It Works

  1. You create a file like Button.module.css
  2. Import it as a JavaScript object: import styles from './Button.module.css'
  3. Use classes as object properties: <button className={styles.button}>
  4. Webpack/Vite transforms .button into something like _Button_button_12abc

‚ö° Syntax Example


/* Button.module.css */
.button {
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;
}

.primary {
  background-color: #00f5ff;
  color: #000;
}

.secondary {
  background-color: #3a5a7f;
  color: #fff;
}

.disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
          

// Button.jsx
import styles from './Button.module.css';

function Button({ children, variant = 'primary', disabled = false }) {
  return (
    <button 
      className={`${styles.button} ${styles[variant]} ${disabled ? styles.disabled : ''}`}
      disabled={disabled}
    >
      {children}
    </button>
  );
}

export default Button;

üí° Why This Is Brilliant

  • No naming conflicts: Two components can have a .button class without collision
  • Zero runtime overhead: Compiled away ‚Äî pure CSS in production
  • Familiar syntax: Write standard CSS you already know
  • Tooling support: Works with Webpack, Vite, Create React App
üß† Mental Model: Think of CSS Modules as a "CSS compiler" that renames your classes to be globally unique while keeping your code clean.
üß™ Deep Dive: How CSS Modules Handle Class Composition

CSS Modules supports composition ‚Äî combining multiple classes into one.

‚ö° Composition Example


/* Button.module.css */
.base {
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;
}

.primary {
  composes: base; /* ‚Üê Inherit all styles from .base */
  background-color: #00f5ff;
  color: #000;
}

.large {
  composes: base; /* ‚Üê Reusable base styles */
  font-size: 18px;
  padding: 16px 32px;
}

.success {
  composes: primary; /* ‚Üê Inherits from primary, then overrides */
  background-color: #00cc88;
}
          

// Component usage
<button className={styles.primary}>Primary Button</button>
<button className={styles.large}>Large Button</button>
<button className={styles.success}>Success Button</button>

üîç What Gets Compiled

After compilation, the HTML becomes:


<button class="_Button_base_1a2b3 _Button_primary_4c5d6">Primary Button</button>
<button class="_Button_base_1a2b3 _Button_large_7e8f9">Large Button</button>
<button class="_Button_base_1a2b3 _Button_primary_4c5d6 _Button_success_0g1h2">Success Button</button>

‚ö†Ô∏è Common Pitfalls

  • Don't use :global() unless you need global styles ‚Äî it defeats the purpose
  • Class names must be valid CSS identifiers (no hyphens in variable names)
  • Can't use dynamic class names directly: {styles[myVar]} works, but {styles.${myVar}} doesn't

12.2 Styled-components

üìå Summary: Write CSS as JavaScript Components

Styled-components lets you write actual CSS inside your JavaScript using tagged template literals. Each component gets its own unique class name automatically.

üîπ Core Philosophy

"Component-scoped styles are better than global styles." ‚Äî styled-components makes this philosophy effortless.

‚ö° Basic Syntax


import styled from 'styled-components';

// Create a component with embedded CSS
const Button = styled.button`
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;

  background-color: #00f5ff;
  color: #000;

  &:hover {
    background-color: #00c0e6;
    transform: translateY(-2px);
    box-shadow: 0 4px 8px rgba(0, 245, 255, 0.3);
  }

  &:disabled {
    opacity: 0.5;
    cursor: not-allowed;
  }
`;

// Use it like any other component
function App() {
  return (
    <div>
      <Button>Click Me</Button>
      <Button disabled>Disabled</Button>
    </div>
  );
}

üí° Why This Is Powerful

  • Automatic scoping: No class name conflicts
  • Dynamic styling: Pass props to change styles
  • Server-side rendering: Built-in support
  • Theming: Built-in context-based theming

üî• Dynamic Styling with Props


const Button = styled.button`
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;

  background-color: ${props => props.primary ? '#00f5ff' : '#3a5a7f'};
  color: ${props => props.primary ? '#000' : '#fff'};

  &:hover {
    background-color: ${props => props.primary ? '#00c0e6' : '#2d4a6e'};
  }

  // Size variants
  ${props => props.large && `
    font-size: 18px;
    padding: 16px 32px;
  `}
`;

// Usage
<Button primary>Primary Button</Button>
<Button large>Large Button</Button>
<Button primary large>Big Primary Button</Button>

üé® Theming with styled-components


import { ThemeProvider } from 'styled-components';

const theme = {
  colors: {
    primary: '#00f5ff',
    secondary: '#3a5a7f',
    success: '#00cc88'
  },
  spacing: {
    small: '8px',
    medium: '16px',
    large: '24px'
  }
};

const Button = styled.button`
  padding: ${props => props.theme.spacing.medium};
  background-color: ${props => props.theme.colors.primary};
  color: #000;
  border-radius: 6px;
  border: none;
  cursor: pointer;
`;

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button>Styled Button</Button>
    </ThemeProvider>
  );
}
üß† Mental Model: styled-components turns CSS into a React component. The component *is* the style.

styled-components allows you to extend existing styled components ‚Äî no CSS inheritance needed!

‚ö° Component Extension

üî• Nested Components

  border: 'none',

    backgroundColor: '#00c0e6'

12.3 Emotion

üìå Summary: The High-Performance CSS-in-JS Library

Emotion is a performant CSS-in-JS library with excellent server-side rendering and zero-runtime overhead options. It's faster than styled-components and more flexible than CSS Modules.

üîπ Two Ways to Use Emotion

  1. css prop: Inline styles via JSX attribute
  2. styled: Like styled-components, but with better performance

‚ö° Using the css prop (Recommended)


import { css } from '@emotion/react';

const buttonStyle = css`
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;
  background-color: #00f5ff;
  color: #000;

  &:hover {
    background-color: #00c0e6;
    transform: translateY(-2px);
  }
`;

function Button({ children }) {
  return (
    <button css={buttonStyle}>
      {children}
    </button>
  );
}

‚ö° Dynamic css prop with props


import { css } from '@emotion/react';

function Button({ variant = 'primary', size = 'medium', children }) {
  const buttonStyles = css`
    padding: ${size === 'large' ? '16px 32px' : '12px 24px'};
    border: none;
    border-radius: 6px;
    font-size: 16px;
    cursor: pointer;
    
    background-color: ${variant === 'primary' ? '#00f5ff' : '#3a5a7f'};
    color: ${variant === 'primary' ? '#000' : '#fff'};

    &:hover {
      background-color: ${variant === 'primary' ? '#00c0e6' : '#2d4a6e'};
    }
  `;

  return (
    <button css={buttonStyles}>
      {children}
    </button>
  );
}

‚ö° Using styled API (like styled-components)


import { styled } from '@emotion/react';

const Button = styled('button')`
  padding: 12px 24px;
  border: none;
  border-radius: 6px;
  font-size: 16px;
  cursor: pointer;
  background-color: #00f5ff;
  color: #000;

  &:hover {
    background-color: #00c0e6;
  }
`;

// Or with props
const Button = styled('button')(({ primary, large }) => ({
  padding: large ? '16px 32px' : '12px 24px',
  backgroundColor: primary ? '#00f5ff' : '#3a5a7f',
  color: primary ? '#000' : '#fff',
  border: 'none',
  borderRadius: '6px',
  fontSize: '16px',
  cursor: 'pointer',
  '&:hover': {
    backgroundColor: primary ? '#00c0e6' : '#2d4a6e'
  }

‚öôÔ∏è Deep Dive: Emotion's Zero-Runtime Mode

Emotion has a special mode called "zero-runtime" that compiles CSS at build time ‚Äî eliminating the need for JavaScript to apply styles at runtime!

‚ö° Setup in vite.config.js


// vite.config.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { vitePluginAsmCss } from '@emotion/babel-plugin';

export default defineConfig({
  plugins: [
    react({
      babel: {
        plugins: [
          ['@emotion/babel-plugin', { sourceMap: true, autoLabel: 'dev-only' }]
        ]
      }
    })
  ]
});

What Happens?

  • During build: css={...} gets converted to static class names
  • Production bundle: No emotion runtime included!
  • Result: Performance equals CSS Modules, with the flexibility of CSS-in-JS
üåü Pro Tip: Use Emotion with zero-runtime mode for maximum performance. It's the best of both worlds: CSS Modules' speed + CSS-in-JS' flexibility.

12.4 CSS-in-JS Patterns

üìå Summary: Design Patterns for Professional CSS-in-JS

Using CSS-in-JS isn't just about syntax ‚Äî it's about architecture. Here are battle-tested patterns used by top teams.

üîπ Pattern 1: Atomic Styles with Object Syntax

Break UI into tiny, reusable style objects ‚Äî perfect for design systems.


// styles/atomic.js
export const spacing = {
  xs: '4px',
  sm: '8px',
  md: '16px',
  lg: '24px',
  xl: '32px'
};

export const radius = {
  sm: '4px',
  md: '8px',
  lg: '12px'
};

export const colors = {
  primary: '#00f5ff',
  secondary: '#3a5a7f',
  success: '#00cc88',
  danger: '#ff4757',
  text: '#e6f1ff',
  bg: '#0a192f'
};

export const typography = {
  h1: {
    fontSize: '2rem',
    fontWeight: '700',
    lineHeight: '1.2'
  },
  button: {
    fontSize: '1rem',
    fontWeight: '600',
    textTransform: 'uppercase',
    letterSpacing: '0.05em'
  }
};

// Usage
import { colors, spacing, typography } from './styles/atomic';

const Button = styled.button`
  padding: ${spacing.md};
  background-color: ${colors.primary};
  color: ${colors.text};
  border: none;
  border-radius: ${radius.md};
  ${typography.button}
`;

üîπ Pattern 2: Compound Components

Create components that work together ‚Äî like a Card with Header and Body.


import { styled } from '@emotion/react';

  padding: 16px;
  color: #ccc;

üîπ Pattern 3: Hook-Based Style Generation

Generate styles dynamically based on component state or props using custom hooks.


import { css } from '@emotion/react';

// Custom hook
function useButtonStyle(variant, size) {
  return css`
    padding: ${size === 'large' ? '16px 32px' : '12px 24px'};
    background-color: ${variant === 'primary' ? '#00f5ff' : '#3a5a7f'};
    color: ${variant === 'primary' ? '#000' : '#fff'};
    border: none;
    border-radius: 6px;
    font-size: 16px;
    cursor: pointer;
    transition: all 0.2s ease;

    &:hover {
      background-color: ${variant === 'primary' ? '#00c0e6' : '#2d4a6e'};
      transform: translateY(-1px);
    }

    &:active {
      transform: translateY(0);
    }
  `;
}

// Component
function Button({ variant = 'primary', size = 'medium', children }) {
  const buttonStyle = useButtonStyle(variant, size);

  return (
    <button css={buttonStyle}>
      {children}
    </button>
  );
}

üîπ Pattern 4: Global Styles with Theme Provider

Apply consistent base styles across your entire app.


import { Global, ThemeProvider } from '@emotion/react';

const theme = {
  colors: {
    primary: '#00f5ff',
    secondary: '#3a5a7f',
    text: '#e6f1ff',
    bg: '#0a192f'
  },
  fonts: {
    main: "'Roboto', sans-serif",
    mono: "'Roboto Mono', monospace"
  }
};

const GlobalStyles = () => (
  <Global
    styles={`
      * {
        box-sizing: border-box;
      }
      
      body {
        margin: 0;
        padding: 0;
        font-family: ${theme.fonts.main};
        background-color: ${theme.colors.bg};
        color: ${theme.colors.text};
      }
      
      h1, h2, h3 {
        color: ${theme.colors.primary};
      }
      
      a {
        color: ${theme.colors.primary};
        text-decoration: none;
      }
      
      a:hover {
        text-decoration: underline;
      }
    `}
  />
);

function App() {
  return (
    <ThemeProvider theme={theme}>
      <GlobalStyles />
      <!-- Your app content -->
    </ThemeProvider>
  );
}

üéØ Final Wisdom: Choose Your Weapon Wisely

  • For beginners: Start with CSS Modules ‚Äî simple, predictable, no magic
  • For teams: Use Emotion with zero-runtime ‚Äî best balance of power and performance
  • For rapid prototyping: Styled-components ‚Äî great DX, slightly heavier
  • Avoid: Mixing global CSS with CSS-in-JS unless absolutely necessary
