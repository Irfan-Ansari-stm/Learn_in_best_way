1.1 What is React and Why Use It?

Executive Summary | Foundational Concept

üîç Executive Summary

React is a JavaScript library developed by Facebook (now Meta) for building user interfaces, particularly single-page applications where data changes over time. Unlike full frameworks, React focuses solely on the view layer ‚Äî making it lightweight and highly composable.

Why React?

  • Component-Based Architecture: Break UI into reusable pieces
  • Virtual DOM: Efficient updates without reloading the entire page
  • Unidirectional Data Flow: Predictable state management
  • Huge Ecosystem: Thousands of libraries and tools
  • Industry Standard: Used by Facebook, Instagram, Airbnb, Netflix
üîç Forensic Detail

React was created in 2011 by Jordan Walke, a software engineer at Facebook, to solve the problem of efficiently updating dynamic content on Facebook's news feed. Before React, developers used direct DOM manipulation with jQuery ‚Äî which became unmanageable as applications grew.

React introduced the concept of a Virtual DOM ‚Äî a lightweight copy of the real DOM. When state changes, React:

  1. Creates a new Virtual DOM tree
  2. Compares it with the previous one (diffing algorithm)
  3. Calculates minimal set of changes needed
  4. Applies only those changes to the real DOM

This process is called reconciliation. The result? Dramatically improved performance even with complex UIs.

üåê Origin Story

React wasn't born in a lab ‚Äî it was forged in the fires of Facebook's scaling challenges. In 2011, Facebook's news feed was becoming increasingly dynamic. Every new post, comment, or like required manual DOM manipulation ‚Äî leading to bugs and poor performance.

Jordan Walke took inspiration from XHP (Facebook's PHP framework) and created JSX ‚Äî a syntax extension that lets you write HTML-like code in JavaScript. He called it "React" because components react to state changes.

React was open-sourced in May 2013 and quickly gained traction. By 2015, it had surpassed Angular in GitHub stars. Today, React powers over 40% of top websites according to Wappalyzer.

Fun Fact: The first version of React didn't have hooks! Hooks were introduced in React 16.8 (2019) to solve the complexity of class components.

üîç Deep Dive: JSX vs Templates

React uses JSX ‚Äî JavaScript XML ‚Äî which allows mixing HTML and JavaScript in the same file. This might seem strange at first, but it offers powerful advantages:

Advantages of JSX:

  • Full JavaScript power: loops, conditionals, functions
  • Compile-time errors caught before runtime
  • IDE autocomplete and type checking (TypeScript)
  • No need to learn custom template syntax

Downside: Requires build tool (Webpack/Babel) to convert JSX to JavaScript.

üéØ Real-World Choice Guide

Choose React if:

  • You want maximum flexibility and ecosystem choice
  • Your team knows JavaScript well
  • You're building a dynamic application with frequent UI updates
  • You plan to use React Native for mobile apps later

Choose Angular if:

  • You're building enterprise applications with strict architecture
  • You prefer TypeScript-first development
  • You need built-in solutions for routing, forms, HTTP client

Choose Vue if:

  • You're a beginner and want quick results
  • You're working on smaller projects or prototypes
  • You prefer simplicity and gentle learning curve

üí° Pro Tip: 90% of job postings for frontend roles require React. Mastering React opens doors to React Native, Next.js, and more.

üìå Installation Steps

Visit nodejs.org and download the LTS version.

# Verify installation after download
node --version
# Should show v18.x.x or higher

npm --version
# Should show 8.x.x or higher

Alternative: Use nvm (Node Version Manager) ‚Äî recommended for professionals

# macOS/Linux
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Windows (PowerShell as Administrator)
winget install OpenJS.NodeJS.LTS

# Then install specific version
nvm install 18
nvm use 18

Why nvm? Allows you to switch between Node versions easily ‚Äî essential when working on multiple projects with different requirements.
