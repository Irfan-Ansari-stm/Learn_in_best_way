8.1 Handling Events in React

Event Handling Philosophy

In React, event handling follows a consistent pattern that's similar to HTML but with important differences. Events are named using camelCase rather than lowercase, and you pass a function as the event handler rather than a string.

Core Concept: React events are synthetic events that wrap native browser events, providing cross-browser compatibility and performance optimizations.
Basic Event Syntax

Compare HTML and React event handling:

{`// HTML - Traditional approach <button onclick="handleClick()">Click me</button> // React - Modern approach <button onClick={handleClick}>Click me</button> // HTML - With inline JavaScript <input onkeyup="validateInput(this.value)"> // React - With function reference <input onKeyUp={(e) => validateInput(e.target.value)} />`}

Key Differences:

  • React uses camelCase: onClick, not onclick
  • React passes functions, not strings
  • React events are synthetic (cross-browser compatible)
  • Preventing default behavior requires explicit call to e.preventDefault()
Common Event Types

React supports most standard DOM events with consistent naming:

{`// Mouse Events onClick, onContextMenu, onDoubleClick, onDrag, onDragEnd, onDragEnter, onDragExit, onDragLeave, onDragOver, onDragStart, onDrop, onMouseDown, onMouseEnter, onMouseLeave, onMouseMove, onMouseOut, onMouseOver, onMouseUp // Form Events onChange, onInput, onSubmit, onFocus, onBlur, onReset, onPaste // Keyboard Events onKeyDown, onKeyPress, onKeyUp // Touch Events onTouchCancel, onTouchEnd, onTouchMove, onTouchStart // Selection Events onSelect // Clipboard Events onCopy, onCut, onPaste // Animation Events onAnimationStart, onAnimationEnd, onAnimationIteration // Transition Events onTransitionEnd`}
Simple Event Example

A complete example showing event handling in action:

{`import { useState } from 'react'; function Counter() { const [count, setCount] = useState(0); // Event handler function const handleIncrement = () => { setCount(count + 1); }; const handleDecrement = () => { setCount(count - 1); }; const handleReset = () => { setCount(0); }; return ( <div> <p>Count: {count}</p> <button onClick={handleIncrement}>+1</button> <button onClick={handleDecrement}>-1</button> <button onClick={handleReset}>Reset</button> <!-- You can also use arrow functions directly --> <button onClick={() => setCount(count + 10)}>+10</button> </div> ); }`}
Important: When you write onClick={handleIncrement}, you're passing a reference to the function. When you write onClick={handleIncrement()}, you're calling the function immediately during render!

8.2 SyntheticEvent System

What is SyntheticEvent?

React's SyntheticEvent is a cross-browser wrapper around the browser's native event. It ensures consistent behavior across different browsers and provides additional features.

Core Insight: SyntheticEvent is React's way of abstracting away browser inconsistencies so you don't have to worry about them.
Properties and Methods

SyntheticEvent has the same interface as the browser's native event, plus some additional features:

{`function EventHandler() { const handleClick = (event) => { console.log('Event type:', event.type); // 'click' console.log('Target element:', event.target); // The element clicked console.log('Current target:', event.currentTarget); // Element with event listener console.log('Page X/Y:', event.pageX, event.pageY); // Coordinates relative to page console.log('Client X/Y:', event.clientX, event.clientY); // Coordinates relative to viewport // Prevent default behavior event.preventDefault(); // Stop propagation (bubbling) event.stopPropagation(); // Check for modifier keys console.log('Ctrl key pressed:', event.ctrlKey); console.log('Shift key pressed:', event.shiftKey); console.log('Alt key pressed:', event.altKey); console.log('Meta key pressed:', event.metaKey); }; return ( <button onClick={handleClick}>Click me</button> ); }`}
Property Description
bubbles Boolean indicating if event bubbles up
cancelable Boolean indicating if event can be canceled
currentTarget The element whose event listener triggered the event
defaultPrevented Boolean indicating if preventDefault() was called
eventPhase Which phase of the event flow is being processed
isTrusted Whether the event was generated by a user action
target The element where the event originated
timeStamp Time when the event was created
type Type of event (click, submit, etc.)
Event Pooling and Performance

For performance reasons, SyntheticEvent objects are pooled. This means the event object is reused after the event handler completes.

{`// ❌ BAD - Trying to access event asynchronously function BadHandler() { const handleClick = (event) => { setTimeout(() => { console.log(event.target.value); // ❌ This will be null! }, 1000); }; return <button onClick={handleClick}>Click me</button>; } // ✅ GOOD - Extract values immediately function GoodHandler() { const handleClick = (event) => { const value = event.target.value; // Extract immediately setTimeout(() => { console.log(value); // ✅ This works! }, 1000); }; return <button onClick={handleClick}>Click me</button>; } // ✅ ALSO GOOD - Call persist() function AlsoGoodHandler() { const handleClick = (event) => { event.persist(); // Prevents pooling setTimeout(() => { console.log(event.target.value); // ✅ This works! }, 1000); }; return <button onClick={handleClick}>Click me</button>; }`}
Important: If you need to access event properties asynchronously, either extract the values you need immediately or call event.persist(). Otherwise, you'll get null values due to event pooling.
Why SyntheticEvents Exist

React's SyntheticEvent system solves several real-world problems:

  • Cross-browser compatibility: Different browsers implement events slightly differently
  • Performance optimization: Event pooling reduces memory allocation
  • Consistent API: Same methods and properties across all browsers
  • Security: Normalized event data prevents potential exploits
  • Integration with React's rendering cycle: Events work seamlessly with React's reconciliation process
Real-world analogy: Think of SyntheticEvent as a universal adapter for electrical devices. Instead of having different plugs for each country (browser), React provides one standardized plug (SyntheticEvent) that works everywhere.

8.3 Event Handler Patterns

Pattern 1: Inline Arrow Functions

Useful for simple, one-time handlers, but avoid in loops or frequently rendered components.

{`// ✅ OK for simple cases function Button() { return ( <button onClick={() => alert('Clicked!')}>Click me</button> ); } // ❌ BAD - Creates new function on every render function TodoList({ todos }) { return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={() => deleteTodo(todo.id)}>Delete</button> </li> ))} </ul> ); }`}
Performance Warning: Inline arrow functions create a new function instance on every render, which can cause unnecessary re-renders of child components that receive these functions as props.
Pattern 2: Named Function Declarations

Best for complex logic and better performance.

{`// ✅ GOOD - Named function declaration function TodoList({ todos, onDelete }) { const handleDelete = (id) => { // Complex deletion logic here onDelete(id); }; return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={() => handleDelete(todo.id)}>Delete</button> </li> ))} </ul> ); }`}
Pattern 3: Bind Method (Class Components)

Legacy approach for class components (not needed in functional components).

{`// Class component legacy pattern (avoid in new code) class TodoItem extends React.Component { constructor(props) { super(props); this.handleClick = this.handleClick.bind(this); } handleClick() { this.props.onDelete(this.props.id); } render() { return ( <li> {this.props.text} <button onClick={this.handleClick}>Delete</button> </li> ); } }`}
Pattern 4: Custom Hooks for Event Logic

Extract complex event handling into reusable custom hooks.

{`// Custom hook for form validation function useFormValidation(initialValues, validationRules) { const [values, setValues] = useState(initialValues); const [errors, setErrors] = useState({}); const handleChange = (e) => { const { name, value } = e.target; setValues(prev => ({ ...prev, [name]: value })); // Validate immediately if (validationRules[name]) { const error = validationRules[name](value); setErrors(prev => ({ ...prev, [name]: error })); } }; const handleSubmit = (e) => { e.preventDefault(); // Validate all fields const newErrors = {}; Object.keys(validationRules).forEach(key => { newErrors[key] = validationRules[key](values[key]); }); setErrors(newErrors); // Only submit if no errors if (Object.values(newErrors).every(error => !error)) { // Submit logic here console.log('Form submitted:', values); } }; return { values, errors, handleChange, handleSubmit }; } // Usage function LoginForm() { const validationRules = { email: (value) => !value ? 'Email is required' : !/\S+@\S+\.\S+/.test(value) ? 'Email is invalid' : '', password: (value) => !value ? 'Password is required' : value.length < 8 ? 'Password must be at least 8 characters' : '' }; const { values, errors, handleChange, handleSubmit } = useFormValidation( { email: '', password: '' }, validationRules ); return ( <form onSubmit={handleSubmit}> <input name="email" value={values.email} onChange={handleChange} placeholder="Email" /> {errors.email && <span className="error">{errors.email}</span>} <input name="password" type="password" value={values.password} onChange={handleChange} placeholder="Password" /> {errors.password && <span className="error">{errors.password}</span>} <button type="submit">Login</button> </form> ); }`}
Pattern 5: Event Delegation Pattern

Handle events at a higher level instead of attaching listeners to many elements.

{`// ❌ BAD - Attaching individual handlers to each button function ButtonGroup() { const buttons = ['Home', 'About', 'Contact', 'Services', 'Blog']; return ( <div> {buttons.map(buttonName => ( <button key={buttonName} onClick={() => navigateTo(buttonName)} > {buttonName} </button> ))} </div> ); } // ✅ GOOD - Event delegation with single handler function ButtonGroup() { const handleButtonClick = (event) => { const buttonName = event.target.textContent; navigateTo(buttonName); }; return ( <div onClick={handleButtonClick}> {/* Parent container */} {['Home', 'About', 'Contact', 'Services', 'Blog'].map(buttonName => ( <button key={buttonName}>{buttonName}</button> ))} </div> ); }`}
Performance Benefit: Instead of 5 separate event listeners, we have just one. This is especially beneficial when dealing with large lists or dynamic content.

8.4 Passing Arguments to Event Handlers

Problem: Event Handler Context

You often need to pass additional data to event handlers beyond the event object itself.

{`// ❌ BAD - Can't pass arguments easily function TodoList({ todos }) { const handleDelete = () => { // How do I know which todo to delete?! }; return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={handleDelete}>Delete</button> </li> ))} </ul> ); }`}
Solution 1: Arrow Functions with Parameters

Wrap your handler in an arrow function to pass additional parameters.

{`// ✅ GOOD - Using arrow function to pass arguments function TodoList({ todos, onDelete }) { return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={() => onDelete(todo.id)}>Delete</button> </li> ))} </ul> ); } // Pass multiple arguments function ProductList({ products, addToCart }) { return ( <ul> {products.map(product => ( <li key={product.id}> {product.name} - ${product.price} <button onClick={() => addToCart(product.id, product.price)}>Add to Cart</button> </li> ))} </ul> ); }`}
Performance Note: While this works, it creates a new function on every render. For performance-critical applications, consider using useCallback or other patterns.
Solution 2: bind() Method

Use bind() to create bound functions with preset arguments.

{`// ✅ GOOD - Using bind() method function TodoList({ todos, onDelete }) { const handleDelete = (id) => { onDelete(id); }; return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={handleDelete.bind(null, todo.id)}>Delete</button> </li> ))} </ul> ); } // Another example function ColorPicker({ colors, onSelectColor }) { const handleColorSelect = (color) => { onSelectColor(color); }; return ( <div> {colors.map(color => ( <button key={color} style={{ backgroundColor: color }} onClick={handleColorSelect.bind(null, color)} > {color} </button> ))} </div> ); }`}
Solution 3: Data Attributes (Recommended)

Store data in HTML attributes and retrieve it from the event object.

{`// ✅ BEST PRACTICE - Using data attributes function TodoList({ todos, onDelete }) { const handleDelete = (event) => { const id = event.currentTarget.dataset.id; onDelete(id); }; return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button data-id={todo.id} onClick={handleDelete} >Delete</button> </li> ))} </ul> ); } // Multiple data attributes function ProductList({ products, addToCart }) { const handleAddToCart = (event) => { const productId = event.currentTarget.dataset.id; const price = parseFloat(event.currentTarget.dataset.price); const name = event.currentTarget.dataset.name; addToCart(productId, price, name); }; return ( <ul> {products.map(product => ( <li key={product.id}> {product.name} - ${product.price} <button data-id={product.id} data-price={product.price} data-name={product.name} onClick={handleAddToCart} >Add to Cart</button> </li> ))} </ul> ); }`}
Why this is best:
  • Creates only one event handler function
  • No new functions created on every render
  • Works with React's event delegation
  • Accessible and semantic
  • Easy to debug
Solution 4: useCallback Hook for Optimization

When you need to pass arguments and want optimal performance.

{`import { useCallback } from 'react'; function TodoList({ todos, onDelete }) { // Memoize the handler function const handleDelete = useCallback((id) => { onDelete(id); }, [onDelete]); // Dependencies array return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button onClick={() => handleDelete(todo.id)}>Delete</button> </li> ))} </ul> ); } // Even better - combine with data attributes function TodoList({ todos, onDelete }) { const handleDelete = useCallback((event) => { const id = event.currentTarget.dataset.id; onDelete(id); }, [onDelete]); return ( <ul> {todos.map(todo => ( <li key={todo.id}> {todo.text} <button data-id={todo.id} onClick={handleDelete} >Delete</button> </li> ))} </ul> ); }`}
Important: Use useCallback judiciously. Only memoize handlers when you have proven performance issues. Overusing it can make your code more complex without benefits.
Advanced Example: Complex Event Handling

Handling events with multiple parameters and conditions.

{`function ShoppingCart({ items, updateQuantity, removeItem }) { const handleQuantityChange = (event, itemId) => { const newValue = parseInt(event.target.value, 10); // Validate input if (isNaN(newValue) || newValue < 0) { event.target.value = 0; return; } updateQuantity(itemId, newValue); }; const handleRemove = (itemId) => { if (window.confirm('Are you sure you want to remove this item?')) { removeItem(itemId); } }; return ( <ul> {items.map(item => ( <li key={item.id}> <div> <span>{item.name}</span> <input type="number" min="0" value={item.quantity} onChange={(e) => handleQuantityChange(e, item.id)} /> <span>${item.price}</span> <button onClick={() => handleRemove(item.id)}>Remove</button> </div> </li> ))} </ul> ); }`}

8.5 Form Handling Basics

Why Form Handling Is Special

Forms are unique because they involve user input that needs to be tracked, validated, and submitted. React handles forms differently than traditional HTML forms.

Core Concept: In React, form state should be managed by React state, not by the DOM itself.
Basic Form Structure

Let's create a simple contact form:

{`import { useState } from 'react'; function ContactForm() { const [formData, setFormData] = useState({ name: '', email: '', message: '' }); const handleChange = (event) => { const { name, value } = event.target; setFormData(prev => ({ ...prev, [name]: value })); }; const handleSubmit = (event) => { event.preventDefault(); // Prevent page refresh console.log('Form submitted:', formData); // Here you would typically send data to an API }; return ( <form onSubmit={handleSubmit}> <div> <label htmlFor="name">Name:</label> <input type="text" id="name" name="name" value={formData.name} onChange={handleChange} /> </div> <div> <label htmlFor="email">Email:</label> <input type="email" id="email" name="email" value={formData.email} onChange={handleChange} /> </div> <div> <label htmlFor="message">Message:</label> <textarea id="message" name="message" value={formData.message} onChange={handleChange} ></textarea> </div> <button type="submit">Send Message</button> </form> ); }`}
Handling Different Input Types

React handles various form elements consistently:

{`function AdvancedForm() { const [formData, setFormData] = useState({ name: '', email: '', age: '', gender: 'male', newsletter: false, bio: '', avatar: null }); const handleChange = (event) => { const { name, value, type, checked } = event.target; const inputValue = type === 'checkbox' ? checked : value; setFormData(prev => ({ ...prev, [name]: inputValue })); }; const handleFileChange = (event) => { setFormData(prev => ({ ...prev, avatar: event.target.files[0] })); }; return ( <form> <div> <label>Name: <input type="text" name="name" value={formData.name} onChange={handleChange} /> </label> </div> <div> <label>Email: <input type="email" name="email" value={formData.email} onChange={handleChange} /> </label> </div> <div> <label>Age: <input type="number" name="age" value={formData.age} onChange={handleChange} /> </label> </div> <div> <label>Gender: <select name="gender" value={formData.gender} onChange={handleChange} > <option value="male">Male</option> <option value="female">Female</option> <option value="other">Other</option> </select> </label> </div> <div> <label> <input type="checkbox" name="newsletter" checked={formData.newsletter} onChange={handleChange} /> Subscribe to newsletter </label> </div> <div> <label>Bio: <textarea name="bio" value={formData.bio} onChange={handleChange} ></textarea> </label> </div> <div> <label>Avatar: <input type="file" name="avatar" accept="image/*" onChange={handleFileChange} /> </label> </div> </form> ); }`}
Handling Multiple Forms

Use different state variables or structure for complex form scenarios:

{`function MultiStepForm() { const [step, setStep] = useState(1); const [personalData, setPersonalData] = useState({ firstName: '', lastName: '', email: '' }); const [addressData, setAddressData] = useState({ street: '', city: '', zipCode: '', country: '' }); const handlePersonalChange = (e) => { const { name, value } = e.target; setPersonalData(prev => ({ ...prev, [name]: value })); }; const handleAddressChange = (e) => { const { name, value } = e.target; setAddressData(prev => ({ ...prev, [name]: value })); }; const nextStep = () => { if (step < 2) setStep(step + 1); }; const prevStep = () => { if (step > 1) setStep(step - 1); }; return ( <div> <h2>Step {step} of 2</h2> {step === 1 && ( <form> <input name="firstName" value={personalData.firstName} onChange={handlePersonalChange} placeholder="First Name" /> <input name="lastName" value={personalData.lastName} onChange={handlePersonalChange} placeholder="Last Name" /> <input name="email" value={personalData.email} onChange={handlePersonalChange} placeholder="Email" /> <button type="button" onClick={nextStep}>Next</button> </form> )} {step === 2 && ( <form> <input name="street" value={addressData.street} onChange={handleAddressChange} placeholder="Street Address" /> <input name="city" value={addressData.city} onChange={handleAddressChange} placeholder="City" /> <input name="zipCode" value={addressData.zipCode} onChange={handleAddressChange} placeholder="ZIP Code" /> <input name="country" value={addressData.country} onChange={handleAddressChange} placeholder="Country" /> <button type="button" onClick={prevStep}>Back</button> <button type="submit">Submit</button> </form> )} </div> ); }`}

8.6 Controlled vs Uncontrolled Components

Controlled Components

In controlled components, form data is handled by React state. The component's state is the source of truth.

{`// ✅ Controlled Component - React manages the value function ControlledInput() { const [value, setValue] = useState(''); const handleChange = (event) => { setValue(event.target.value); }; return ( <input type="text" value={value} onChange={handleChange} /> ); }`}
Advantages:
  • Full control over form data
  • Easier validation and sanitization
  • Immediate feedback to users
  • Consistent with React's unidirectional data flow
  • Easier testing
Uncontrolled Components

In uncontrolled components, form data is handled by the DOM itself. You use refs to access form values.

{`// ❌ Uncontrolled Component - DOM manages the value import { useRef } from 'react'; function UncontrolledInput() { const inputRef = useRef(null); const handleSubmit = (event) => { event.preventDefault(); console.log('Value:', inputRef.current.value); // Access DOM directly }; return ( <form onSubmit={handleSubmit}> <input type="text" ref={inputRef} defaultValue="Initial value" /> <button type="submit">Submit</button> </form> ); }`}
When to use Uncontrolled: Rarely! Only when integrating with non-React libraries or when you need direct DOM manipulation.
Comparison Table

Data Source React state DOM

Update Mechanism onChange handler updates state Refs read current value
Validation Easier - happens in state update Harder - requires DOM inspection

Real-World Example: Login Form

Let's build a complete login form using controlled components:

{`import { useState } from 'react'; function LoginForm() { const [formData, setFormData] = useState({ username: '', password: '', rememberMe: false }); const [errors, setErrors] = useState({}); const [isLoading, setIsLoading] = useState(false); const handleChange = (event) => { const { name, value, type, checked } = event.target; const inputValue = type === 'checkbox' ? checked : value; setFormData(prev => ({ ...prev, [name]: inputValue })); // Clear error when field is modified if (errors[name]) { setErrors(prev => ({ ...prev, [name]: '' })); } }; const validateForm = () => { const newErrors = {}; if (!formData.username.trim()) { newErrors.username = 'Username is required'; } if (!formData.password) { newErrors.password = 'Password is required'; } else if (formData.password.length < 6) { newErrors.password = 'Password must be at least 6 characters'; } setErrors(newErrors); return Object.keys(newErrors).length === 0; }; const handleSubmit = async (event) => { event.preventDefault(); if (!validateForm()) { return; } setIsLoading(true); try { // Simulate API call await new Promise(resolve => setTimeout(resolve, 1000)); console.log('Login successful:', formData); // Redirect or show success message } catch (error) { console.error('Login failed:', error); setErrors({ submit: 'Login failed. Please try again.' }); } finally { setIsLoading(false); } }; return ( <form onSubmit={handleSubmit}> <div> <label htmlFor="username">Username:</label> <input type="text" id="username" name="username" value={formData.username} onChange={handleChange} className={errors.username ? 'error' : ''} /> {errors.username && <span className="error-message">{errors.username}</span>} </div> <div> <label htmlFor="password">Password:</label> <input type="password" id="password" name="password" value={formData.password} onChange={handleChange} className={errors.password ? 'error' : ''} /> {errors.password && <span className="error-message">{errors.password}</span>} </div> <div> <label> <input type="checkbox" name="rememberMe" checked={formData.rememberMe} onChange={handleChange} /> Remember me </label> </div> {errors.submit && <div className="error-message">{errors.submit}</div>} <button type="submit" disabled={isLoading}> {isLoading ? 'Logging in...' : 'Login'} </button> </form> ); }`}
Why this approach wins: Every aspect of the form is under React's control. We can validate instantly, show loading states, manage errors, and disable buttons—all with predictable state transitions.

8.7 Form Validation Fundamentals

Why Validation Matters

Form validation ensures data integrity, improves user experience, and prevents errors before they reach your backend.

Golden Rule: Always validate on the frontend for user experience, but never rely on it alone—always validate on the backend too!
Types of Validation

There are several approaches to form validation:

{`// 1. Field-by-field validation (real-time) // Validates each field as user types or loses focus // 2. On-submit validation // Validates all fields when form is submitted // 3. Mixed approach (recommended) // Real-time for obvious errors (like email format) // On-submit for complex rules (like password strength) // Example implementation function ValidationExample() { const [formData, setFormData] = useState({ email: '', password: '', confirmPassword: '' }); const [errors, setErrors] = useState({}); // Real-time validation for email format const validateEmail = (email) => { const emailRegex = /^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$/; return emailRegex.test(email) ? '' : 'Invalid email format'; }; // On-submit validation for password matching const validatePasswords = (password, confirmPassword) => { if (password !== confirmPassword) { return 'Passwords do not match'; } if (password.length < 8) { return 'Password must be at least 8 characters'; } return ''; }; const handleChange = (event) => { const { name, value } = event.target; setFormData(prev => ({ ...prev, [name]: value })); // Real-time validation for email if (name === 'email') { setErrors(prev => ({ ...prev, email: validateEmail(value) })); } }; const handleSubmit = (event) => { event.preventDefault(); // Reset errors setErrors({}); // Validate email const emailError = validateEmail(formData.email); if (emailError) { setErrors(prev => ({ ...prev, email: emailError })); } // Validate passwords const passwordError = validatePasswords(formData.password, formData.confirmPassword); if (passwordError) { setErrors(prev => ({ ...prev, password: passwordError })); } // If no errors, proceed if (!emailError && !passwordError) { console.log('Form submitted successfully!'); } }; return ( <form onSubmit={handleSubmit}> <div> <label>Email: <input type="email" name="email" value={formData.email} onChange={handleChange} /> </label> {errors.email && <span className="error">{errors.email}</span>} </div> <div> <label>Password: <input type="password" name="password" value={formData.password} onChange={handleChange} /> </label> {errors.password && <span className="error">{errors.password}</span>} </div> <div> <label>Confirm Password: <input type="password" name="confirmPassword" value={formData.confirmPassword} onChange={handleChange} /> </label> </div> <button type="submit">Register</button> </form> ); }`}
Validation Rules Library

Create a reusable validation utility:

{`// utils/validation.js export const validators = { required: (value) => value ? '' : 'This field is required', minLength: (min) => (value) => value && value.length >= min ? '' : \`Must be at least \${min} characters\`, maxLength: (max) => (value) => value && value.length <= max ? '' : \`Must be no more than \${max} characters\`, email: (value) => { const emailRegex = /^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$/; return emailRegex.test(value) ? '' : 'Invalid email format'; }, phone: (value) => { const phoneRegex = /^\\+?[1-9]\\d{1,14}$/; return phoneRegex.test(value) ? '' : 'Invalid phone number'; }, url: (value) => { const urlRegex = /^(https?:\\/\\/)?([\\da-z.-]+)\\.([a-z.]{2,6})([/\\w .-]*)*\\/?$/; return urlRegex.test(value) ? '' : 'Invalid URL'; }, matches: (field) => (value, allValues) => value === allValues[field] ? '' : 'Fields do not match' }; // Example usage const validationRules = { email: [validators.required, validators.email], password: [ validators.required, validators.minLength(8), validators.maxLength(128) ], confirmPassword: [ validators.required, validators.matches('password') ] };`}
Advanced Validation with Custom Hooks

Create a comprehensive form validation hook:

{`import { useState } from 'react'; function useFormValidation(initialValues, validationRules) { const [values, setValues] = useState(initialValues); const [errors, setErrors] = useState({}); const [touched, setTouched] = useState({}); const handleChange = (e) => { const { name, value } = e.target; setValues(prev => ({ ...prev, [name]: value })); // Validate on change for real-time feedback if (touched[name]) { validateField(name, value); } }; const handleBlur = (e) => { const { name } = e.target; setTouched(prev => ({ ...prev, [name]: true })); validateField(name, values[name]); }; const validateField = (name, value) => { if (!validationRules[name]) return; const fieldErrors = []; validationRules[name].forEach(rule => { const error = rule(value, values); if (error) fieldErrors.push(error); }); setErrors(prev => ({ ...prev, [name]: fieldErrors.length ? fieldErrors[0] : '' })); }; const validateForm = () => { const newErrors = {}; Object.keys(validationRules).forEach(field => { const fieldErrors = []; validationRules[field].forEach(rule => { const error = rule(values[field], values); if (error) fieldErrors.push(error); }); newErrors[field] = fieldErrors.length ? fieldErrors[0] : ''; }); setErrors(newErrors); return Object.values(newErrors).every(error => !error); }; const resetForm = () => { setValues(initialValues); setErrors({}); setTouched({}); }; return { values, errors, touched, handleChange, handleBlur, validateForm, resetForm }; } // Usage function RegistrationForm() { const validationRules = { email: [ (value) => value ? '' : 'Email is required', (value) => /^[^\\s@]+@[^\\s@]+\\.[^\\s@]+$/.test(value) ? '' : 'Invalid email format' ], password: [ (value) => value ? '' : 'Password is required', (value) => value.length >= 8 ? '' : 'Password must be at least 8 characters', (value) => /[A-Z]/.test(value) ? '' : 'Password must contain uppercase letter', (value) => /[0-9]/.test(value) ? '' : 'Password must contain number' ], confirmPassword: [ (value, values) => value === values.password ? '' : 'Passwords do not match' ] }; const { values, errors, touched, handleChange, handleBlur, validateForm } = useFormValidation({ email: '', password: '', confirmPassword: '' }, validationRules); const handleSubmit = async (e) => { e.preventDefault(); if (!await validateForm()) { return; } // Submit form data console.log('Submitting:', values); }; return ( <form onSubmit={handleSubmit}> <div> <label>Email: <input type="email" name="email" value={values.email} onChange={handleChange} onBlur={handleBlur} /> </label> {(touched.email || errors.email) && <span className="error">{errors.email}</span>} </div> <div> <label>Password: <input type="password" name="password" value={values.password} onChange={handleChange} onBlur={handleBlur} /> </label> {(touched.password || errors.password) && <span className="error">{errors.password}</span>} </div> <div> <label>Confirm Password: <input type="password" name="confirmPassword" value={values.confirmPassword} onChange={handleChange} onBlur={handleBlur} /> </label> {(touched.confirmPassword || errors.confirmPassword) && <span className="error">{errors.confirmPassword}</span>} </div> <button type="submit">Register</button> </form> ); }`}
Pro Tip: This approach separates validation logic from presentation, making your components cleaner and testable.
Accessibility Considerations

Valid forms must be accessible:

{`function AccessibleForm() { const [email, setEmail] = useState(''); const [error, setError] = useState(''); const handleChange = (e) => { const value = e.target.value; setEmail(value); // Clear error when user starts typing if (error) { setError(''); } }; const validateAndSubmit = (e) => { e.preventDefault(); if (!email.includes('@')) { setError('Please enter a valid email address'); return; } // Submit form }; return ( <form onSubmit={validateAndSubmit}> <label htmlFor="email"> Email Address <input type="email" id="email" name="email" value={email} onChange={handleChange} aria-invalid={!!error} aria-describedby={error ? "email-error" : undefined} /> </label> {error && ( <span id="email-error" className="error" role="alert"> {error} </span> )} <button type="submit">Submit</button> </form> ); }`}
Accessibility Best Practices:
  • Use aria-invalid to indicate invalid state
  • Use aria-describedby to link error messages to inputs
  • Use role="alert" for dynamic error messages
  • Ensure keyboard navigation works properly
  • Provide clear instructions for required fields
