25.1 Mobile Development with React Native

React Native is a framework that allows you to build native mobile applications for iOS and Android using JavaScript and React.

Definition
React Native is an open-source framework developed by Facebook (Meta) that enables developers to create native mobile applications for iOS and Android using React, a JavaScript library for building user interfaces. It compiles to native components rather than web views, providing near-native performance.
Detail

Traditional mobile development requires separate codebases:

  • iOS: Swift or Objective-C
  • Android: Java or Kotlin

React Native solves this by allowing you to write one codebase in JavaScript/React that runs natively on both platforms:

  1. You write components using React syntax
  2. React Native translates these components into native iOS and Android components
  3. The app runs with native performance, not in a web view
  4. You can use native modules when needed

Key advantages:

  • Code reuse: Share up to 90% of code between platforms
  • Fast development: Hot reloading speeds up iteration
  • Single team: One team can build for both platforms
  • React ecosystem: Leverage existing React knowledge and libraries
  • Native performance: Uses native components, not web views

It's important to understand that React Native doesn't use web technologies like HTML/CSS under the hood. Instead, it maps React components to their native equivalents:

  • <View> ‚Üí iOS UIView / Android ViewGroup
  • <Text> ‚Üí iOS UILabel / Android TextView
  • <Image> ‚Üí iOS UIImageView / Android ImageView
Cross-platform Development

React Native enables true cross-platform development with platform-specific adaptations:

// Single component works on both iOS and Android
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello World!</Text>
      <Text style={styles.subtitle}>Built with React Native</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f5f5f5',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 10,
    color: '#333',
  },
  subtitle: {
    fontSize: 16,
    color: '#666',
  },
});

When you run this code:

  • On iOS: Renders with San Francisco font, rounded corners, and iOS-specific animations
  • On Android: Renders with Roboto font, material design shadows, and Android-specific transitions

This means your app looks and feels native on each platform while sharing the same logic and structure.

Common patterns:

  • Shared business logic: API calls, state management, authentication
  • Platform-specific UI: Navigation patterns, button styles, layout spacing
  • Feature availability: Camera access, push notifications, biometrics

25.2 Setting up Development Environment

Setting up a React Native development environment involves installing the necessary tools to build, test, and deploy mobile applications on both iOS and Android platforms.

Definition
The React Native development environment consists of the required software tools, SDKs, and configurations needed to create, compile, and run React Native applications on physical devices or emulators for iOS and Android.
Detail

There are two main approaches to setting up React Native:

  1. Bare React Native: Direct setup with native toolchains
  2. Expo: Managed workflow with simplified tooling

For beginners, we recommend starting with Expo because it handles complex setup automatically. We'll cover both approaches.

Core requirements:

  • Node.js: JavaScript runtime
  • npm or yarn: Package manager
  • React Native CLI: Command-line interface
  • Android Studio: For Android development (Windows/macOS)
  • Xcode: For iOS development (macOS only)
  • Emulators/Simulators: Test apps without physical devices

Important note: You need macOS to develop iOS apps. Windows/Linux users can only develop Android apps.

Requirements and Prerequisites

For Android Development (Windows/macOS/Linux):

  • Node.js v18+ (LTS)
  • npm v9+ or yarn v1.22+
  • Android Studio (includes SDK, emulator, and tools)
  • Java Development Kit (JDK) 11+
  • Android SDK Platform-Tools
  • Android SDK Build-Tools

For iOS Development (macOS only):

  • Node.js v18+ (LTS)
  • npm v9+ or yarn v1.22+
  • Xcode 14+
  • Command Line Tools for Xcode
  • CocoaPods (dependency manager for iOS)

For cross-platform development:

  • macOS machine (required for iOS builds)
  • Physical iOS device (recommended for testing)
  • Physical Android device (recommended for testing)
  • GitHub account (for version control)

25.3 Core Components and APIs

React Native provides a set of core components and APIs that allow you to build native mobile applications using JavaScript and React principles.

Definition
Core components in React Native are the fundamental building blocks for creating user interfaces, including View, Text, Image, Button, ScrollView, FlatList, and Touchable components. These components map directly to native iOS and Android UI elements.
Detail

Unlike web development where you use HTML tags, React Native uses its own component system:

  • View: Equivalent to div (container)
  • Text: Equivalent to span/p (text display)
  • Image: Equivalent to img (image display)
  • ScrollView: Scrollable container
  • FlatList: Optimized list component
  • TextInput: Input field
  • Button: Pressable button
  • TouchableOpacity: Touchable element with opacity feedback

Key differences from web:

  • No CSS‚Äîstyling uses JavaScript objects
  • Flexbox is the primary layout system
  • No browser rendering engine
  • Components are native, not HTML elements
  • Navigation is handled differently

Every React Native app must have at least one root component that returns JSX.

Flexbox Layout System

React Native uses Flexbox for layout, which is similar to web CSS but with some key differences:

  • Default direction: Column (not row like web)
  • Default alignment: Center (not left)
  • No float or position properties
  • Units are pixel-based (no % or em)

Essential Flexbox properties:

// Container styles
const containerStyle = {
  flex: 1,           // Takes available space
  flexDirection: 'column',  // Row or column
  justifyContent: 'center', // Main axis: flex-start, center, flex-end, space-around, space-between
  alignItems: 'center',     // Cross axis: flex-start, center, flex-end, stretch
  padding: 20,
};

// Item styles
const itemStyle = {
  flex: 1,           // Grow to fill space
  flexBasis: 100,    // Initial size
  flexGrow: 1,       // How much to grow
  flexShrink: 1,     // How much to shrink
  alignSelf: 'center', // Override parent's alignItems
};

Practical examples:

// Horizontal layout
<View style={{ flexDirection: 'row', justifyContent: 'space-between' }}>
  <View style={{ width: 100, height: 100, backgroundColor: '#00d1ff' }}></View>
  <View style={{ width: 100, height: 100, backgroundColor: '#ffb74d' }}></View>
  <View style={{ width: 100, height: 100, backgroundColor: '#00d1ff' }}></View>
</View>

// Vertical layout
<View style={{ flexDirection: 'column', justifyContent: 'space-around', padding: 20 }}>
  <View style={{ width: 100, height: 50, backgroundColor: '#00d1ff' }}></View>
  <View style={{ width: 100, height: 50, backgroundColor: '#ffb74d' }}></View>
  <View style={{ width: 100, height: 50, backgroundColor: '#00d1ff' }}></View>
</View>

// Center everything
<View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
  <Text>Centered Content</Text>
</View>

Pro tip: Always use flex: 1 on your root View to make it take full screen space.

ScrollView and FlatList

Two ways to render lists in React Native:

ScrollView

Render all items at once‚Äîgood for small lists:

<ScrollView style={{ padding: 10 }}>
  {[...Array(10)].map((_, i) => (
    <View key={i} style={{ 
      backgroundColor: '#00d1ff', 
      padding: 20, 
      marginVertical: 5, 
      borderRadius: 8 
    }}>
      <Text>Item {i + 1}</Text>
    </View>
  ))}
</ScrollView>

FlatList

Render only visible items‚Äîessential for long lists:

import React from 'react';
import { View, Text, FlatList, StyleSheet } from 'react-native';

const data = Array.from({ length: 100 }, (_, i) => ({
  id: i + 1,
  title: `Item ${i + 1}`,
}));

export default function App() {
  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.text}>{item.title}</Text>
    </View>
  );

  return (
    <FlatList
      data={data}
      renderItem={renderItem}
      keyExtractor={(item) => item.id.toString()}
      style={{ flex: 1 }}
      contentContainerStyle={{ padding: 10 }}
    />
  );
}

const styles = StyleSheet.create({
  item: {
    backgroundColor: '#00d1ff',
    padding: 20,
    marginVertical: 5,
    borderRadius: 8,
  },
  text: {
    fontSize: 16,
    color: '#000',
  },
});

Key advantages of FlatList:

  • Memory efficient (only renders visible items)
  • Supports pull-to-refresh
  • Supports infinite scrolling
  • Has built-in performance optimizations
  • Can handle thousands of items smoothly

import React, { useState } from 'react';
import { View, Text, TextInput, FlatList, TouchableOpacity, StyleSheet, Alert } from 'react-native';

export default function TodoApp() {
  const [todos, setTodos] = useState([]);
  const [input, setInput] = useState('');

  const addTodo = () => {
    if (input.trim()) {
      setTodos([...todos, { id: Date.now(), text: input.trim(), completed: false }]);
      setInput('');
    }
  };

  const toggleTodo = (id) => {
    setTodos(todos.map(todo => 
      todo.id === id ? { ...todo, completed: !todo.completed } : todo
    ));
  };

  const deleteTodo = (id) => {
    Alert.alert(
      'Delete Todo',
      'Are you sure you want to delete this todo?',
      [
        { text: 'Cancel', style: 'cancel' },
        { 
          text: 'Delete', 
          style: 'destructive', 
          onPress: () => setTodos(todos.filter(todo => todo.id !== id)) 
        }
      ]
    );
  };

  const renderTodo = ({ item }) => (
    <View style={styles.todoItem}>
      <TouchableOpacity 
        onPress={() => toggleTodo(item.id)}
        style={[styles.checkbox, item.completed && styles.checked]}

        style={styles.deleteButton}
      >
        <Text style={styles.deleteText}>‚úï</Text>

  );

25.4 Navigation in React Native

Navigation in React Native allows users to move between different screens or views within an application, creating a cohesive user experience.

Definition
Navigation in React Native refers to the mechanism that enables users to navigate between different screens or views within a mobile application. This is typically implemented using the React Navigation library, which provides various navigator types like Stack, Tab, and Drawer navigators.
Detail

React Native doesn't have built-in navigation. You must use a third-party library:

  • React Navigation: Most popular, feature-rich
  • React Native Navigation: Native performance, complex setup

We'll focus on React Navigation as it's the industry standard.

Three main navigator types:

  1. Stack Navigator: Sequential navigation (like web pages)
  2. Tab Navigator: Bottom/top tabs for switching between views
  3. Drawer Navigator: Side menu navigation

Tab Navigator creates bottom tabs for switching between primary sections:

// App.js
import * as React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';

import HomeScreen from './screens/HomeScreen';
import SearchScreen from './screens/SearchScreen';
import ProfileScreen from './screens/ProfileScreen';
import Ionicons from 'react-native-vector-icons/Ionicons';

const Tab = createBottomTabNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Tab.Navigator
        screenOptions={({ route }) => ({
          tabBarIcon: ({ focused, color, size }) => {
            let iconName;
            
            if (route.name === 'Home') {
              iconName = focused ? 'home' : 'home-outline';
            } else if (route.name === 'Search') {
              iconName = focused ? 'search' : 'search-outline';
            } else if (route.name === 'Profile') {
              iconName = focused ? 'person' : 'person-outline';
            }
            
            return <Ionicons name={iconName} size={size} color={color} />;
          },
          tabBarActiveTintColor: '#00d1ff',
          tabBarInactiveTintColor: '#999',
        })}
      >
        <Tab.Screen name="Home" component={HomeScreen} />
        <Tab.Screen name="Search" component={SearchScreen} />
        <Tab.Screen name="Profile" component={ProfileScreen} />
      </Tab.Navigator>
    </NavigationContainer>
  );
}

25.5 Platform-specific Code

Platform-specific code allows you to write different implementations for iOS and Android within the same React Native application, ensuring optimal user experience on each platform.

Definition
Platform-specific code in React Native refers to techniques that allow developers to write conditional code or use platform-specific components and styles based on whether the application is running on iOS or Android, ensuring native-like behavior and appearance on each platform.
Detail

While React Native promotes code reuse, sometimes you need platform-specific behavior:

  • Design guidelines differ (Material Design vs Human Interface)
  • API availability varies
  • Performance characteristics differ
  • User expectations vary by platform

Four main approaches:

  1. Platform module: Detect current platform
  2. Platform extensions: Separate files for each platform
  3. Conditional rendering: Render different components
  4. Platform-specific styles: Different styling rules

Follow Apple's Human Interface Guidelines and Google's Material Design for best results.

Definition
Building and deployment in React Native refers to the process of compiling your application code into native iOS (.ipa) and Android (.apk/.aab) packages that can be installed on devices and distributed through the Apple App Store and Google Play Store respectively.
Detail

There are two main workflows:

  1. Expo Managed Workflow: Simplified, cloud-based builds
  2. Bare React Native Workflow: Full control, local builds

  1. Testing: On emulators and physical devices

  3. Building: Generate native binaries
  4. Signing: Apply digital signatures
  5. Uploading: Submit to app stores
  6. Review: App store approval process
  7. Release: Make app available to users

For Android, you need a keystore:

# Generate keystore (one time)
keytool -genkey -v -keystore my-upload-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000

Then configure android/app/build.gradle:

android {
  ...
  signingConfigs {
    release {
      if (project.hasProperty('MYAPP_UPLOAD_STORE_FILE')) {
        storeFile file(MYAPP_UPLOAD_STORE_FILE)
        storePassword MYAPP_UPLOAD_STORE_PASSWORD
        keyAlias MYAPP_UPLOAD_KEY_ALIAS
        keyPassword MYAPP_UPLOAD_KEY_PASSWORD
      }
    }
  }
  ...
}
Publishing to App Store and Google Play

Google Play Store:

  1. Create Google Play Developer account ($25 one-time fee)
  2. Prepare app listing (title, description, screenshots)
  3. Generate signed AAB (Android App Bundle) file
  4. Upload to Google Play Console
  5. Set pricing and distribution
  6. Submit for review
  7. Wait for approval (usually 1-7 days)
  8. Release to production

Apple App Store:

  1. Create Apple Developer account ($99/year)
  2. Enroll in Apple Developer Program
  3. Register App ID in Apple Developer portal
  4. Create App Store Connect record
  5. Prepare app assets (icons, screenshots, descriptions)
  6. Archive app in Xcode
  7. Upload via Transporter or Xcode
  8. Complete metadata in App Store Connect
  9. Submit for review (typically 1-3 days)
  10. Respond to reviewer feedback if needed
  11. Release to App Store

Required assets for both stores:

  • App icon (1024√ó1024 PNG)
  • App screenshots (various sizes)
  • App description (100-4000 characters)
  • Keywords (iOS) / Categories (Android)
  • Privacy policy URL
  • Support contact information

Complete Deployment Workflow with Expo

  1. Finalize App: Test thoroughly on both platforms
  2. Update App Info: Edit app.json:
    3. {
  "expo": {
    "name": "My Amazing App",
    "slug": "my-amazing-app",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "updates": {
      "fallbackToCacheTimeout": 0
    },
    "assetBundlePatterns": [
      "**/*"
    ],
    "ios": {
      "supportsTablet": true
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#FFFFFF"
      }
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
  3. Build App:
    4. # Build for Android
expo build:android -t apk

# Build for iOS
expo build:ios -t simulator  # For testing
expo build:ios -t ipa        # For App Store
  4. Monitor Build Status:
    5. # Check status
expo build:status
  5. Download Builds:
    6. # Download Android APK
expo fetch:android:keystore

# Download iOS IPA
expo fetch:ios:certs
  6. Deploy to Stores:
    • Android: Upload APK/AAB to Google Play Console
    • iOS: Upload IPA to App Store Connect via Transporter
  7. Release:
    • Fill out all app store metadata
    • Submit for review
    • Wait for approval
    • Release to production

Pro tips:

  • Use semantic versioning: MAJOR.MINOR.PATCH
  • Always test on physical devices before release
  • Keep backup of signing keys
  • Use CI/CD for automated builds
  • Monitor crash reports after launch
Uses
  • Startups: Rapid deployment to market
  • Enterprises: Controlled releases with staging environments

  • Agencies: Client app delivery pipelines
